From Representation Learning to World Modeling

Hazel Nam and Lucas Maes trace a research arc: learn useful representations first, then learn latent-space dynamics that can predict, notice violations, and plan actions.

🎓 Hazel (Heejeong) Nam · Lucas Maes ⏱ ~50 min ▶ Watch
Why this matters

A robot does not need to predict every pixel of a leaf shaking in the background. It needs to know what will happen if it pushes the blue ball, whether a bar should fall under gravity, and which action sequence moves the T-block into the goal. The talk's central claim is that intelligence needs representations that preserve predictive structure, not reconstructions that spend capacity on irrelevant detail.

This is the bridge from classic self-supervised representation learning to world models: first learn a compact state; then learn how that state changes under actions. It cross-links naturally with Note 18 on predictive coding and Note 39 on world models for biology.

Running analogy — a mental physics sketchbook

Imagine sketching a little comic strip before you act. You do not render every pixel; you draw the objects, their positions, and arrows for likely motion. JEPA-style world models try to learn that sketchbook: object/state embeddings plus rules for how the sketch updates when you act.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh. Each is one line; skim, and dive into any that feel shaky.

  • Self-supervised learning — train on structure already present in data, such as predicting a masked part or matching paired views; here it supplies the representation before labels or rewards.
  • Contrastive vs. masked prediction — contrastive methods such as SimCLR/CLIP pull matched views together and push mismatches apart; masked methods such as BERT/MAE hide part of the input and predict it.
  • Embedding / latent space — a vector space where the model stores what matters; JEPA predicts there rather than in raw pixels.
  • Energy-based model and collapse — low energy means compatible pairs; without constraints, a latent model can collapse by mapping every input to the same vector.
  • Model-based RL / planning — learn a simulator, roll out candidate actions inside it, and choose actions that reach a goal.

1.From pictures to latent futures

Hazel opens with a simple progression. An autoregressive predictor says: given the previous state, predict the next state. But the world is not only a movie; it is a place where actions intervene. A world model therefore asks, “if I am in state \(s_t\) and take action \(a_t\), what state \(s_{t+1}\) should I expect?”

1.1 World models as action-conditioned simulators

The talk identifies three design questions for a useful world model:

  • State representation: what compact state faithfully reflects the scene?
  • Transition model: what regularities govern change, such as gravity or object contact?
  • Dynamics model: how should the state react to an action?
Key equation — action-conditioned prediction

$$ z_{t+1} \approx f_\theta(z_t, a_t) $$

Here \(z_t\) is the learned latent state at time \(t\), \(a_t\) is the action, and \(f_\theta\) is the learned transition/predictor. The core shift is from pixels \(x_t\) to latents \(z_t = E_\phi(x_t)\).

1.2 JEPA versus pixel reconstruction

Generative world models compare predictions to raw future pixels. JEPA instead encodes both the context and the target, then compares in representation space. Hazel's reason is uncertainty: future pixels contain details a planner may not need, while a good latent can keep the information that is predictive and actionable.

Key equation — JEPA prediction loss

$$ \mathcal L_{\text{pred}} = \left\|p_\theta(E_\phi(x), a) - E_\psi(y)\right\|_2^2 $$

\(x\) is the context, \(y\) is the target or future, \(E_\phi\) and \(E_\psi\) encode them, and \(p_\theta\) predicts the target embedding. If \(y\) contains irrelevant texture, the target embedding can learn to ignore it.

Common misconception

“No decoder” is the whole point. Not quite. The deeper point is selective prediction: the model should assign low energy to plausible futures and high energy to incompatible futures, without being forced to model every unpredictable visual detail.

✓ Check yourself — why is a moving leaf a bad pixel-level target for driving?

Because the leaf may dominate pixel loss while contributing little to the driving decision. A latent target can represent road geometry, objects, and motion while discarding leaf texture or exact flutter.

2.Causal-JEPA: make the model need object interactions

Hazel's Causal-JEPA part asks a sharper question: if the scene is made of objects, can the model learn how objects influence one another? The talk's concrete examples are Push-T, CLEVRER, and PHYRE: pushing a T-shaped block, answering counterfactual video questions, and judging physical plausibility.

2.1 Objects, slots, and masking

Instead of representing an image as many DINO patches, Causal-JEPA uses object-centric representations. Slot attention is the mental model: features are gathered into a fixed number of “baskets,” ideally one per object. The twist is object masking. If one object's future slot is hidden, the predictor cannot succeed by copying or interpolating that object alone; it must attend to relevant other objects.

The monkey-and-banana test

Hazel's motivating example: if the banana is hidden but the monkey keeps chewing, a good world model should infer that the banana is shrinking. If the monkey is hidden while the banana disappears, it should infer that the monkey is eating. Masking turns that intuition into a training pressure.

Key equation — masked object prediction

$$ \mathcal L_{\text{C-JEPA}} = \sum_{(i,t)\in M}\left\|\hat z_{i,t} - z_{i,t}\right\|_2^2 $$

\(i\) indexes an object slot, \(t\) indexes time, \(M\) is the set of masked slots, \(\hat z_{i,t}\) is the predictor's guess, and \(z_{i,t}\) is the target slot embedding.

2.2 Actions as nodes, not pasted-on features

The talk criticizes concatenating the same action embedding onto every visual token. In Causal-JEPA, the action is treated more like another node in an interaction graph. That matches the causal motivation: objects and actions are entities whose relations explain change. Hazel is careful about epistemic status: the method does not recover the true causal graph; it learns temporally directed predictive dependencies under assumptions.

✓ Check yourself — why does masking help more than “object slots only”?

Object slots make the state cleaner, but the model can still learn shortcuts. Masking removes the easy path for selected objects, so the predictor must use an influence neighborhood: the other objects/history/actions sufficient to predict the hidden slot.

Limits named in the talk

Causal-JEPA depends on faithful object-centric encoders. Occlusion, appearing/disappearing objects, slot swapping, and choosing how many slots to mask remain pain points. The “causal” claim is about predictive dependencies, not guaranteed recovery of a true causal graph.

3.LeWorldModel: end-to-end JEPA without the usual tricks

Lucas then asks how to train the JEPA world model itself simply. Prior JEPAs often rely on frozen encoders, exponential moving averages, stop-gradients, masking recipes, or multi-term anti-collapse losses. LeWorldModel's thesis is deliberately minimal: encode the current and next observations, predict the next embedding from current embedding plus action, and add a single regularizer to prevent collapse.

3.1 Avoiding collapse with SIGReg

Collapse is the trivial solution \(E(x)=c\) for every input. Prediction becomes easy, but the representation contains no state. LeWorldModel uses SIGReg, the Sketched Isotropic Gaussian Regularizer: project batch embeddings onto many random one-dimensional directions and push those projected distributions toward a standard Gaussian. By the Cramér-Wold intuition, matching many one-dimensional projections constrains the full high-dimensional distribution.

Key equation — LeWorldModel loss

$$ \mathcal L = \underbrace{\left\|p_\theta(z_t,a_t)-z_{t+1}\right\|_2^2}_{\text{predict the next latent}} + \lambda\,\underbrace{\mathcal R_{\text{SIGReg}}(z_t,z_{t+1})}_{\text{avoid collapse}} $$

Only \(\lambda\) balances the prediction term and the regularizer. The talk emphasizes this as a practical simplification compared with six-term alternatives such as PLDM-style losses.

✓ Check yourself — why would constant embeddings minimize prediction loss?

If every observation encodes to \(c\), then the predictor can always output \(c\). The error \(\|c-c\|^2\) is zero, but the representation has discarded the world state. Anti-collapse regularization forbids that degenerate distribution.

4.Worked example — a JEPA objective in 2-D

Numbers make the latent-prediction idea concrete. Suppose a context embedding predicts the target embedding for the next/hidden part of a scene.

Step-by-step with real numbers

Let the actual target embedding be \(z=[2,1]\) and the predictor output be \(\hat z=[1.5,1.2]\). The squared-error JEPA loss is

$$\begin{aligned} \mathcal L &= \|\hat z-z\|_2^2 \\ &= (1.5-2)^2 + (1.2-1)^2 \\ &= (-0.5)^2 + (0.2)^2 \\ &= 0.25 + 0.04 = 0.29. \end{aligned}$$

Now compare with pixel reconstruction. Imagine the target has two meaningful latent coordinates but also a nuisance pixel detail \(\epsilon\) for texture/noise. Two futures can share \(z=[2,1]\) while having \(\epsilon=0.1\) or \(\epsilon=0.9\). A pixel loss must chase that difference; a latent JEPA loss can ignore it if \(E\) maps both to the same predictive state.

QuantityValueInterpretation
Prediction[1.5, 1.2]The model's sketch of the hidden/future target.
Target embedding[2.0, 1.0]The target encoder's sketch of what actually happened.
Error vector[-0.5, 0.2]Underpredicted coordinate 1, slightly overpredicted coordinate 2.
Loss0.29The training signal for the predictor and, in end-to-end versions, the encoder.
✓ Check yourself — if \(\hat z=[1.8,0.7]\), what is the loss?

\((1.8-2)^2+(0.7-1)^2=(-0.2)^2+(-0.3)^2=0.04+0.09=0.13\). This prediction is closer than the first one.

✓ Check yourself — why can embedding prediction avoid wasting capacity on noise?

The target encoder can map nuisance variations to nearly the same latent state when they do not help predict dynamics or solve downstream tasks. Pixel reconstruction cannot do that because every visible detail contributes directly to the loss.

5.End-to-end trace — one predictive/world-model training step

Follow one masked-frame training example through the pipeline. The shapes are illustrative: \(B\) is batch size, \(K\) object slots or tokens, \(d\) latent width, and \(m\) masked targets.

StepWhat happensShape
0 · inputTake a masked image/video context or current frame \(x_t\), plus action \(a_t\) if this is control.x_t: B×H×W×C; a_t: B×A
1 · encoderEncode the visible context into latent tokens or object slots.z_t: B×K×d
2 · context embeddingKeep visible slots/history; replace selected future/object slots with learned mask tokens.context: B×K×d; mask M: B×m
3 · predictorA transformer predicts target embeddings from context, history, and action-as-node/action-conditioning.z_hat_target: B×m×d
4 · target encoderEncode the masked/future part \(y\) to obtain the target embedding.z_target: B×m×d
5 · lossCompare predicted and actual target embeddings; add anti-collapse regularization when training end-to-end.scalar L = MSE + λR
6 · updateBackpropagate through predictor and, for LeWorldModel-style training, through the encoder too.θ, φ updated
Representation-prediction vs. pixel-reconstruction

Pixel reconstruction asks, “can you redraw the frame?” Representation prediction asks, “can you predict the state that matters?” That is why the learned model can be useful for planning even when it is not a perfect video generator.

6.Planning with learned latents — and what remains open

Once a world model predicts latents, an agent can plan by rolling it forward. Lucas describes sampling or optimizing action sequences, rolling out \(z_{t+1}, z_{t+2},\dots\), and minimizing distance to a goal embedding. This is close in spirit to Dreamer-style model-based RL, but the talk emphasizes JEPA latent prediction rather than reconstructive latent dynamics.

Key equation — latent planning objective

$$ \min_{a_{t:t+H-1}} \sum_{h=1}^{H} d\!\left(z_{t+h}(a_{t:t+h-1}), z_g\right) $$

The planner searches for an action sequence whose rolled-out latent states get close to the goal latent \(z_g\). In the talk, distance can be mean-squared error in latent space.

The empirical claims in the lecture are practical rather than final: Causal-JEPA improves counterfactual/object-dynamics behavior in CLEVRER, Push-T, and PHYRE-style settings; LeWorldModel can be trained end-to-end from pixels with a small model and, in reported tasks, plan much faster than patch-heavy baselines. The frontier is still open: longer horizons, hierarchy, real robotics, partial observability, goal specification, and evaluating whether VLA-style policies really understand action consequences.

✓ Check yourself — why can LeWorldModel plan faster than patch-level DINO World Model?

Patch-level models roll out many tokens and pay transformer attention costs over them. LeWorldModel represents the state with a single/global embedding in the talk, so each rollout step is much cheaper.

Opinion / epistemic status

The speakers argue that physical AI needs world models because reliable interaction requires predicting action consequences. That is a research thesis, not a settled theorem. The evidence is strongest in controlled tasks; real-world robotics, long-horizon hierarchy, and goal specification remain active research problems.

Glossary

TermPlain meaning
World modelA learned simulator that predicts future state, often conditioned on actions.
JEPAJoint-Embedding Predictive Architecture: predict a target's embedding rather than reconstructing raw input.
Representation collapseA degenerate solution where every input maps to the same embedding.
Object-centric representationA latent state organized around objects or slots instead of image patches alone.
Slot attentionAn attention mechanism that binds features to a fixed set of learned object slots.
Influence neighborhoodThe predictively sufficient set of other objects/history/actions needed to infer a masked object state.
SIGRegSketched Isotropic Gaussian Regularizer; pushes latent embeddings toward a non-collapsed Gaussian distribution using random projections.
Model predictive controlPlan by repeatedly rolling out a model over a horizon, taking an action, and replanning.
