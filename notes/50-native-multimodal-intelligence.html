From Language Models to Native Multimodal Intelligence

Victoria Lin's lecture reframes multimodality as language modeling over mixed streams: text, images, audio, video, and actions can be tokens in one learned world, not accessories bolted onto a text model.

🎓 Victoria Lin · Thinking Machines Lab ⏱ ~50 min ▶ Watch
Why this matters

Most of the useful world is not plain text. A robot sees motion, a student reads diagrams, a doctor compares scans and reports, and a designer alternates between words and sketches. The tempting shortcut is: keep a strong language model, attach a vision encoder, and call the result multimodal. Lin's core point is sharper: the next generation of systems needs to learn from multiple modalities from the start, so cross-modal structure is represented inside the model rather than translated at the boundary.

Running analogy

Think of a native multimodal model as a single orchestra score. Text, image patches, audio frames, and action tokens are different instruments, but the conductor reads one score and learns how they enter together. A bolt-on model is more like a translator standing between two separate orchestras: useful, but many timing and harmony cues never reach the shared performance.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh. Each is one line; skim, and jump to the linked note where useful.

  • LLM next-token prediction — a transformer predicts a probability distribution over the next token; here that token may be text, image code, audio code, or an action.
  • Bolt-on multimodal connector — many systems encode an image, project it through an adapter, and feed it to a text LLM; see Note 31 for the late-fusion bridge.
  • Tokenization — converting raw signals into model-sized units; text uses BPE-like pieces, images can use patches or learned codes, audio can use frames/codes.
  • VQ-VAE / discrete image tokens — an image patch embedding can be replaced by the nearest entry in a learned codebook, turning images into IDs the transformer can predict.
  • Early vs late fusion — early fusion trains one model on interleaved modalities; late fusion joins specialized components after they have already learned mostly separate representations.

1.The misconception: multimodal ≠ LLM + image adapter

Suppose a model sees a picture of a cat on a sofa and answers, "the cat is sleeping." In a late-fusion system, the picture may first become a compact visual embedding, then a connector maps that embedding into the language model's token space. The LLM is still mostly a text reasoner being conditioned by a visual summary. That is powerful, and it made many image-question-answering systems practical. But it is not the same as training the model's core on image, text, audio, and video as first-class sequence elements.

1.1 Late/bolt-on fusion vs native/early fusion

The contrast is not "bad vs good." Late fusion is efficient and often excellent for understanding tasks. Native early fusion changes the training philosophy: instead of asking a frozen text model to accept converted visual features, we train one transformer over a mixed stream so that attention, representations, and objectives can couple modalities throughout the depth of the network.

Common misconception

"Multimodal = LLM + an image adapter." That mental model explains many deployed vision-language products, but it misses Lin's target: a native multimodal model treats modalities as part of the same modeling problem, which enables deeper cross-modal reasoning and any-to-any generation.

✓ Check yourself — where does late fusion bottleneck information?

At the connector boundary. The visual encoder and adapter must compress the image into a form the text LLM can use. If spatial layout, fine visual detail, or temporal cues are not preserved in that representation, later text reasoning cannot recover them.

2.One stream, many modalities

Lin's language-model analogy is the simplest doorway: LLMs became powerful by predicting the next symbolic token at scale. Native multimodal models ask whether the same training logic can extend beyond symbols. The first move is to represent each modality as tokens, then train on interleaved sequences such as: [text: describe], [image-code: 1842], [image-code: 77], [text: because], and so on.

2.1 Tokenizing images, audio, and video

Text tokenization is familiar: byte-pair or unigram tokenizers break strings into IDs. Images can be patchified into fixed-size tiles such as 16×16 pixels, then encoded as continuous vectors or quantized through a VQ-VAE codebook into discrete image-code IDs. Audio can be transformed into spectral frames or neural codec tokens. Video can be viewed, approximately, as a sequence of image frames plus time.

Key equation — unified sequence modeling

$$p(x_{1:T}) = \prod_{t=1}^{T} p\!\left(x_t \mid x_{

Here \(x_t\) is the token at position \(t\). The vocabulary is a union of modality vocabularies: ordinary text IDs, image-code IDs, and audio-code IDs. In continuous variants, \(x_t\) may be an embedding segment rather than a discrete ID, but the transformer still processes a sequence.

2.2 Discrete tokens are elegant; continuous embeddings are often stronger

Chameleon tests the clean hypothesis: discretize every modality, then use cross-entropy next-token training on interleaved text and image sequences. This makes image generation look like language generation. But Lin emphasizes the tradeoff: discretization can lose information for image understanding, and image-code generation can be token-inefficient. Continuous encoders such as SigLIP-style features often help understanding; VAE/diffusion-style representations often help generation.

✓ Check yourself — why might one image need hundreds or thousands of tokens?

An image contains dense spatial detail. If each token covers only a small patch or code region, a high-resolution image requires many positions. Compression helps, but too much compression drops information needed for understanding or faithful generation.

3.Worked example — one mixed-vocabulary softmax, by hand

Now compute the smallest honest native-multimodal step. The model has one next-token head over a combined vocabulary: text tokens plus image-code tokens. At this position, it could either continue the sentence or start emitting an image code.

Step-by-step with real numbers

Candidate next tokens and logits:

$$\ell(\text{"cat"})=2.0,\quad \ell(\text{"on"})=0.0,\quad \ell(\text{IMG\_17})=1.0,\quad \ell(\text{IMG\_42})=0.5$$

Softmax normalizer:

$$Z=e^{2.0}+e^{0.0}+e^{1.0}+e^{0.5}=7.389+1.000+2.718+1.649=12.756$$

So the next-token probabilities are:

$$p(\text{"cat"})=\frac{7.389}{12.756}=0.579,\quad p(\text{IMG\_17})=\frac{2.718}{12.756}=0.213$$

$$p(\text{IMG\_42})=\frac{1.649}{12.756}=0.129,\quad p(\text{"on"})=\frac{1.000}{12.756}=0.078$$

The same head can emit a word or an image code. If it emits text for several steps, we get a caption or reasoning trace. If it emits image-code tokens, a decoder can turn those codes into pixels.

✓ Check yourself — why does a shared token space enable any-to-any generation?

Because the output distribution is not restricted to text. The model conditions on any previous mixture of modalities and chooses the next item from a combined set. Repeating that step can produce text→image, image→text, audio→text, or interleaved documents, depending on the prompt and sampled tokens.

✓ Check yourself — if the logit for IMG_17 rose from 1.0 to 2.5, what changes?

The new normalizer is \(e^2+e^0+e^{2.5}+e^{0.5}=7.389+1+12.182+1.649=22.220\). Now \(p(\text{IMG\_17})=12.182/22.220=0.548\), so the model is more likely to switch into image-code generation than to emit "cat".

4.Generation: from understanding-only VLMs to Omni models

Lin separates two broad families. Many state-of-the-art products accept multimodal input but output text: image/video/audio in, answer in words out. Gemini, Qwen, and Kimi-like systems often operate this way in core chat settings. Omni models go further: they can output non-text modalities too, such as GPT-4o-style speech/image interactions, Chameleon-style interleaved documents, or unified systems with diffusion heads.

4.1 Autoregressive image tokens

In a Chameleon-like model, image patches are quantized through a codebook, so image generation becomes next-code prediction. The training loss can be ordinary cross-entropy across both text and image-code tokens.

Key equation — cross-entropy over mixed tokens

$$\mathcal{L}_{AR}= -\sum_{t=1}^{T} \log p_\theta(x_t\mid x_{

Every term asks: given all earlier text and image tokens, how much probability did the model assign to the true next token?

4.2 Diffusion heads and hybrid objectives

Transfusion-style models keep interleaved text-image sequences, but generate image segments with a diffusion objective instead of one discrete code at a time. Text keeps causal autoregression; image segments may use bidirectional attention and denoising. This improves image quality and token efficiency, but it reintroduces the open representation dilemma: the best encoding for understanding may differ from the best encoding for generation.

Key equation — one hybrid training sketch

$$\mathcal{L}=\mathcal{L}_{text\ AR}+\lambda\,\mathbb{E}_{t,\epsilon}\left[\lVert \epsilon-\epsilon_\theta(z_t,t,c)\rVert_2^2\right]$$

\(\mathcal{L}_{text\ AR}\) trains next text tokens. The second term trains an image generator to predict noise \(\epsilon\) from a noisy latent \(z_t\), timestep \(t\), and context \(c\). The weight \(\lambda\) balances modalities.

Why native matters

Early fusion gives the model more places to bind "the red object in the image," "the word red," and "the sound event that follows" into shared computation. That can improve grounding, enable cross-modal transfer, support interleaved planning-before-generation, and train under a single objective instead of a pipeline of handoffs.

✓ Check yourself — why does better understanding help generation more clearly than the reverse?

If a model understands objects, layout, text, and intent better, it can plan a more faithful image or infographic before generating. But spending many tokens learning to generate pixels does not automatically improve image-question answering; Lin notes that evidence for generation→understanding transfer is still limited.

5.End-to-end trace — interleaved input to interleaved output

Follow one concrete request: the user provides a small diagram and asks, "Explain this process, then sketch the corrected version." A native model can keep the explanation and new image codes in one sequence.

0 · interleaved inputUser text is tokenized; the input image is patchified and converted to image-code tokens or continuous image embeddings.[T₁...T₁₂] + [I₁...I₂₅₆]

What the trace reveals

In late fusion, the image often enters as a summary handed to a text model. In early fusion, the image and text tokens coexist through many layers, so the model can repeatedly revise cross-modal correspondences before it emits either words or pixels. One token stream is not cosmetic; it creates more depth for grounding.

6.Challenges and the road ahead

The lecture ends with optimism, but not hype. Native multimodal modeling is active research, not a solved recipe. Lin highlights that today's systems are strongest for digital information processing: charts, PDFs, web images, coding-with-vision, and multimodal chat. Physical-world intelligence still adds spatial-temporal reasoning, real-time perception, control, and embodiment.

6.1 Modality competition and stability

Modalities differ in density and learning dynamics. A text token can encode a concept like "cat"; image/video tokens may spend many positions representing texture, lighting, and redundant frames. If one objective dominates, the model may overfit one instrument and drown out another. Mixture-of-Transformers-style architectures route tokens through modality-specific parameters while preserving joint attention, improving non-text generation and training controllability in Lin's examples.

6.2 Evaluation and compute

Native models are expensive: more modalities mean longer sequences, heavier data pipelines, and more failure modes. Evaluation is also harder than text perplexity. We need to measure grounding, spatial reasoning, generation quality, instruction following, safety, real-time interaction, and whether any-to-any capability genuinely transfers rather than merely coexists.

Speculation, clearly marked

Speculation: the next phase may look less like one universal monolith and more like coherent native systems: shared multimodal backbones, specialized heads or routed parameters for dense modalities, and tighter links to agents/robotics. The likely direction is not "replace language," but use language as one strong abstraction inside richer perception-action loops.

✓ Check yourself — why might next-frame prediction not transfer like next-token prediction?

Language is a compressed abstraction of human knowledge and reasoning, so predicting text often forces semantic structure. Video frames are dense and redundant; predicting the next frame may spend much capacity on local texture and motion rather than reusable reasoning.

Glossary

TermPlain meaning
Native multimodal modelA model trained from the start on multiple modalities, usually with shared sequence modeling or shared backbone computation.
Late fusion / bolt-onA pipeline where a separate modality encoder feeds an adapter or connector into a mostly pretrained text LLM.
Early fusionCombining modalities early in training/architecture so cross-modal interactions happen throughout the model.
Mixed token streamAn interleaved sequence containing tokens or embeddings from different modalities.
VQ-VAE codebookA learned table of discrete codes used to quantize continuous image/audio embeddings into token IDs.
Omni modelA model that can both consume and generate multiple modalities, not only output text.
TransfusionA hybrid approach combining autoregressive text modeling with diffusion-style image generation in a unified transformer setting.
Mixture of TransformersAn architecture using modality-specific transformer parameters while preserving joint attention across modalities.
