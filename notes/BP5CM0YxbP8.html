Transformers in Vision: Tackling Problems in Computer Vision

How a model with almost no built-in vision knowledge can learn images by treating patches like words — and why scale makes that trade worth it.

🎓 Topic: Vision Transformers (ViT) ⏱ ~50 min ▶ Watch on YouTube
Why this lecture matters

Computer vision used to be the kingdom of convolutions. CNNs won because they already assume images are local, spatial, and hierarchical. ViT asks a dangerous question: what if we remove most of those assumptions and let scale teach the model vision? The answer is the modern recipe behind many image, video, detection, and multimodal systems.

Running analogy

Imagine cutting a poster into square sticky-notes and laying them on a table. A CNN scans nearby notes with the same magnifying glass. A Vision Transformer puts all notes in one meeting: every patch can ask every other patch, from the first layer, “are you relevant to understanding this image?”

Prerequisites — 30-second refresher

Skim these; dive into any that feel shaky.

  • Image tensor — an image is \(H\times W\times C\) (height × width × channels, e.g. 224×224×3).
  • Linear projection — multiply a flattened vector by a learned matrix to change its dimension.
  • Self-attention / softmax — tokens attend to each other; scores → weights via softmax.
  • Token — here, one image patch turned into a vector.

1.Why vision was hard for transformers

Start with a concrete image: a dog on grass with a tennis ball in the corner. A human instantly connects the dog, the ball, the posture, and the scene. A model has to solve two opposite problems at once: detect tiny local edges and textures, then combine far-apart evidence into one interpretation.

1.1 CNNs bake in the right guesses

A convolutional neural network is not a blank learner. It is full of inductive biases: assumptions that make learning easier when they match the world.

  • Locality. Early features look at small neighborhoods — edges, corners, color blobs.
  • Translation equivariance. The same filter is applied everywhere, so “cat ear” can be recognized whether it appears left or right.
  • Hierarchy. Stacking convolutions and pooling grows the receptive field: edges → textures → parts → objects.
Intuition

An inductive bias is a head start, not magic. If the task really has local patterns, the model needs fewer examples. This is why classic CNNs are so data-efficient on ImageNet-scale training.

✓ Check yourself — why is translation equivariance useful for images?

Because the identity of an object usually does not depend on its absolute location. If an eye detector works in the top-left of an image, the same detector should work in the center or bottom-right too. Convolution gets this by reusing one filter across all positions.

1.2 So why try transformers at all?

A vanilla transformer has almost none of those image-specific biases. Self-attention sees a set of tokens and, without help, does not know that two neighboring patches are neighbors. That sounds bad — until scale changes the bargain.

  • Generality. The same architecture can process words, patches, video frames, molecules, or trajectories once they are tokenized.
  • Global context from layer 1. A patch in the top-left can attend to a patch in the bottom-right immediately, not after many convolutional layers.
  • Scaling behavior. With enough data and compute, fewer hand-coded assumptions can become an advantage: the model can learn the biases it needs instead of inheriting fixed ones.
The real question

ViT is not “CNNs were wrong.” It is: when data and model size become enormous, is it better to hand-code vision structure or learn it? The lecture’s answer is the key tradeoff: CNNs win small; transformers win at scale.

2.ViT — the core idea

Dosovitskiy et al. made the simplest possible move: treat an image as a sequence. Not a sequence of pixels — that would be too long — but a sequence of fixed-size patches.

2.1 Images become tokens

For a standard ViT-B/16 input, a 224×224 RGB image is split into 16×16 patches. That makes a 14×14 grid: 196 visual tokens. Each patch is flattened and passed through one learned linear projection, just like a word embedding table maps word IDs to vectors.

Patch embedding equation

If the image is \(H\times W\) and patch size is \(P\times P\), the number of patches is

$$N = \frac{HW}{P^2}.$$

Each RGB patch is a vector in \(\mathbb{R}^{P^2C}\), then a learned matrix projects it to the transformer width \(D\).

✓ Check yourself — why not make every pixel a token?

A 224×224 image has 50,176 pixels. Full self-attention would form roughly \(50{,}176^2\) pairwise scores per layer — far too expensive. Patches reduce the sequence to 196 tokens for ViT-B/16, making global attention practical.

2.2 The [CLS] token and position embeddings

Two small details make the sequence usable for classification.

  • [CLS] token. ViT prepends one learnable vector, inherited from BERT. It is not a patch. During self-attention, it gathers information from all patches; its final hidden state becomes the image representation passed to the classifier.
  • Position embeddings. Self-attention is permutation-invariant: without positions, it sees the same bag of patch vectors even if you shuffle the image. ViT adds a learned position vector to each patch embedding so token 17 “knows” where it came from in the grid.
ViT input sequence

$$z_0 = [x_{\mathrm{cls}};\, x_p^1E;\, x_p^2E;\, \ldots;\, x_p^NE] + E_{\mathrm{pos}}$$

The bracket means concatenate the class token and all patch tokens; \(E_{\mathrm{pos}}\) restores spatial order.

Easy mistake

The [CLS] token is not a human-readable label and not the word “class.” It is a learned scratchpad. After many attention layers, the network has trained that vector to summarize the whole image for the MLP head.

✓ Check yourself — what breaks if you remove position embeddings?

The transformer no longer knows spatial arrangement. A shuffled grid of patches would look nearly identical as a sequence. For vision, “eye above nose above mouth” matters; attention alone does not encode that order.

3.Data-hungriness: the central tradeoff

The original ViT result is subtle and important: on ImageNet-scale training, ViT often underperformed comparable CNNs. But when pretrained on huge supervised datasets such as JFT-300M and then transferred, ViT beat strong convolutional baselines.

Key tradeoff

ViT removes the CNN’s helpful visual priors, so it must learn locality, equivariance, and hierarchy from data. With too little data, that is wasteful. With enormous data and compute, the weaker prior becomes flexibility: the model can discover whatever structure best transfers.

This connects directly to the lecture’s emphasis on visual representation learning: pretrain a big model once on broad upstream data, then adapt it to many downstream tasks with limited examples. The practical recipe is not just “use a transformer”; it is scale the data, scale the model, be patient, then transfer.

✓ Check yourself — why can “less bias” be both worse and better?

Less bias means fewer built-in shortcuts. That hurts when data is scarce because the model must rediscover obvious image structure. But it helps at massive scale because the model is less constrained by a hand-designed recipe and can learn task-general structure directly from data.

4.Attention maps and interpretability

Because ViT is built from self-attention, we can inspect which patches attend to which other patches. Early CNN layers mostly see local neighborhoods. ViT layers can place long-range links immediately: a head can connect the dog’s face to the ball, or a wheel to the rest of a bicycle, even when they are far apart in the grid.

Attention rollout

Raw attention in one layer is only a local snapshot. Attention rollout multiplies attention matrices through layers, usually including residual connections, to estimate how information flows from input patches to the final [CLS] representation. It is not a perfect explanation, but it gives a useful map of “which patches influenced the decision.”

Interpretability caveat

Attention is not automatically a faithful causal explanation. A bright attention map says information flowed along that route, not that the model would fail without it. Treat it as a diagnostic lens, then verify with ablations or masking.

✓ Check yourself — what is special about ViT attention in early layers?

It can be global immediately. Even the first transformer block may connect distant patches, while a CNN usually needs several layers before distant pixels can influence each other through an enlarged receptive field.

5.Beyond classification

Once an image is a sequence, many computer-vision tasks can be reframed as token prediction, set prediction, or dense per-pixel prediction.

5.1 DETR: detection as set prediction

DETR uses a CNN backbone plus a transformer encoder-decoder to predict a set of objects directly. Instead of hand-designed anchors and non-maximum suppression pipelines, it learns object queries and matches predicted boxes to ground truth with a bipartite matching loss.

5.2 Segmentation, hybrids, and Swin

  • Segmentation. Models such as SETR show that transformer encoders can support dense prediction: classify every pixel or region, not just one image label.
  • Hybrid CNN + transformer. Many systems use CNNs to extract local features, then transformers for global reasoning — a pragmatic blend of bias and flexibility.
  • Swin Transformer. Swin reintroduces vision structure with hierarchical stages and shifted window attention. It gives transformers CNN-like locality and pyramid features while retaining attention’s flexibility.
The field’s compromise

Pure ViT proves the point: transformers can learn vision from patches. Later models often add back locality, windows, or hierarchy because real systems care about compute, data efficiency, and dense multi-scale outputs.

✓ Check yourself — why does Swin reintroduce windows and hierarchy?

Full global attention over many high-resolution patches is expensive, and dense tasks need multi-scale features. Windowed attention reduces compute; shifted windows let information cross boundaries; hierarchical downsampling creates feature pyramids useful for detection and segmentation.

6.CNN vs ViT

DimensionCNNVision Transformer (ViT)
Inductive biasStrong: locality, translation equivariance, hierarchy via convolution and pooling.Weak: mostly sequence modeling plus learned positions; image structure must be learned.
Data efficiencyUsually better with limited labeled data because the architecture already matches images.Data-hungry; shines after large-scale pretraining and transfer.
Global contextBuilt gradually as receptive fields grow through depth.Available from the first layer through self-attention between any patches.
Compute patternConvolutions scale well on grids; cost grows roughly linearly with pixels for fixed kernels.Full attention costs \(O(N^2)\) in number of patches; patch size/windowing matters.
Typical strengthEfficient visual feature extraction and strong baselines.Scalable pretraining, global reasoning, architecture reuse across modalities.
Memory hook

CNNs are like a skilled local inspector walking the image with a reusable checklist. ViT is like a conference call among all patches: expensive, less guided, but capable of making global connections immediately.

8.Worked numeric micro-example: patchify arithmetic

For the standard ViT-B/16 setting, start with a 224×224 RGB image and cut it into 16×16 patches. The bookkeeping is the whole trick: pixels become a short sequence of patch tokens.

Patchify arithmetic

$$\left(\frac{224}{16}\right)^2 = 14^2 = 196\ \text{patches}$$

Each patch is flattened into

$$16\cdot16\cdot3 = 768\ \text{numbers}.$$

A learned linear layer maps each flattened patch from \(768\) numbers to the transformer width \(d\), for example \(d=768\):

$$\mathbb{R}^{768}\rightarrow\mathbb{R}^{d}.$$

Then ViT prepends one learned [CLS] token:

$$196 + 1 = 197\ \text{tokens}.$$

✓ Check yourself — how many tokens for patch size 32?

\((224/32)^2 = 7^2 = 49\) patch tokens. After prepending [CLS], the transformer receives \(49+1=50\) tokens.

9.End-to-end trace through ViT

Using \(d=768\), one image flows through ViT as a sequence whose length changes once — when [CLS] is added — and whose width stays \(d\) through the encoder.

StepWhat happensShape
1Start with one RGB image.image (224×224×3)
2Split into 16×16 patches and flatten each patch.196 patches (196×768 flattened)
3Apply the learned linear patch embedding.linear patch embed (196×d) = (196×768)
4Prepend the learned [CLS] token.prepend CLS (197×d) = (197×768)
5Add learned position embeddings.+position embeddings (197×d) = (197×768)
6Run \(L\) Transformer encoder blocks.Transformer encoder ×L (197×d) = (197×768)
7Take the final [CLS] row as the image representation.take CLS row (d) = (768)
8Feed that vector to the MLP classification head.MLP head → logits (1000)
What the final [CLS] vector means

The final [CLS] vector is a learned summary of the whole image: after \(L\) rounds of self-attention, it has gathered evidence from all patch tokens and becomes the vector the classifier reads. Position embeddings are needed because attention alone does not know patch order; they tell the model whether a patch came from the top-left, center, bottom-right, and so on.

Glossary

TermPlain meaning
Inductive biasA built-in modeling assumption that helps learning when it matches the data.
ConvolutionA small learned filter applied across an image to detect local patterns everywhere.
Translation equivarianceIf the input shifts, the feature map shifts in the same way; the detector is reused across locations.
Patch tokenA flattened image patch projected into the transformer’s embedding dimension.
[CLS] tokenA learned token whose final hidden state is used as the whole-image representation.
Position embeddingA learned vector added to each token so attention knows where the patch came from.
JFT-300MA very large internal Google image dataset used for supervised pretraining in ViT-scale experiments.
Attention rolloutA method for combining attention matrices across layers to estimate patch influence on the final prediction.
DETRDetection Transformer: frames object detection as direct set prediction.
Swin TransformerA hierarchical vision transformer using shifted local attention windows for efficient dense vision tasks.
