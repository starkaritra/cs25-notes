On the Biology of a Large Language Model — Josh Batson (Anthropic)

We can build LLMs that translate rare languages, reason through medical cases, and write rhymes — yet still not know how the computation happens inside. Batson’s answer: treat a trained model like a biological specimen and build microscopes for its internal mechanisms.

🎓 Josh Batson ⏱ ~50 min ▶ Watch
Why this matters

Black-box behavior is not enough when models become competent enough to receive trust. A model can answer a leap-year question with a mix of true facts and false conclusion; it can stop drawing seven fingers without anyone knowing whether the underlying concept improved; it can give a polished explanation that hides the strategy it actually used. Mechanistic interpretability asks: what internal mechanism produced this output?

Running analogy

Think of Claude as a living cell. Training is the evolutionary process that produced a complex organism; the transformer architecture is only the scaffold. Sparse autoencoders are microscopes, features are cell parts, circuits are pathways, and interventions are like optogenetics: perturb one component and watch the organism’s behavior change.

Prerequisites — 30-second refresher

You will follow this note smoothly if these are fresh. Each item is one line; skim, then jump back to Note 8 on Transformer Circuits if the circuits language feels shaky.

  • Transformer residual stream — each layer reads and writes a vector state; mechanistic interpretability studies which directions get written and why. See Note 8.
  • Activation vector — the list of numbers at one token and layer; a direction in this space can represent a concept or subroutine.
  • Autoencoder — a model trained to compress an input and reconstruct it; here the hidden code is forced to be sparse.
  • Sparsity — only a few code entries are active at once; this matches the hunch that a model uses a small set of concepts on any token.
  • Superposition — models can pack more useful features than dimensions by overlapping them, so one neuron may participate in many unrelated concepts.
  • Circuit — a causally connected set of components that implements a behavior, such as induction heads in Note 8.

1.Biology, not blueprint-reading

Batson starts with a contrast: we know the training algorithm, the architecture, and the matrix multiplications, but that is like saying a cell is “chemistry all the way down.” Correct, yet not explanatory. If Claude translates Circassian from an in-context dictionary or mishandles February 29, we want the internal story: which representations were formed, which pathways used them, and which competing mechanisms won.

1.1 Grown systems need microscopes

Language models are grown, not built. We initialize weights randomly, feed data like nutrients, and optimize toward a loss like sunlight. The result is an organic computational object whose useful mechanisms were not hand-designed. The “biology of an LLM” is therefore an empirical science: collect observations, build instruments, propose mechanisms, perturb the system, and revise the map.

Key equation — the tautology is not the explanation

$$ x_{\ell+1}=x_{\ell}+\operatorname{Attn}_{\ell}(x_{\ell})+\operatorname{MLP}_{\ell}(x_{\ell}) $$

Here \(x_{\ell}\) is the residual-stream activation at layer \(\ell\). This equation says how the transformer updates numbers. It does not yet say whether the model is representing “Texas,” planning a rhyme, suppressing a refusal, or reusing an arithmetic lookup.

Established versus hypothesized

Established: these methods can find human-interpretable components and some causal pathways in real models. Hypothesized: the recovered graph is the full mechanism. Batson is careful: current maps are partial, MLP-focused, sometimes approximate, and attention is not yet fully explained.

✓ Check yourself — why is “it is matrix multiplication” an unsatisfying answer?

Because it does not let you predict behavior under distribution shift, identify a failure mechanism, or know what to change. “Matrix multiplication” is the substrate; a mechanism names the represented variables and causal pathway.

2.Superposition: the model’s filing cabinet is overcrowded

Suppose you open the cell and label every visible blob “neuron.” Early vision models sometimes had pleasing neurons: edge detectors, eye detectors, car detectors. But Batson shows language-model neurons that fire for code, Chinese text, math, and Socrates. The neuron is not always the biological part we should name.

2.1 Why neurons are polysemantic

The reason is superposition: the model may need to represent far more concepts than it has clean orthogonal dimensions. So it packs many feature directions into the same activation space. A single neuron coordinate then becomes a projection of several overlapping features, making it polysemantic — active for many unrelated-looking inputs.

Key equation — features are directions, not necessarily neurons

$$ x \approx \sum_{i\in S} f_i d_i \quad \text{with} \quad |S|\ll m $$

\(x\) is an activation vector, \(d_i\) is a learned feature direction, \(f_i\ge 0\) is that feature’s strength, and \(S\) is the small active set. The model may have \(m\) feature directions even when the activation space has only \(d_{\text{model}}\) dimensions.

Common misconception

“If neurons are messy, the model is uninterpretable.” Not necessarily. The mistake is assuming the visible coordinate basis is the right basis. Biology did not stop at “the tissue looks messy”; it invented stains, microscopes, and cell types.

✓ Check yourself — what would count as evidence that a direction is a real feature?

It activates on semantically related examples, generalizes across surface forms, and causally changes behavior when amplified or suppressed. Example: a code-error feature fires for bugs across languages; suppressing it makes the model miss bugs, while activating it can make the model act as if a bug exists.

3.Sparse autoencoders: dictionary learning as a microscope

Batson’s team collects many activation vectors from Claude, then trains a sparse autoencoder (SAE) to reconstruct each activation as a sparse combination of learned directions. The important move is not “compress the model.” It is “find a dictionary whose entries behave like monosemantic microscope labels.”

3.1 Dictionary learning objective

In classical dictionary learning, we factor a data matrix into dictionary atoms and sparse coefficients. Anthropic scaled this idea with one-layer sparse autoencoders, including very large dictionaries such as tens of millions of features for Claude 3 Sonnet.

Key equation — SAE reconstruction plus sparsity

$$ \mathcal{L}(x)=\left\|x-\hat{x}\right\|_2^2+\lambda\left\|f(x)\right\|_1, \qquad \hat{x}=\sum_{i=1}^{m} f_i(x)d_i $$

The first term demands faithful reconstruction of the original activation \(x\). The \(L_1\) penalty, weighted by \(\lambda\), makes most feature activations \(f_i(x)\) zero. The learned directions \(d_i\) are the candidate “cell parts.”

Why the Golden Gate Bridge example matters

The same feature activates for “Golden Gate Bridge,” translations, an image of the bridge, and indirect mentions like driving from San Francisco to Marin. That is not a mere word detector; it looks like a reusable concept. When Anthropic amplified the feature, “Golden Gate Claude” became bizarrely bridge-obsessed — a vivid feature-clamping demonstration.

✓ Check yourself — why is reconstruction necessary?

If the SAE only produced sparse labels but failed to reconstruct activations, it might be inventing a pretty story unrelated to the model’s computation. Reconstruction forces the dictionary to explain the actual vector the transformer used.

4.Circuits and attribution graphs: from parts to pathways

A list of cell parts is not yet biology. Once features are available, Batson’s team asks how they connect across layers to produce an output. Their attribution graphs trace which active features causally feed into other features and, ultimately, into the next-token logits.

4.1 Case studies from Claude

The lecture’s anchor example is: “The capital of the state containing Dallas is …” The graph shows features for Dallas, states, Texas, state capitals, and Austin. Suppressing or swapping parts changes the answer in coherent ways: California leads to Sacramento, Georgia to Atlanta, the Byzantine Empire to Constantinople. That is evidence for a manipulable abstract pathway, not only a memorized string.

Key equation — causal contribution, not mere correlation

$$ \text{effect}_{i\to y}\;\approx\;\operatorname{logit}_y(x+\alpha d_i)-\operatorname{logit}_y(x) $$

\(d_i\) is a feature direction, \(\alpha\) is an intervention strength, and \(\operatorname{logit}_y\) is the score for output token \(y\). The exact machinery is richer, but the principle is simple: perturb the candidate part and see whether the output changes as the proposed mechanism predicts.

Across the paper and talk, the same microscope reveals several motifs:

  • Multi-step reasoning. Medical diagnosis examples route symptoms through a latent diagnosis such as preeclampsia, then to the requested extra symptom such as visual disturbances.
  • Multilingual shared concepts. Translation-equivalent sentences start language-specific, converge to shared middle-layer concepts like smallness/oppositeness/largeness, then become language-specific again near output.
  • Addition and arithmetic. Claude does not simply run the human carrying algorithm; it often computes parallel facts such as last digit, magnitude band, and lookup-like subcomponents that combine to produce 36 + 59 = 95.
  • Planning ahead. In a rhyming couplet, a “rabbit/bunny” target feature can activate at the newline before the model writes the line that ends in “rabbit.” One-token-at-a-time output can still be guided by future targets.
  • Refusals and hallucination. A generic “I can’t answer” tendency can be suppressed by known-answer features. If the known-answer signal is boosted for a fake name, the model may hallucinate a sport or paper.
  • Jailbreaks and hidden goals. Some jailbreaks exploit timing: the model begins a continuation before the harmful topic is fully recognized. In unfaithful reasoning examples, the model can use a hint to choose an answer while producing an explanation that sounds like independent math.
Common misconception

“Next-token prediction means no planning.” The output is sampled one token at a time, but the forward pass can contain features for future constraints. A poet can choose the destination “rabbit” before writing the words that lead there.

✓ Check yourself — why does multilingual overlap peak in the middle layers?

Early layers still encode language-specific tokens and spelling. Late layers must prepare language-specific output words. Middle layers have room to represent the abstract concept — such as smallness or oppositeness — that translations share.

5.Worked example — superposition and sparse features by hand

Numbers make the superposition idea concrete. We will pack three concept directions into a two-neuron activation space, then show how a sparse code can recover the active concepts.

Step-by-step with real numbers

Imagine two neuron coordinates, so activations live in \(\mathbb{R}^2\). Define three feature directions:

$$ d_1=(1,0),\qquad d_2=(0,1),\qquad d_3=\left(\frac{1}{\sqrt{2}},\frac{1}{\sqrt{2}}\right)\approx(0.707,0.707). $$

Suppose the true active concepts are feature 1 with strength 2 and feature 3 with strength 1:

$$ x=2d_1+1d_3=(2,0)+(0.707,0.707)=(2.707,0.707). $$

If we stare only at neurons, both coordinates are active, so the activation looks like a blend. Raw dot products are also ambiguous:

$$ x\cdot d_1=2.707,\qquad x\cdot d_2=0.707,\qquad x\cdot d_3=\frac{2.707+0.707}{\sqrt{2}}\approx2.414. $$

A sparse reconstruction instead asks for a small active set:

$$ \min_{f\ge0}\left\|x-\sum_i f_i d_i\right\|_2^2+\lambda\|f\|_1. $$

The sparse answer \(f=(2,0,1)\) reconstructs exactly:

$$ 2d_1+0d_2+1d_3=(2.707,0.707)=x. $$

So the right explanation is not “neuron 1 and neuron 2 fired.” It is “feature 1 and feature 3 were active; feature 2 was not.”

✓ Check yourself — if \(x'=d_2+d_3\), what sparse code reconstructs it?

Compute \(x'=(0,1)+(0.707,0.707)=(0.707,1.707)\). The sparse code is \(f=(0,1,1)\): feature 2 and feature 3 are active, feature 1 is off.

✓ Check yourself — why does sparsity yield interpretability here?

Without sparsity, many mixtures can approximate the same 2-D vector, and all three dot products may be positive. The sparse penalty favors the explanation with the fewest active feature directions, matching the assumption that only a small number of concepts are used at one token.

6.End-to-end trace — reading a computation with attribution graphs

Now follow one concrete question through the interpretability pipeline. The prompt is: “The capital of the state containing Dallas is”. The model answers: Austin.

StepWhat happensWhat we see
0 · promptThe text enters the transformer and each token gets embedded into residual-stream vectors.tokens: The / capital / state / containing / Dallas / is
1 · run model, cache activationsForward pass predicts the next token while we save activations at layers and token positions.cached vectors x[layer, token]
2 · decompose with SAEEach dense activation is reconstructed as a sparse set of active feature directions.Dallas feature; state feature; capital-answer feature
3 · build attribution graphTrace backward from the Austin logit to features that causally influenced it, then to features that influenced those.Dallas → Texas; state capital → capital name; Texas → Austin
4 · test the circuitSuppress or swap feature groups in the base model and check whether the output changes coherently.California swap → Sacramento; Georgia swap → Atlanta
5 · interpret cautiouslyGroup related features into a human-readable mechanism, while tracking errors and unexplained components.partial mechanism, not a complete proof
Features are the right unit more often than neurons

The trace works because “Dallas,” “Texas,” and “state capital” can be represented as directions and feature groups that cut across neuron coordinates. If we insisted on individual neurons, the map would look like a messy tissue smear; with sparse features, it starts to look like a pathway.

✓ Check yourself — what would make this trace stronger evidence than a pretty diagram?

Interventions. If suppressing the Texas-related features reduces Austin, or swapping in California produces Sacramento, the graph is not merely descriptive; it has passed a causal test.

7.Why it matters — safety, auditing, steering, and humility

Mechanistic interpretability is not only intellectual curiosity. If a future model is entrusted with high-stakes work, we need ways to audit whether it is using the intended information, whether it is hiding a goal, whether a refusal is calibrated, and whether a hallucination was caused by overconfident recall.

What this microscope can already help with
  • Safety auditing: inspect whether harmful, deceptive, or sycophantic features are active in a behavior.
  • Debugging hallucinations: separate “knows the entity” from “must answer anyway” pathways.
  • Steering: clamp or suppress features to test and sometimes change behavior, as in Golden Gate Claude.
  • Generalization studies: see whether arithmetic, multilingual concepts, or medical abstractions are reused across contexts.
Common misconception

“An attribution graph is the model’s complete thought.” No. Current graphs are lossy scientific instruments. They rely on learned replacement models, sparse dictionaries, human feature labeling, and imperfect reconstruction. Batson’s strongest claim is not that the organism is solved; it is that we can now do real experiments on parts of it.

Epistemic status — a practical inequality

$$ \text{useful map} \;\ne\; \text{complete territory} $$

The recovered features and pathways can be causally useful while still missing attention-mediated selection, redundant backup circuits, or mechanisms that only appear under other prompts. Treat every graph as a microscope image with resolution limits.

✓ Check yourself — why might reasoning models reduce some hallucinations?

Batson’s speculation is that one forward pass must choose whether to answer before all possible checks finish. Letting a model think out loud or spend adaptive compute gives it more chances to verify the answer before committing to the next visible token.

Glossary

TermPlain meaning
Mechanistic interpretabilityReverse-engineering a neural network into human-understandable internal mechanisms that causally explain behavior.
Residual streamThe running vector state in a transformer that attention and MLP blocks read from and write to.
Activation vectorThe numeric state at a token and layer; interpretability methods study directions inside this vector space.
FeatureA direction or pattern in activation space that corresponds to a concept, behavior, or computational subroutine.
Polysemantic neuronA neuron coordinate that activates for multiple unrelated-looking concepts because several features overlap in that coordinate.
SuperpositionPacking more features than dimensions by representing features as non-orthogonal directions in the same space.
Sparse autoencoder (SAE)An autoencoder trained to reconstruct activations using only a small number of active dictionary features.
Monosemantic featureA feature that appears to have one coherent meaning across many activating examples.
Attribution graphA graph of active features and causal influences that traces how an output was produced.
Feature clampingArtificially increasing or decreasing a feature’s activation to test or steer model behavior.
Cross-layer transcoderA replacement-style interpretability model that approximates MLP computations across layers with interpretable sparse features.
Epistemic statusA label for how strongly a claim is supported: established result, plausible interpretation, speculation, or limitation.
