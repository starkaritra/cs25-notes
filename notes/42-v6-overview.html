Overview of Transformers (CS25 V6)

Steven and Karan reopen the transformer story at V6: a tight mechanics recap, then the frontier — reasoning models, agents, multimodality, long context, efficiency, and alternatives such as SSMs.

🎓 Steven and Karan ⏱ ~1 hr ▶ Watch
Why this overview now?

CS25 V6 begins after transformers have moved from “a clever sequence model” to the default substrate for language, vision, biology, speech, code, agents, and scientific modeling. The lecture’s point is not to re-teach every detail — Note 2 does that — but to ask what changed by V6: models now reason longer, call tools, see and hear, remember more context, serve billions of requests, and face hard questions about hallucination, alignment, and whether attention is the final architecture.

Running analogy — a research lab with a shared whiteboard

Think of a transformer as a lab meeting. Each token is a researcher. Attention is who reads whose notes on the shared whiteboard. The MLP is private thinking after the discussion. Long context is a bigger whiteboard; agents let the lab use instruments; SSMs/Mamba ask whether a compact lab notebook can replace rereading the whole board.

Prerequisites — 30-second refresher

Keep these fresh; this note points back to earlier overview/mechanics notes instead of reteaching everything.

  • Dot product — measures vector alignment; here it scores how much a query token should read a key token. See the deep mechanics in Note 2.
  • Softmax — turns raw scores into nonnegative weights summing to 1; it makes attention a weighted average instead of a hard lookup.
  • Matrix-multiply shapes — if tokens are stored as a matrix \\(X\in\mathbb{R}^{T\times d}\\), multiplying by learned weights produces Q/K/V matrices with one row per token.
  • Embedding / token — text, patches, audio chunks, or other inputs become vectors; Notes 26 and 34 are the prior CS25 overview snapshots.

1.Why an overview now?

In the transcript, Steven and Karan open by saying this sixth iteration is about both the architecture behind modern AI and “what’s coming next.” That is the right framing: the field is no longer debating whether transformers work. It is debating where the remaining bottlenecks are.

1.1 The V6 map: from architecture to systems

The lecture arc is: early ML → deep/self-supervised learning → embeddings/RNNs → transformers → pre-training/data → post-training/reasoning → agents/multimodal applications → weaknesses → possible post-transformer architectures. The overview is therefore a state-of-the-field briefing, not only a model diagram.

What changed since the earlier overviews?

Notes 26 and 34 captured earlier snapshots: scaling, general foundation models, and broad applications. V6 adds the frontier stack: test-time compute for reasoning, RL-style post-training, tool-using agents, native multimodality, long-context systems, serving efficiency, and serious interest in state-space / hybrid alternatives.

✓ Check yourself — what is the lecture’s “why now” in one sentence?

Because transformers are now the operating layer of modern AI, and the frontier has moved from “can we scale them?” to “can we make them reason reliably, use tools, handle long/multimodal context efficiently, stay aligned, and perhaps be replaced or augmented?”

2.Fast mechanics recap: self-attention as content-based routing

Suppose the current token is bank. It should route information differently in “river bank” versus “bank account.” Static embeddings cannot solve that alone; the token must read its context. Self-attention is the routing rule: each token creates a query, compares it with other tokens’ keys, and mixes their values.

2.1 The block: communication, then computation

For an input matrix \\(X\in\mathbb{R}^{T\times d}\\), learned projections make \\(Q=XW_Q\\), \\(K=XW_K\\), and \\(V=XW_V\\). A decoder-only transformer masks future positions, so token \\(t\\) can read tokens \\(\le t\\), not future answers.

Key equation — scaled dot-product attention

$$\text{Attention}(Q,K,V)=\text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

Here \\(Q\\) asks “what am I looking for?”, \\(K\\) advertises “what information do I contain?”, \\(V\\) is the information to pass along, and \\(d_k\\) is the key/query width used to keep score magnitudes stable.

Common misconception

“Attention is memory.” More precisely, attention is a read operation over token representations. The memory-like object during inference is the KV-cache: stored keys/values from previous tokens. This distinction matters when comparing transformers with SSMs, which carry a compact recurrent state instead.

✓ Check yourself — why does a decoder-only model need a causal mask?

During next-token prediction, position \\(t\\) must not read future tokens \\(t+1, t+2,\dots\\). The causal mask sets those future scores to \\(-\infty\\) before softmax, giving them zero weight.

3.The current frontier the talk emphasizes

The V6 frontier is a stack. Better base models still matter, but the visible leap comes from how models spend compute at inference time, how they are post-trained, what external systems they can use, what modalities they ingest, and how efficiently they serve long sequences.

3.1 Reasoning models and test-time compute

Standard prompting asks for an answer. Chain-of-thought-style reasoning asks the model to spend more tokens decomposing the problem before answering. Tree-of-thought and search variants spend even more test-time compute by exploring multiple candidate reasoning paths and selecting among them.

Compute shift — not a theorem, a useful accounting identity

$$\text{Total capability budget}\approx C_{\text{pretrain}}+C_{\text{post-train}}+C_{\text{test-time}}+C_{\text{tools}}$$

The point is conceptual: V6 systems no longer rely only on pre-training scale. They also buy performance with reasoning tokens, RL-style feedback, retrieval, tools, and environment interaction.

3.2 RL for reasoning and process supervision

The transcript highlights post-training: RLHF, Direct Preference Optimization (DPO), AI feedback, DeepSeek-style GRPO, and process rewards. The key move is to reward not only the final answer but the trajectory: are the intermediate steps useful, faithful, and robust?

  • Outcome reward: “Did the final answer work?” Useful but sparse.
  • Process reward: “Was each step on track?” More expensive to label, but can reduce reward hacking and brittle reasoning traces.
  • Group-relative updates: compare multiple sampled answers rather than one pair, giving richer preference signal.

3.3 Agents and tool use

An agent wraps a model in a loop: observe the environment, plan, act, inspect results, revise. The lecture’s coding-agent example is concrete: ask the agent to build something; it writes code, runs tests, observes failure, and edits again. Tool use also makes reasoning less purely verbal: a model can call Python, retrieve documents, use a browser, query a database, or operate an app.

✓ Check yourself — why do tools help math and code reasoning?

They move fragile intermediate work into systems with exact execution or retrieval. A model may propose a calculation; Python can execute it. A model may need a fact; retrieval can ground it. The model’s role becomes planning, checking, and integrating, not doing every operation in hidden activations.

3.4 Native multimodality and domain foundation models

The talk surveys transformers beyond language: vision transformers split images into patches; CLIP-style contrastive learning aligns text and image representations; medical and neuroscience models adapt the transformer recipe to data such as fMRI. The V6 frontier is increasingly native multimodality: models trained to reason across text, image, audio, video, action, and structured scientific signals rather than bolting a captioner onto a text model.

Why “native” matters

If the lab only receives written reports, it can discuss images indirectly. If the lab has microscopes, cameras, audio, and instruments in the room, the shared whiteboard can contain grounded observations. Native multimodal training makes the whiteboard itself cross-modal.

3.5 Very long context and memory

Long-context transformers can ingest huge prompts, but the lecture warns that context is not the same as learning. Many systems are still stateless across sessions. Current memory often lives outside the model: vector databases, summaries, structured lists/graphs, or compressed context. The hard question is conflict-aware updating: what should happen when new information contradicts old memory?

Context cost anchor

$$\text{attention scores per head} \in \mathbb{R}^{T\times T},\qquad \text{KV-cache size} \propto L\,T\,d$$

\\(T\\) is sequence length, \\(L\\) is the number of layers, and \\(d\\) is model width. Longer context stresses both computation and serving memory.

3.6 Efficiency: MoE, KV-cache, FlashAttention

Serving frontier models is a systems problem. Three ideas show the pattern:

  • Mixture-of-Experts (MoE) activates only some expert sub-networks per token, increasing parameter count without paying every parameter on every token.
  • KV-cache engineering stores past keys/values for fast autoregressive decoding, but memory bandwidth and cache size become bottlenecks.
  • FlashAttention reorganizes exact attention computation to reduce memory traffic, making attention faster without changing the mathematical result.

3.7 Architecture alternatives: SSMs, Mamba, linear attention, hybrids

The lecture closes by saying CS25 is “Transformers United,” but V6 must think outside the box. State-space models maintain a compressed internal state over time, often scaling linearly with sequence length. Mamba is a selective SSM: the update can depend on the input, improving what the state chooses to keep. Linear attention approximations and kernelized attention try to remove the quadratic score matrix. Hybrids mix attention’s precise lookup with SSM/recurrent efficiency. This previews the V6 SSM talk in Note 44.

✓ Check yourself — what does an SSM trade away relative to attention?

It trades some precise, content-addressed access to every previous token for a compact state that is cheaper to update over long sequences. That can be excellent for streaming and long contexts, but exact recall may be harder.

4.Worked micro-example — two-token scaled dot-product attention

We compute one tiny attention head by hand. Let there be two tokens, key/query width \\(d_k=2\\), and value width 2:

Setup

$$Q=K=\begin{bmatrix}1&0\\0&1\end{bmatrix},\qquad V=\begin{bmatrix}1&2\\3&0\end{bmatrix}$$

Token 1’s value is \\([1,2]\\); token 2’s value is \\([3,0]\\). Because \\(Q=K=I\\), each token matches itself more than the other token.

Step 1 — raw dot products.

$$QK^\top=\begin{bmatrix}1&0\\0&1\end{bmatrix}$$

Step 2 — scale by \\(\sqrt{d_k}=\sqrt{2}\approx1.414\\).

$$\frac{QK^\top}{\sqrt{2}}=\begin{bmatrix}0.707&0\\0&0.707\end{bmatrix}$$

Step 3 — softmax each row. For row 1:

$$\text{softmax}([0.707,0])=\left[\frac{e^{0.707}}{e^{0.707}+e^0},\frac{e^0}{e^{0.707}+e^0}\right]\approx[0.670,0.330]$$

By symmetry, row 2 is \\([0.330,0.670]\\), so

$$W\approx\begin{bmatrix}0.670&0.330\\0.330&0.670\end{bmatrix}$$

Step 4 — weighted values.

$$WV\approx\begin{bmatrix}0.670&0.330\\0.330&0.670\end{bmatrix}\begin{bmatrix}1&2\\3&0\end{bmatrix}=\begin{bmatrix}1.66&1.34\\2.34&0.66\end{bmatrix}$$

Token 1 mostly keeps its own value but borrows some from token 2; token 2 does the reverse. This is content-based routing in numbers.

✓ Check yourself — why divide by \\(\sqrt{d_k}\\)?

As \\(d_k\\) grows, random dot products tend to have larger variance. Without scaling, softmax can become too peaky early: one token gets almost all the weight and gradients become less useful. Dividing by \\(\sqrt{d_k}\\) keeps score magnitudes in a range where softmax can compare alternatives smoothly.

5.End-to-end trace — one decoder-only block

Trace a concrete sequence of \\(T=4\\) tokens through one decoder-only block with model width \\(d=8\\), one attention head for simplicity, and \\(d_k=8\\). Real models use many heads and many stacked blocks; the shape logic is the same.

StepWhat happensShape
0 · inputToken IDs become embeddings; positional information is added.tokens+pos: 4×8
1 · QKVThree learned linear maps produce queries, keys, and values.Q,K,V: 4×8
2 · causal scoresCompute \\(QK^\top/\sqrt{8}\\); mask future positions.scores: 4×4
3 · softmaxEach row becomes attention weights over allowed previous/current tokens.weights: 4×4
4 · attention outputWeights multiply values, mixing information across positions.attn: 4×8
5 · residual + normAdd the attention result back to the block input, then normalize.4×8
6 · MLPApply a position-wise feed-forward network: private computation per token.4×8 → 4×m → 4×8
7 · residualAdd the MLP result back; pass to the next block or output head.4×8
Communication vs computation

Attention is communication: tokens read from other tokens on the shared whiteboard. The MLP is computation: each token transforms what it now knows. Decoder-only language models alternate these two moves dozens of times, then predict the next token from the final representation.

✓ Check yourself — where does the \\(T\times T\\) bottleneck appear?

At the score and weight matrices. For \\(T=4\\), it is tiny: \\(4\times4\\). For million-token contexts, the full pairwise matrix is the core reason vanilla attention is expensive.

6.Open problems after V6

The lecture does not present “AGI solved.” It ends with weaknesses and research directions. The useful way to remember them is by asking what the lab still cannot do reliably.

  • Reasoning reliability — longer chains can help, but they can also be unfaithful, brittle, or optimized for looking plausible.
  • Efficiency and serving — frontier models are expensive to train and run; context length, KV-cache memory, latency, and energy all matter.
  • Context length and memory — a huge prompt is not the same as durable learning. Systems need reliable updating, forgetting, and conflict resolution.
  • Evaluation — static benchmarks saturate; agentic, multimodal, and long-horizon tasks need better tests and clearer failure definitions.
  • Alignment and safety — models can reward-hack, take shortcuts, generalize badly out of distribution, or produce explanations that are not faithful to the actual computation.
  • World models and alternatives — JEPA-style world modeling, SSMs/Mamba, linear attention, and hybrids ask whether next-token prediction plus attention is enough.
Common misconception

“More context means the model knows more.” Context can provide temporary evidence, but it does not necessarily update the model’s weights or long-term behavior. The transcript explicitly distinguishes inference-time context/memory from true continual learning.

✓ Check yourself — hallucination according to the talk is not merely “wrong text.” What is the deeper framing?

The talk frames hallucination through world models: a model has an internal learned world model, while the task has a reference world model plus a view and conflict policy. A hallucination is a claim false relative to that task-grounded reference, not simply any surprising output.

Glossary

TermPlain meaning
Self-attentionA mechanism where each token computes how much to read from other tokens in the same sequence.
Decoder-only transformerA transformer architecture trained to predict the next token using only previous/current tokens.
Test-time computeExtra computation spent during inference, such as longer reasoning, search, verification, or tool calls.
Process supervisionTraining signal that evaluates intermediate reasoning steps, not only final answers.
AgentA model-driven loop that observes, plans, acts with tools, observes results, and iterates toward a goal.
KV-cacheStored keys and values from previous tokens used to speed autoregressive decoding.
MoEMixture-of-Experts: a model where only selected expert sub-networks activate for each token.
FlashAttentionAn exact attention algorithm optimized to reduce memory traffic and improve speed.
State-space model (SSM)A sequence model that updates a compact state over time instead of attending to every past token.
MambaA selective SSM architecture that makes state updates input-dependent, improving long-sequence modeling.
World modelA representation of how an environment or domain works, used for prediction, planning, and judging hallucination.
