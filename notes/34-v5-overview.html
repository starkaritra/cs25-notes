STANFORD CS25 · TRANSFORMERS UNITED · V5

Overview of Transformers (CS25 V5, 2025)

The V5 opener resets the transformer map for 2025: the mechanics are familiar, but the frontier has moved toward reasoning, post-training, agents, multimodality, long context, efficiency, and architectures that ask whether attention is the final answer.

🎓 Steven, Karan, Chelsea, and Jenny (CS25 V5 instructors) ⏱ ~1 hr ▶ Watch
Why this matters

In earlier CS25 versions, an overview could still feel like an architecture lecture. In 2025, transformers are the operating substrate for chat assistants, code models, image/video systems such as Sora-like generators, scientific models, medical-imaging pipelines, and tool-using agents. The lecture therefore asks a sharper question: if the block is now familiar, where is progress actually coming from?

Running analogy

Keep one picture: a transformer is a research lab. Tokens are researchers at desks; attention is who each researcher consults; the residual stream is the shared whiteboard; the MLP is private scratch-work. In 2025 the lab has gained a budget for overtime, external tools, new sensors, specialist teams, and a manager that can reward correct reasoning.

Prerequisites — 30-second refresher

This note keeps mechanics tight. For the deeper pass, revisit Note 2; for the V4 overview, compare Note 26.

  • Attention / dot product / softmax — dot products score query-key matches; softmax turns scores into routing weights; values are mixed by those weights.
  • Chain-of-thought — ask or train a model to emit intermediate reasoning tokens before the final answer; these tokens cost inference compute but can improve hard-task accuracy.
  • RL / reward — reinforcement learning adjusts behavior toward outputs with higher reward; in reasoning, rewards are often verifiable answers such as math/code correctness.
  • MoE — mixture-of-experts routes each token through a few specialist subnetworks instead of activating every parameter; see Note 5 and Note 29.

1.Why an overview now?

The lecture begins with the reason CS25 exists: transformers moved from a research architecture to a basic technology of daily life. The 2025 twist is that the obvious scaling story is no longer enough. The instructors split the opener into mechanics, pre-training/data strategy, post-training, applications, and remaining weaknesses.

1.1 What changed by 2025

Karan's quick mechanics pass names models such as GPT-4o, o3, and DeepSeek alongside vision, speech, biology, and video systems. Steven then emphasizes data strategy: not only how much data, but what kind, in what mixture, and at what phase. Chelsea's section shifts to post-training: chain-of-thought, tree-of-thought, program-of-thought, RLHF, DPO, RLAIF, GRPO, self-refinement, ReAct, and tree-search agents.

The big update from V4 is therefore not “transformers still work.” It is that many frontier gains now come from training and inference procedures around the block: better data curricula, better rewards, more deliberate reasoning at test time, and systems that connect the model to tools and memory.

1.2 The course map

The instructors also frame the quarter: guest lectures will cover state-of-the-art transformers across language, vision, biology, robotics, agents, and more. This note is the map you keep open while watching the rest: it tells you which ideas are core mechanics and which ideas are 2025 frontier layers.

✓ Check yourself — name two non-architecture sources of progress in the lecture.

Data strategy during pre-training and post-training with feedback/RL are two. Test-time compute, tool use, retrieval, and multimodal data are also valid.

2.Fast mechanics recap: tokens ask each other for context

Suppose the token bank appears in “the canoe reached the bank.” A static embedding would blur riverbank and financial bank. A transformer lets the token consult its neighbors, so meaning becomes contextual.

2.1 Self-attention as content-based routing

Each token creates three learned views: a query (“what am I looking for?”), a key (“what do I match?”), and a value (“what information do I carry?”). In the lab analogy, a researcher writes a question, scans everyone else's abstracts, and copies useful notes from the best matches.

Key equation — scaled dot-product attention

$$\text{Attention}(Q,K,V)=\text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

\(Q\) is the matrix of queries, \(K\) keys, \(V\) values, and \(d_k\) the key/query width. \(QK^\top\) gives every token-to-token match score; division by \(\sqrt{d_k}\) keeps scores numerically tame; softmax converts scores to weights; multiplying by \(V\) returns the routed information.

2.2 Decoder-only blocks

The modern LLM core is usually decoder-only: tokens flow through masked self-attention, an MLP, residual connections, normalization, and then repeat for many layers. At the end, the model predicts the next token, appends it, and runs again.

Multi-head attention is several committees in the lab asking different questions at once: one head might route syntax, another entity identity, another arithmetic setup. More layers let the shared whiteboard accumulate progressively more abstract features.

Common misconception

“Attention is a database lookup.” It is closer to differentiable, content-based routing. The model does not pick one earlier token; it forms a weighted mixture, and those weights are learned indirectly through next-token prediction and later post-training.

✓ Check yourself — why does a decoder-only model need a causal mask?

During next-token training and generation, position \(t\) must not read tokens after \(t\). The causal mask blocks future positions so the model learns to predict from the prefix only.

3.The 2025 frontier: the block is stable, the workflow is changing

The lecture's deepest 2025 story is that raw pre-training still matters, but many headline improvements now happen after or around it: at post-training time, at inference time, and at system-design time.

3.1 Reasoning models and test-time compute

Reasoning models such as OpenAI's o1/o3 family and DeepSeek-R1 popularized a new scaling axis: spend more compute while answering. Instead of one short completion, the model may generate longer reasoning traces, try multiple paths, verify an answer, or search over candidate solutions.

In the lab analogy, this is overtime. A researcher can answer immediately, or the lab can fund more scratch-work, more independent attempts, and more checking. The promise is better math/code/problem-solving; the cost is latency, tokens, and money.

Inference-time scaling — independent attempts

$$\text{pass@}k = 1-(1-p)^k$$

If one attempt solves a verifiable problem with probability \(p\), then \(k\) independent attempts fail only if all \(k\) fail. This is not a complete theory of reasoning models, but it captures why “try more / think longer” can buy accuracy.

3.2 RL for reasoning: from preference to verifiable reward

The transcript reviews RLHF, DPO, RLAIF, and then GRPO, the DeepSeek-associated method that ranks groups of responses rather than only pairs. The reasoning-specific idea is especially powerful when the reward is verifiable: a math answer is correct, a unit test passes, a proof checker accepts, or a code solution matches expected behavior.

Preference rewards ask “which answer do humans like?” Verifiable rewards ask “did the attempted solution work?” That can scale supervision for reasoning because the system can generate many attempts and score them with a checker.

✓ Check yourself — why are math and code attractive for RL reasoning?

They often provide cheap, objective feedback: a final numeric answer can be checked, and code can be run against tests. That makes reward less dependent on slow human preference labels.

3.3 Agents, tools, and memory

Chelsea defines an AI agent as a system that perceives an environment, chooses actions toward a goal, acts iteratively, tracks state, uses memory, and may call tools. ReAct combines reasoning with actions such as retrieval or API calls; Language Agent Tree Search expands this into multiple planning branches with self-reflection and search.

In lab terms, the transformer is no longer only a thinker. It can ask a calculator, call a search engine, query a database, run code, revise the plan, and keep notes for the next step. The unresolved problem is reliability: an agent that can act is useful only if its plans, tool calls, and self-evaluations are trustworthy.

3.4 Multimodal and native multimodality

The lecture moves from language to vision transformers, CLIP-style image-text alignment, GPT-4o-like vision-language systems, video, speech, and neuroscience/fMRI applications. The key trick is tokenization: images become patches, brain time series become parcels over time, audio/video become sequences, and attention routes across them.

The 2025 frontier is native multimodality: not a text model with a bolt-on image encoder, but a system trained to reason across text, images, audio, and video as first-class streams. That matters for embodied systems, medical diagnostics, tutoring, and real-time communication.

3.5 Long context and efficiency

Long context promises models that can read books, repos, records, and long videos. But vanilla attention scales quadratically in sequence length because it forms all pairwise scores. Efficiency work therefore becomes part of the frontier, not an implementation detail.

  • KV-cache stores previous keys and values during generation so each new token does not recompute the whole prefix.
  • FlashAttention computes exact attention with memory-aware tiling, reducing bandwidth bottlenecks on GPUs.
  • MoE activates only a subset of experts per token, increasing parameter count without paying full dense compute every time.
  • Distillation and small models bring capabilities closer to phones, local devices, and lower-cost deployment.
Why long context is expensive

$$\text{attention score matrix size} = T\times T \quad \Rightarrow \quad \mathcal{O}(T^2)\text{ pairwise scores}$$

For sequence length \(T\), every token can score every other token. Doubling context roughly quadruples the raw score matrix, before hardware-aware optimizations.

3.6 Alternatives to attention: preview of V6

Steven explicitly points beyond the transformer: if pre-training scale is saturating, new architectures may matter again. State-space models such as Mamba replace explicit all-pairs attention with learned recurrent state updates that can process long sequences more efficiently. They are not a drop-in answer to every LLM problem, but they reopen the architectural question for V6: do we need attention everywhere, or only where content-addressable routing is worth the cost?

✓ Check yourself — when would attention be worth its quadratic cost?

When a token may need to retrieve information from many arbitrary earlier positions by content, such as resolving references, comparing distant facts, or cross-modal alignment. If local or compressible state is enough, state-space or other efficient alternatives may be attractive.

4.Worked micro-example — test-time compute by the numbers

Take a verifiable math problem. Suppose one independent reasoning attempt solves it with probability \(p=0.4\). If we sample \(k\) attempts and accept any verified correct answer, the only way to fail is for every attempt to fail.

Step-by-step with real numbers

$$\text{pass@}k = 1-(1-p)^k$$

For \(p=0.4\), the failure probability per attempt is \(1-p=0.6\).

$$\text{pass@}1 = 1-0.6^1 = 1-0.6 = 0.40$$

$$\text{pass@}3 = 1-0.6^3 = 1-0.216 = 0.784 \approx 0.78$$

$$\text{pass@}5 = 1-0.6^5 = 1-0.07776 = 0.92224 \approx 0.92$$

So “think longer / try more” can trade inference compute for accuracy: five attempts are much more likely to contain a correct solution, but they can cost roughly five times the generated reasoning tokens before verification overhead.

✓ Check yourself — what is the cost tradeoff if each attempt emits 800 reasoning tokens?

One attempt costs about 800 reasoning tokens; three cost about 2,400; five cost about 4,000, plus verifier overhead. Accuracy rose from 0.40 to about 0.92, but latency and token cost also rose substantially.

5.End-to-end trace — one decoder-only block plus the reasoning loop

Follow one prompt prefix through a decoder-only transformer. The shapes use \(T\) tokens and hidden width \(d\).

0 · inputTokenize the prompt and add positional information.tokens+pos (T×d)
1 · QKVLinear projections create queries, keys, and values for each head.Q,K,V (T×d_k per head)
2 · causal scoresCompute \(QK^\top/\sqrt{d_k}\) and mask future positions.scores (T×T)
3 · softmaxTurn scores into routing weights over allowed previous tokens.weights (T×T)
4 · attention outputMix values using the weights, concatenate heads, project back.attn (T×d)
5 · residualAdd attention output to the shared stream.stream (T×d)
6 · MLPApply position-wise nonlinear transformation: private scratch-work for each token.MLP (T×d)
7 · residualAdd MLP output back; repeat blocks.stream (T×d)
8 · logitsMap final hidden states to next-token probabilities.vocab logits
9 · reasoning loopGenerate reasoning tokens, optionally sample more attempts, verify, then emit final answer.... → reasoning tokens → answer
Train-time vs test-time compute

Train-time compute changes the lab's weights: it makes the researchers better before deployment. Test-time compute gives the same lab more time, scratch-paper, branches, and tool calls for one problem. Reasoning models exploit both, but the 2025 frontier made the second knob much more visible.

✓ Check yourself — where does the model “learn” in this trace?

During ordinary inference, weights do not update; the trace only creates activations and new tokens. Learning happens during pre-training, fine-tuning, RL, or explicit continual-learning/model-editing methods.

6.Open problems: what the lecture says is still missing

The instructors close by naming weaknesses that still separate today's systems from robust, human-like intelligence. The list is not anti-transformer; it is the research agenda that makes the rest of CS25 necessary.

6.1 Reliability, hallucination, and evaluation

Reasoning traces can be persuasive without being correct. Agents can plan confidently and still call the wrong tool. Multimodal systems can ground poorly. Better evaluations must test not only final answers, but calibration, robustness, long-horizon behavior, and whether a model knows when it is uncertain.

6.2 Cost and controllability

Test-time compute improves some tasks but can make inference expensive. Long context and large multimodal inputs raise memory and latency costs. Interpretability and mechanistic work matter because developers need to know why a model produced an answer, not only that it produced one.

6.3 Alignment and continual learning

The lecture highlights alignment, pluralistic preferences, ethical reasoning, and the lack of true continual learning. RAG can retrieve new facts, but it does not update the model's “brain.” Fine-tuning updates weights but risks catastrophic forgetting. Model editing, MEMIT, lifelong MoE, prompt memories, and progressive prompts are attempts to close this gap, each with limits.

Common misconception

“If we add enough reasoning tokens, reliability is solved.” More attempts can raise pass rates when a verifier exists, but many real tasks lack clean verification. For open-ended advice, safety, medicine, law, and long-horizon agency, more fluent reasoning can still be wrong or misaligned.

✓ Check yourself — why is continual learning harder than RAG?

RAG adds an external memory lookup; continual learning changes the model's internal capabilities or facts without destroying old skills. That requires targeted, stable updates and protection against catastrophic forgetting.

Glossary

TermPlain meaning
Self-attentionA way for tokens in the same sequence to route information to each other by learned similarity.
Decoder-only modelA transformer language model that predicts the next token from previous tokens using causal masking.
Test-time computeExtra computation spent during inference: longer reasoning traces, multiple samples, search, or tool calls.
Chain-of-thoughtIntermediate reasoning tokens generated before a final answer.
GRPOGroup Relative Policy Optimization, an RL method that compares groups of outputs and was popularized by DeepSeek reasoning work.
VerifierA checker that can score whether an answer or action succeeded, such as tests, exact answers, or proof validation.
AgentA model-centered system that pursues goals through iterative decisions, memory, and actions such as tool calls.
Native multimodalityTraining and operating across modalities such as text, image, audio, and video as first-class inputs/outputs.
KV-cacheStored keys and values from previous tokens that make autoregressive generation faster.
FlashAttentionA memory-efficient exact attention algorithm that reduces GPU memory traffic.
MoEMixture-of-experts: route each token through selected specialist subnetworks instead of one dense network.
State-space modelAn alternative sequence architecture that maintains a learned recurrent state rather than all-pairs attention.
