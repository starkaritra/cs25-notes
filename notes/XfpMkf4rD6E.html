Introduction to Transformers

Andrej Karpathy's mechanics-first view: a transformer is a message-passing computer over tokens — communication by attention, computation by MLPs.

🎓 Presenter: Andrej Karpathy ⏱ ~1h15m ▶ Watch on YouTube
Why this lecture matters

Most transformer explanations begin with a wall of tensor shapes. Karpathy begins one level lower: what computation is the model trying to perform? His answer is simple and durable. Each token is a node in a graph; attention is the rule by which nodes exchange information; the MLP is the per-node thinking step after communication.

Running analogy

Imagine a seminar room where every word in a sentence is a student holding a private notebook. On each round, a student asks a question (query), others advertise what they know (keys), and the student copies useful notes from them (values). The whole transformer is many rounds of this seminar: talk, think, talk, think.

Prerequisites — 30-second refresher

Skim these; dive into any that feel shaky.

  • Dot product — \(a\cdot b=\sum_i a_i b_i\); a scalar for how aligned two vectors are (attention's similarity score).
  • Softmax — scores → positive weights summing to 1.
  • Matrix multiply shapes — \((T\times d)(d\times k)=(T\times k)\); tracking shapes is how you read attention.
  • Vector / embedding — a token represented as a length-\(d\) vector.

1.The short history: why attention was needed

Note 1 covers the historical arc in detail, so keep only the motivation here. Before 2017, sequence models were mostly recurrent: RNNs, LSTMs, GRUs, and encoder-decoder seq2seq systems for translation. They could process variable-length text, but their computation was a long thin chain.

1.1 The seq2seq bottleneck

Concrete example first. Suppose the English sentence is "I grew up in France ... I speak fluent ___". A recurrent model must carry the clue France through many updates before it can predict French. In translation, the problem is worse: the whole source sentence was often compressed into one fixed vector before decoding.

Bahdanau's fix

Do not force the decoder to remember the source sentence through one vector. Keep all encoder states around, and let the decoder soft-search them whenever it needs a word. This was Bahdanau attention: a softmax-weighted average over source states. The 2017 Transformer kept the attention idea and removed the recurrence.

✓ Check yourself — what did attention fix before transformers existed?

It fixed the encoder bottleneck. Instead of translating from one compressed vector, the decoder could dynamically look back at all source states and blend the relevant ones for each output word.

2.Attention as message-passing on a directed graph

Karpathy's central framing: attention is a communication phase. Forget text for a moment. We have a directed graph; each node stores a vector; edges say who is allowed to send information to whom. One attention layer performs one round of data-dependent message passing.

2.1 Query, key, value: three views of the same token

Each token vector is projected into three smaller vectors:

  • Query \(q\): "what am I looking for?"
  • Key \(k\): "what information do I contain?"
  • Value \(v\): "what message will I send if someone listens to me?"

Those are just learned linear maps: \(q=xW_Q\), \(k=xW_K\), \(v=xW_V\). The names are only helpful after the intuition: a query scores keys, and the resulting weights mix values.

Seminar-room translation

A token saying "it" emits a query like "I need the thing I refer to." Nearby nouns emit keys like "I am an object/person/place." If "robot" is compatible, its value vector is copied strongly into the representation of "it".

✓ Check yourself — why are keys and values separate?

The key is used to decide whether to listen; the value is the content transferred if we do listen. This lets a token advertise one aspect for matching while sending a different learned representation as its message.

3.Scaled dot-product attention

Now the whole communication rule can be written in one line:

Core equation

$$\text{Attention}(Q,K,V)=\text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

Read it left to right:

  1. \(QK^\top\): every query dot-products every key. A large dot product means "my question matches your advertised information." Geometrically, aligned vectors have high similarity.
  2. divide by \(\sqrt{d_k}\): if keys/queries have dimension \(d_k\), random dot products have variance proportional to \(d_k\). Without scaling, scores get large, softmax saturates into near one-hot weights, and gradients become brittle. The scale keeps logits in a healthy range.
  3. softmax: converts raw compatibility scores into nonnegative weights that sum to 1 for each receiving token.
  4. multiply by \(V\): use those weights to take a weighted average of value messages.
Common mistake

The \(\sqrt{d_k}\) is not an arbitrary decoration. It is a variance-control trick. If the logits are too large, softmax becomes nearly one-hot too early, so most positions receive almost no gradient.

✓ Check yourself — what exactly is normalized by softmax?

For each receiving token, softmax normalizes its compatibility scores over the allowed sending tokens. Each row of the attention matrix becomes a distribution over "who I listen to."

4.Self-attention, cross-attention, and causal attention

The same equation creates different modules depending on where \(Q,K,V\) come from and which graph edges are allowed.

ModeWhere queries come fromWhere keys/values come fromWhat it is for
Self-attentionsame sequencesame sequencetokens in one input communicate with each other
Cross-attentiondecoder sequenceencoder outputdecoder retrieves information from a source sequence
Causal self-attentioncurrent prefixcurrent prefix, past onlyautoregressive next-token prediction without cheating

4.1 Causal masking: hiding the future

For GPT-style generation, token \(t\) must predict the next token using only positions \(\le t\). If it could attend to future tokens in the training batch, the answer would leak. So we add a triangular mask before softmax: disallowed scores become \(-\infty\), and softmax turns them into zero probability.

✓ Check yourself — why use \(-\infty\) for masked scores?

Because \(\exp(-\infty)=0\). After softmax, masked future positions receive exactly zero attention weight, so no value information can flow from them.

5.Multi-head attention: many relationship detectors in parallel

One attention head can learn one kind of communication pattern. But language has many relations at once: pronoun-to-noun, adjective-to-noun, subject-to-verb, delimiter-to-matching-delimiter, and so on. Multi-head attention runs several independent attention operations in parallel, concatenates their outputs, and projects back to the model dimension.

Multi-head form

$$\text{head}_i=\text{Attention}(QW_i^Q,KW_i^K,VW_i^V),\qquad \text{MHA}=\text{Concat}(\text{head}_1,\ldots,\text{head}_h)W^O$$

Copy-paste in parallel

Karpathy's phrasing: heads are copy-paste attention in parallel; layers are copy-paste attention in series. Heads let one layer collect different kinds of evidence before the residual stream continues.

6.Positional encodings: giving a set an order

Raw self-attention is permutation-invariant: if you shuffle the token vectors and shuffle them back afterward, the same content interactions happen. That is powerful for sets, but sentences are not sets. "dog bites man" and "man bites dog" contain the same words and different meanings.

So transformers add position information to token embeddings before the first block:

Token identity + location

$$x_t = e_{\text{token}(t)} + p_t$$

In the original paper, \(p_t\) used sinusoidal features such as $$PE_{(pos,2i)}=\sin\!\left(pos/10000^{2i/d_{model}}\right),\quad PE_{(pos,2i+1)}=\cos\!\left(pos/10000^{2i/d_{model}}\right).$$ Modern systems often use learned, relative, or rotary position encodings.

TypeIdeaTradeoff
Sinusoidalfixed sine/cosine features at different frequenciesno learned parameters; extrapolation intuition
Learned absolutea trainable vector for each positionsimple and common; tied to trained context length
Relative / rotaryencode distances or rotate queries/keys by positionoften better for long contexts and modern LLMs
✓ Check yourself — what breaks if we omit position entirely?

The model knows which tokens are present but not their order. It cannot reliably distinguish sequences that are permutations of each other, so syntax and temporal structure become ambiguous.

7.The full Transformer block: communicate, then compute

A modern decoder block is often pre-norm: apply LayerNorm before each sublayer, then add the sublayer output back to the residual stream. The two phases are:

  • Attention: tokens communicate across the graph.
  • Position-wise MLP: each token independently "thinks" on the information it has gathered.
Why residuals matter

Residual connections are gradient highways. A block does not have to rewrite the whole representation; it can add a useful update. During backpropagation, gradients can travel through the identity path instead of crossing dozens of nonlinear transformations.

7.1 Stacking blocks and encoder/decoder structure

✓ Check yourself — what is the MLP doing if attention already mixed tokens?

Attention moves information between tokens. The MLP transforms each token's gathered information locally, adding nonlinear computation at every position. Karpathy calls these the communication phase and the compute phase.

8.Why transformers scale so well

Karpathy emphasizes three properties that made transformers dominate:

  • Expressive forward pass. Attention creates data-dependent edges; prompts can configure behavior at runtime, enabling in-context learning.
  • Optimizable. Residual pathways and LayerNorm keep gradient flow and activation scales manageable.
  • Efficient on GPUs. Training is a shallow, wide computation graph full of matrix multiplications, not a long serial RNN chain.
The scaling tradeoff

Full attention costs \(O(n^2)\) scores for a sequence of length \(n\). That is the price of every token being able to talk to every other token in one layer. Long-context research is largely about reducing, structuring, caching, or externalizing this communication.

9.nanoGPT: the practical view

Karpathy's practical point is disarming: a serious GPT-style model is not magic code. The core architecture can fit in a few hundred readable lines. The complexity is mostly batching, tensor shapes, masks, and scaling the training run.

# one causal self-attention head, schematic PyTorch-ish pseudocode
q = x @ Wq                 # (B, T, d_head)
k = x @ Wk                 # (B, T, d_head)
v = x @ Wv                 # (B, T, d_head)

scores = q @ k.transpose(-2, -1) / sqrt(d_head)  # (B, T, T)
scores = scores.masked_fill(future_mask == 0, -inf)
weights = softmax(scores, dim=-1)
y = weights @ v             # weighted message aggregation
Generation as a loop

A decoder-only transformer predicts the next token, samples or chooses it, appends it to the prefix, and repeats. Once the prefix exceeds the model's context length, old tokens must be cropped, compressed, cached, or moved into some external memory scheme.

✓ Check yourself — why is GPT called decoder-only?

It has the Transformer decoder's causal self-attention stack but no separate source encoder and no cross-attention to an encoded input. It models the next token from the prefix alone.

10.Encoder-only, decoder-only, and encoder-decoder

Once you understand the graph mask and the source of keys/values, the famous model families are easy to compare.

FamilyAttention patternTraining signalBest atExamples
Encoder-onlyfull bidirectional self-attentionmasked/denoising or supervised objectivesclassification, retrieval, sentence understandingBERT, RoBERTa
Decoder-onlycausal self-attentionnext-token predictiongeneration, in-context learning, chat/code modelsGPT series, LLaMA-style models
Encoder-decoderencoder full attention; decoder causal + cross-attentionsequence-to-sequence likelihoodtranslation, summarization, conditional generationoriginal Transformer, T5, BART
One architecture, many domains

The same block can process text tokens, image patches, spectrogram slices, protein residues, or state-action-reward triples. What changes is tokenization, positional/structural encoding, and the mask. That flexibility is why Karpathy calls transformers a general-purpose, efficient, optimizable computer.

12.Worked numeric micro-example: one attention row by hand

Use a toy single self-attention head with three tokens and 2-D query/key/value vectors. Pretend the learned projections have already produced these small integers:

Setup

$$ Q=\begin{bmatrix}1&0\\0&1\\1&1\end{bmatrix},\quad K=\begin{bmatrix}1&1\\1&0\\0&1\end{bmatrix},\quad V=\begin{bmatrix}1&2\\0&1\\3&0\end{bmatrix} $$

First compute every query-key dot product:

$$ QK^\top= \begin{bmatrix} 1&1&0\\ 1&0&1\\ 2&1&1 \end{bmatrix} $$

For token 1, the first row is especially easy to see:

$$ q_1K^\top=[1,0]K^\top=[1,1,0] $$

Because \(d_k=2\), scale the row by \(1/\sqrt{d_k}=1/\sqrt2\):

$$ \frac{[1,1,0]}{\sqrt2}\approx[0.707,0.707,0] $$

Now softmax that row. The exponentials are approximately \([2.028,2.028,1]\), so the normalized attention weights are:

$$ \text{softmax}([0.707,0.707,0])\approx[0.401,0.401,0.198] $$

Finally, token 1 receives the weighted sum of the value rows:

$$ o_1=0.401[1,2]+0.401[0,1]+0.198[3,0]\approx[0.995,1.203] $$

✓ Check yourself — why divide by \(\sqrt{d_k}\)?

Dot products grow in typical magnitude as the key/query dimension grows. Without the \(1/\sqrt{d_k}\) scaling, logits become large, softmax saturates toward one-hot choices, and gradients through the non-winning positions shrink.

13.End-to-end trace through one Transformer block

Concrete setting: one sequence with \(T=5\) tokens, model width \(d=512\), \(h=8\) heads, and per-head width \(d_h=64\). Shapes below omit the batch dimension.

StepWhat happensShape
1Token embeddings are added to positional vectors: tokens + pos.\(T\times d=5\times512\)
2Linear projections produce Q,K,V for attention.\(5\times512\) each
3Within each head, queries compare with keys: scores = QKᵀ.\(T\times T=5\times5\)
4Scale, mask if needed, then apply row-wise softmax.\(5\times5\)
5One head uses those weights to mix value rows.\(T\times d_h=5\times64\)
6Concatenate all \(h=8\) head outputs and project back to the residual width.\(T\times d=5\times512\)
7Add the attention update to the incoming residual stream.\(5\times512\)
8Apply LayerNorm before the next sublayer in the pre-norm view.\(5\times512\)
9Run the position-wise MLP; it may expand internally, but returns to model width.\(T\times d=5\times512\)
10Add the MLP update back to the residual stream.\(T\times d=5\times512\)
Communication vs computation

Attention is communication: each token chooses which other tokens to read from and blends their value messages. The MLP is computation: after communication, each token independently transforms its own gathered information. A Transformer block alternates these two moves: talk across positions, then think locally.

Glossary

TermPlain meaning
TokenThe unit processed by the transformer: word-piece, character, image patch, audio slice, protein residue, or RL event.
Residual streamThe main vector pathway through the network; sublayers add updates to it.
Query \(q\)A learned vector describing what a token is looking for.
Key \(k\)A learned vector describing what a token offers for matching.
Value \(v\)The learned message content transferred through attention weights.
Attention matrixThe row-normalized weights saying how much each token listens to each allowed token.
Causal maskA triangular mask that blocks future positions during autoregressive training.
HeadOne independent attention operation; multi-head attention runs several in parallel.
LayerNormA normalization layer that stabilizes activation scales within each token vector.
Position-wise MLPThe feed-forward network applied independently to every token after communication.
Context lengthThe maximum number of tokens a model can condition on in one forward pass.
