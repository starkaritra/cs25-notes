Representing Part-Whole Hierarchies in a Neural Network (GLOM)

Hinton asks how a neural net with fixed weights could represent a different parse tree for every image; GLOM's answer is to make dynamic embedding vectors agree into islands that stand for parts, objects, and scenes.

🎓 Geoffrey Hinton ⏱ ~50 min ▶ Watch
Why this matters

Every image has its own part-whole hierarchy: a scene contains objects; objects contain parts; parts contain sub-parts. Symbolic AI can allocate memory for a fresh parse tree and draw pointers between nodes. A biological-looking neural net cannot rapidly rewrite its weights or grab arbitrary RAM. So where does the tree live?

GLOM is Hinton's proposed answer: keep the hardware fixed, but let the activations become the dynamic pointers. When several locations at the same abstraction level settle on the identical vector, that agreement is treated as one node in the parse tree.

Running analogy — a city map drawn by local surveyors

Imagine a city where every block has a surveyor with notebooks stacked by scale: curb, room, building, neighborhood, whole city. The surveyors cannot move offices or rewire the city. Instead, nearby surveyors compare notes. When many notebooks at the "building" scale contain the same address, they have formed an island: one building on the map.

Prerequisites — 30-second refresher

Skim these before the proposal; each is a tool Hinton combines rather than a topic he fully re-teaches.

  • Attention / weighted average — score neighbors, softmax the scores into weights, then average their vectors; here this is the transformer-like mechanism that makes nearby columns agree.
  • Autoencoder / reconstruction — hide or corrupt part of the input and train the system to reconstruct it; GLOM can be trained BERT-style by filling in missing patches after settling.
  • Contrastive learning — make two views of the same thing agree while preventing trivial collapse by contrasting with different things; here it encourages the right locations to share a vector.
  • Capsule networks — Hinton's earlier attempt to represent entities and poses with pre-allocated groups of neurons plus routing; GLOM keeps the part-whole ambition but makes every column more universal.
  • Parse tree / part-whole hierarchy — a tree whose nodes are wholes at one level and parts at the next; GLOM tries to represent this tree without symbolic pointers.

1.Dynamic trees, fixed hardware

Start with Hinton's concrete worry. In one image, three patches might make a nose; in another, the same retinal locations might make a wheel, text, or background. The parse tree changes image by image, but the neural machinery is the same hardware every time.

1.1 Why pointers are hard for neurons

In a symbolic program, making a new parse tree is easy: allocate a node for nose, allocate child nodes for nostril and bridge, then store pointers. In a neural network, what a neuron computes is mostly fixed by its incoming weights. Those weights learn slowly; they are not fast scratch memory.

Common misconception

"A deep CNN already has a hierarchy, so the parse tree problem is solved." A CNN has a fixed hierarchy of feature maps. Hinton's question is different: how can one fixed machine represent a different compositional tree for each image, including which pixels belong to the same object right now?

✓ Check yourself — what has to change from image to image?

Not the learned weights. The changing thing is the pattern of activations: which locations agree, which level they agree at, and therefore which temporary parse-tree nodes exist for this image.

1.2 From capsules to universal columns

Hinton frames GLOM as a successor to capsules. Capsules pre-allocated groups of neurons for possible entities and then routed active capsules into a tree. GLOM keeps the idea that hardware is locally grouped, but it removes the strong pre-allocation: a column's embedding can represent a nostril, a wheel, a word fragment, or whatever the local evidence supports.

The shift

Capsules asked, "Which specialized capsule should own this part?" GLOM asks, "Which neighboring columns can agree on the same higher-level vector?" Ownership becomes agreement, not a reserved slot.

2.GLOM's proposed representation

Picture a one-dimensional strip of image patches. At each patch location, GLOM stores a vertical stack of embeddings: low levels for local parts, higher levels for larger wholes, and the top for the scene. Hinton calls these location-based stacks columns.

2.1 Columns, levels, and islands of agreement

In the city analogy, every block's surveyor has notebooks at multiple scales. At the curb scale, neighboring blocks can disagree: brick, glass, door, tree. At the building scale, all blocks belonging to the same building should write the same address. At the city scale, almost every block can share the same city label.

Key equation — an island is equality of activations

$$I_{\ell,k}=\{x : z_{x,\ell} \approx c_{\ell,k}\}$$

Here \(x\) indexes a location/column, \(\ell\) indexes an abstraction level, \(z_{x,\ell}\in\mathbb{R}^d\) is that column's embedding at that level, and \(c_{\ell,k}\) is the shared vector for island \(k\). The island acts like a parse-tree node: "these locations are one thing at level \(\ell\)."

Hinton emphasizes a subtle point: the shared vector is not a learned weight. It is an activation, so it can be new for every image. That makes it eligible to play the role of a dynamic pointer.

✓ Check yourself — what does one object-level island mean?

It means several columns, perhaps seeing superficially different pixels, have settled on the same object-level embedding. In the parse tree, that island behaves like one object node whose children are lower-level islands in those locations.

2.2 Coordinate frames matter

A long middle part of the talk uses mental imagery: the rotated cube, the unfamiliar "hexahedron" view, and the sentence "next weekend we should be visiting relatives." The point is not cube trivia. It is that perception uses structural descriptions plus coordinate frames. The same physical object can have very different internal descriptions when the chosen frame changes.

Key equation — relative and viewer coordinates

$$R_{xv}=R_{wv}R_{xw}$$

Here \(R_{xw}\) is a learned, viewpoint-independent relationship between a whole \(w\) and part \(x\); \(R_{wv}\) is the whole's current pose relative to the viewer; \(R_{xv}\) is the part's pose relative to the viewer. Hinton's claim: relations like \(R_{xw}\) are the kind of stable knowledge that belongs in weights.

Why the cube demo belongs here

If you cannot describe parts relative to wholes, a nose and a mouth are just nearby patterns. With coordinate transforms, a nose can predict a face, and a mouth can predict the same face. Agreement at the higher level becomes evidence that the parts belong together.

✓ Check yourself — what remains stable when viewpoint changes?

The relationship between part and whole, such as "mouth below nose in the face's intrinsic frame," remains stable. The viewer-relative coordinates of both part and whole change together.

3.The update rule: four voices in one column

Now follow one notebook in one surveyor's stack: column \(x\), level \(\ell\), time step \(t\). Hinton says the next embedding is a consensus built from four contributions.

Key equation — GLOM-style consensus update

$$ z_{x,\ell}^{t+1}=\operatorname{Norm}\!\left( W_p z_{x,\ell}^{t} +B_{\ell}\!\left(z_{x,\ell-1}^{t},x\right) +T_{\ell}\!\left(z_{x,\ell+1}^{t},x\right) +\sum_{y\in\mathcal{N}(x)}\alpha_{xy}^{\ell,t}z_{y,\ell}^{t} \right) $$

\(W_p z_{x,\ell}^{t}\) is persistence from the previous state; \(B_{\ell}\) is a bottom-up neural net from the level below; \(T_{\ell}\) is a top-down neural net from the level above; and the final term is a same-level attention-weighted average over nearby columns \(y\in\mathcal{N}(x)\). \(\operatorname{Norm}\) stands for whatever normalization/nonlinearity keeps the vector well-scaled.

3.1 Same-level attention creates islands

The transformer-like part is deliberately simple. If two nearby embeddings at the same level already point in a similar direction, they put more weight on each other. That makes them more similar next step, which makes them attend more strongly, and so on. Agreement becomes self-reinforcing.

Key equation — local attention weight

$$ \alpha_{xy}^{\ell,t}=\frac{\exp\left((z_{x,\ell}^{t})^\top z_{y,\ell}^{t}/\tau\right)} {\sum_{y'\in\mathcal{N}(x)}\exp\left((z_{x,\ell}^{t})^\top z_{y',\ell}^{t}/\tau\right)} $$

\(\alpha_{xy}^{\ell,t}\) is how much column \(x\) listens to neighbor \(y\) at level \(\ell\). \(\tau\) is a temperature: smaller \(\tau\) makes attention more selective. The weighted average is what pulls an island toward one common vector.

Common misconception

"GLOM discovers fixed clusters in finished embeddings." Hinton says the flavor is different from ordinary clustering: the data being clustered are themselves changing over time because bottom-up, top-down, persistence, and attention all keep rewriting the embeddings.

✓ Check yourself — why does attention need similarity gating?

Without gating, all nearby locations would be pushed toward one average, including different objects. Similarity-gated attention lets likely same-object locations agree while keeping incompatible hypotheses apart.

4.Worked micro-example — five locations, two levels

Use a tiny one-dimensional "image" with five locations. At level 1 the locations see local parts. At level 2 they propose object-level embeddings. Real GLOM vectors are high-dimensional; to do arithmetic by hand, we track one scalar coordinate of the level-2 vector.

\(z_i^0\)0.801.001.20-1.00-1.20

Attention says locations 1--3 are mutually similar and locations 4--5 are mutually similar, so use these local attention weights:

  • For locations 1, 2, 3: average the three values with weights \((1/3,1/3,1/3)\).
  • For locations 4, 5: average the two values with weights \((1/2,1/2)\).
  • Mix old state and attention average: \(z_i^{t+1}=0.5z_i^t+0.5\bar z_i^t\).
Step-by-step with toy numbers

For the first proposed object, the attention average is

$$\bar z_{1:3}^{0}=\frac{0.80+1.00+1.20}{3}=1.00.$$

After one update:

$$z_{1:3}^{1}=0.5[0.80,1.00,1.20]+0.5[1.00,1.00,1.00]=[0.90,1.00,1.10].$$

The average is still \(1.00\). After a second update:

$$z_{1:3}^{2}=0.5[0.90,1.00,1.10]+0.5[1.00,1.00,1.00]=[0.95,1.00,1.05].$$

They are converging to the same vector coordinate, \(1.00\). Meanwhile locations 4--5 average to \(-1.10\):

$$z_{4:5}^{1}=0.5[-1.00,-1.20]+0.5[-1.10,-1.10]=[-1.05,-1.15].$$

So the five locations are not all washed into one cluster. They form two islands: \(\{1,2,3\}\) and \(\{4,5\}\).

✓ Check yourself — what would a second island at a lower level mean?

It would mean a smaller part-whole node, not a competing object. For example, at level 1 locations 1--2 might agree on "nostril" while locations 1--3 agree at level 2 on "nose." Islands at different levels can be nested, which is how the parse tree appears.

5.End-to-end trace — follow one column

Now follow one concrete column: location \(x=2\), a patch that begins as a dark curved mark and may become part of a nose, then a face, then a party scene.

StepWhat happensState / shape / value
0 · pixel inputA front-end CNN or patch encoder reads the local retinal patch.patch x=2: dark curve + skin-like context
1 · level-1 part vectorThe lowest GLOM level proposes a local part embedding.\(z_{2,1}^{0}\in\mathbb{R}^d\): "maybe nostril / edge"
2 · bottom-up contributionA learned net asks what whole this part predicts, including pose.\(B_2(z_{2,1}^{0},x=2)\): "nose at pose p"
3 · top-down contributionThe level above predicts what should be present at this location if the current whole hypothesis is right.\(T_2(z_{2,3}^{0},x=2)\): "face expects nose-region evidence here"

What the trace reveals

A column never stops being about its patch. What changes is the scale of the hypothesis written in that patch's stack: local mark → part → object → scene. The parse tree is the pattern of shared hypotheses across locations and levels.

✓ Check yourself — which contribution actually communicates with other locations?

The same-level attention term. Bottom-up, top-down, and persistence are within a column; attention is what lets nearby columns form islands.

6.Training and unification

Hinton presents GLOM as a meeting point for several neural-network ideas. The proposal is not "replace transformers"; it is "use transformer-like attention as one ingredient in a part-whole architecture."

6.1 Reconstruct, agree, and avoid collapse

One training route is BERT-like reconstruction: hide patches, let the recurrent column-level system settle for several iterations, then backpropagate through time from the reconstruction error. But reconstruction alone does not explicitly demand islands, so Hinton adds a contrastive/agreement pressure.

Key equation — reconstruction plus agreement pressure

$$\mathcal{L}=\mathcal{L}_{\text{reconstruct}}+\lambda\mathcal{L}_{\text{agree}}+\mu\mathcal{L}_{\text{contrast}}$$

\(\mathcal{L}_{\text{reconstruct}}\) trains the system to fill missing low-level patches; \(\mathcal{L}_{\text{agree}}\) makes bottom-up and top-down predictions match the consensus embedding; \(\mathcal{L}_{\text{contrast}}\) prevents the degenerate solution where every location and every image has the same vector.

Back to the surveyors

Reconstruction asks each surveyor to redraw missing street details. Agreement asks nearby surveyors to settle on the same building address when their evidence supports it. Contrast prevents the useless city map where every block is labeled "same."

6.3 Ambiguity, Hough transforms, and why parts predict wholes

A circle might be a left eye, a right eye, a front wheel, or a back wheel. Directly routing every possible part-to-part relation would require many transformed messages. Hinton's shortcut is Hough-transform-like: each ambiguous part predicts possible wholes. A nose patch and a mouth patch need not message each other directly; both can predict a face, and their face-level vectors can agree.

The structural trick

It is often easier to check whether two parts predict the same whole than to hand-route every part-to-part relation. Agreement at the whole level resolves ambiguity that was hard at the part level.

6.4 Epistemic status: idea, not established system

Feynman-style honesty

Hinton is explicit: GLOM is a design document / paper of ideas, not a mature demonstrated system. At the time of the talk, he describes it as "vaporware" with only small pieces implemented. Treat the architecture as a research hypothesis that organizes useful ideas, not as an empirical result that has already beaten standard vision transformers.

That honesty makes the lecture more useful, not less. The value is the question it sharpens: can neural activations, through local agreement across levels, play the role that symbolic AI gave to dynamically allocated parse-tree nodes?

✓ Check yourself — why is replication across many patches not necessarily wasteful?

Before the system settles, it does not know which patches belong together. Keeping separate higher-level vectors at every location lets hypotheses merge gradually into islands. Hinton compares biological replication to DNA: many cells can carry the same information because local access is useful.

Glossary

TermPlain meaning
GLOMHinton's proposed architecture where each image location has a stack of embeddings, and part-whole structure is represented by islands of agreement.
ColumnAll embeddings at one spatial location, across multiple abstraction levels.
EmbeddingA dynamic vector activation that represents a hypothesis about what is present at a location and level.
Island of agreementA set of locations whose embeddings at the same level become identical or nearly identical; it functions like one parse-tree node.
Parse treeA part-whole structure: scene → objects → parts → sub-parts.
Bottom-up netA learned mapping that predicts a higher-level whole from a lower-level part.
Top-down netA learned mapping that predicts a lower-level part from a higher-level whole, using location to know which part should appear where.
Contrastive learningA self-supervised objective that pulls related views together while pushing unrelated or confusing views apart enough to prevent collapse.
Neural fieldA function from coordinates to values; in GLOM, location helps the same high-level vector predict different lower-level contents at different places.
CapsuleA group of neurons intended to represent an entity and its pose; GLOM inherits the part-whole goal but makes columns more universal.
