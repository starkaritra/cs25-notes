Overview of Transformers (CS25 V4, 2024)

The V4 opener resets the map: transformers are no longer only an NLP architecture, but the shared substrate for language models, multimodal systems, and tool-using agents in 2024.

🎓 Div, Steven, Emily, and Seonghee (CS25 V4 instructors) ⏱ ~1 hr ▶ Watch
Why this matters

Earlier CS25 overviews could ask whether attention would generalize. By 2024 the answer is everywhere: GPT-style assistants, Gemini, Sora-like video, Med-PaLM, open-weight models, and agents all reuse transformer ideas or orbit them. The useful question is now: what does this architecture make easy, what does it make expensive, and where is the frontier moving?

Running analogy

Think of a transformer as a busy research lab. Each token is a researcher. Attention is who each researcher asks for context; multi-head attention is several committees asking different questions; the MLP is private thinking; the residual stream is the shared notebook. In 2024 that lab has become a company with tools, memory, sensors, and agents around the same meeting protocol.

Prerequisites — 30-second refresher

This note keeps mechanics tight. For the historical arc, see Note 1; for the deep mechanics pass, see Note 2.

  • Embedding / token — text is split into tokens, then each token becomes a vector; transformers operate on vectors.
  • Dot product — measures vector alignment; here it scores query-key similarity.
  • Softmax — turns scores into nonnegative weights that sum to 1, so attention becomes a weighted average.
  • Matrix multiply shapes — a sequence matrix \(T\times d\) can be projected into compatible \(Q,K,V\) matrices.

1.Why an overview now?

The lecture opens with the course mission: bring the latest transformer and LLM research directly to students. That matters because transformers have crossed from “paper to understand” into “infrastructure to design around.”

1.1 The field consolidated

The transcript begins before transformers: rule-based NLP, parsing, RNNs, LSTMs, and early attention. The 2017 Attention Is All You Need paper made attention a full architecture. BERT and GPT pushed it through NLP; then the same recipe spread to vision, protein folding, audio, code, video, robotics, and agent systems.

The claim is not that every modern model is vanilla transformer. It is that the token + attention + scale interface became standard: tokenize a modality, let tokens communicate by attention, train at scale, then adapt using prompts, fine-tuning, retrieval, tools, or preferences.

1.2 What this lecture updates

Since Note 2 covers attention mechanics in depth, V4 spends its depth on the current map: decoder-only LLMs, scaling and emergence, RLHF/DPO, multimodality, long context, efficiency, open versus closed ecosystems, and agents with memory, tools, planning, communication, and reliability problems.

Common misconception

“Transformers won because attention is biologically like human attention.” The engineering win is more concrete: attention gives content-based routing among tokens and trains in parallel far better than recurrent sequence models.

✓ Check yourself — why did recurrence limit older sequence models?

RNNs process tokens step by step and compress history into a hidden state. That hurts long-range dependencies, limits parallel training, and can create vanishing/exploding-gradient problems.

2.Mechanics recap: tokens ask each other for help

For a phrase like “the trophy did not fit in the suitcase because it was too large,” the token “it” needs relevant context from earlier tokens. A transformer learns how each token should query the others.

2.1 Self-attention as content-based routing

Each token creates three learned views. A query says “what am I looking for?”, a key says “what information do I offer?”, and a value is the information copied if the match is useful. The softmax makes routing soft rather than one-book retrieval.

Key equation — scaled dot-product attention

$$\text{Attention}(Q,K,V)=\text{softmax}\!\left(\tfrac{QK^\top}{\sqrt{d_k}}\right)V$$

\(Q\) are queries, \(K\) keys, \(V\) values, and \(d_k\) the key/query width. \(QK^\top\) gives all match scores; \(\sqrt{d_k}\) stabilizes scale; multiplying by \(V\) mixes retrieved content.

✓ Check yourself — in the lab analogy, what is the query and what is the key?

The query is a researcher's question: “who has information I need?” The key is each colleague's label: “what information do I contain?” The score measures how well question and label match.

2.2 Blocks and families

A transformer block alternates communication and computation. Attention lets tokens exchange information. The MLP processes each token locally. Residual connections preserve the shared notebook; normalization keeps signals stable. Multi-head attention repeats routing in parallel so different heads can track syntax, names, positions, or long-range references.

FamilyWhat can attend to what?Typical use
Encoder-onlyBidirectional attention over input tokens.BERT-style representation, classification, retrieval encoders.
Decoder-onlyCausal attention: previous tokens only.GPT-style next-token prediction, chat models, agents.
Encoder-decoderEncoder reads source; decoder self-attends and cross-attends.Translation, summarization, T5/BART-style seq2seq.
✓ Check yourself — what does the MLP add if attention already mixed tokens?

Attention moves information between tokens. The MLP transforms each token's updated vector locally, adding nonlinear computation after communication.

3.The 2024 landscape: what changed around the block

The block is recognizable; the ecosystem around it changed. V4 emphasizes models that are larger, more instruction-following, more multimodal, more tool-connected, and split between closed frontier labs and open-weight communities.

3.1 Decoder-only dominance, scaling, and emergence

The lecture's LLM story centers on autoregressive next-token prediction: train a decoder-only transformer on massive text, then adapt it into an assistant. This objective scales cleanly, uses one interface for prompts and outputs, and can be adapted without task-specific heads.

Key equation — autoregressive language modeling

$$p_\theta(x_1,\ldots,x_T)=\prod_{t=1}^{T}p_\theta(x_t\mid x_{<t})$$

The model predicts token \(x_t\) from earlier tokens \(x_{<t}\). In chat, instructions, retrieved documents, tool outputs, and conversation history are all placed into this same token stream.

Scaling laws make part of progress predictable: more compute, parameters, and data usually reduce loss. The lecture also highlights emergent abilities—task-level jumps such as arithmetic, transliteration, or reasoning that appear in larger models. Emily gives the honest 2024 caveat: some cliffs may reflect nonlinear metrics, but small-model curves still cannot safely bound frontier capabilities.

✓ Check yourself — how can emergence and smooth scaling both be true?

Token-level loss can improve smoothly while an all-or-nothing benchmark stays near zero until a threshold is crossed. The underlying barometer is smooth; the public score can look sudden.

3.2 Tuning, modalities, long context, and efficiency

  • Instruction tuning + RLHF/DPO. Supervised examples teach the assistant format; preference optimization pushes outputs toward what humans prefer. The lecture contrasts RLHF's complex reward/policy loop with DPO's direct chosen-versus-rejected training.
  • Multimodality. Transformers moved beyond text into images, audio, biology, robotics, and video. The common move is tokenization: represent patches, frames, actions, or latent video chunks as sequences.
  • Long context + memory. Gemini-style long contexts are useful, but agents also need persistent memory: store embeddings externally, retrieve relevant memories, and place them into the prompt.
  • Efficiency. FlashAttention makes exact attention faster by improving GPU memory use; KV-cache avoids recomputing old keys/values during generation; Mixture-of-Experts activates only a few expert MLPs per token.
  • Open vs closed. Closed systems push frontier capability and product polish; open models such as LLaMA, Mistral, and Mixtral enable local deployment, auditing, fine-tuning, and broader research.
  • Agents. The lecture's 2024 emphasis is models connected to tools, browsers, memory, planning, reflection, communication, and human-like computer interfaces.
Unifying frame

Many 2024 improvements decide what enters the lab and what tools the lab can use: instructions, retrieved memories, images, code outputs, browser observations, user preferences, and other agents' messages.

4.Open problems: where the lab still breaks

The overview is optimistic, but not a victory lap. The more transformer systems act in the world, the more visible their failure modes become.

4.1 Agents expose the hard edge

Div frames an LLM as a compute chip for natural-language tokens: context length is RAM, retrieval is a disk, tools are peripherals, and agents are the operating system around the chip. That analogy is powerful because it also exposes what is missing: deterministic reliability, testing, permissions, security, and recovery when a plan drifts.

Open problemWhy it matters in 2024Typical mitigation
Quadratic attentionFull attention over \(T\) tokens costs roughly \(O(T^2)\) score interactions.FlashAttention, sparse/windowed attention, retrieval, compression.
Context lengthA long prompt is not stable, updateable memory across conversations.RAG, vector databases, memory summarization, personalization.
ReasoningChain-of-thought helps, but models can skip steps or misunderstand semantics.Verifiers, tool use, self-consistency, decomposition.
HallucinationNext-token likelihood is not truth; false text can sound confident.Retrieval, calibration, citations, verification.
AlignmentSystems that act for users must respect intent, privacy, safety, and values.RLHF/DPO, permissions, audits, human fallback.
EfficiencyFrontier training costs concentrate access in large companies.Open models, MoE, distillation, synthetic/textbook data.
Common misconception

“More autonomy automatically means more usefulness.” The agent section says the opposite: autonomy without error correction, observability, sandboxing, and user permissions can amplify small stochastic mistakes into real-world failures.

✓ Check yourself — why is an agent harder to trust than a one-shot chatbot?

A chatbot returns text you inspect. An agent takes multi-step actions, observes consequences, calls tools, and can drift after an early mistake. Reliability, rollback, permissions, and monitoring become part of the model problem.

5.Worked example — scaled dot-product attention with two tokens

Use the smallest honest attention calculation: two tokens, one head, \(d_k=2\). Compute the output for token 1.

Step-by-step with real numbers

Let token 1 have query \(q_1=[1,1]\). The keys and values are:

$$k_1=[1,0],\quad k_2=[0,1],\qquad v_1=[10,0],\quad v_2=[0,4].$$

Raw dot products:

$$q_1\cdot k_1 = 1\cdot1+1\cdot0=1,\qquad q_1\cdot k_2 = 1\cdot0+1\cdot1=1.$$

Scale by \(\sqrt{d_k}=\sqrt{2}\):

$$s=\left[\tfrac{1}{\sqrt{2}},\tfrac{1}{\sqrt{2}}\right]\approx[0.707,0.707].$$

Softmax of equal scores gives equal weights, \(\alpha=[0.5,0.5]\). Mix the values:

$$o_1=0.5\,[10,0]+0.5\,[0,4]=[5,2].$$

Token 1 returns \([5,2]\): half of token 1's value and half of token 2's value.

✓ Check yourself — what is the role of \(\sqrt{d_k}\)?

Without scaling, dot products grow as vector width grows. Large scores make softmax too peaky and gradients less useful. Dividing by \(\sqrt{d_k}\) keeps score variance roughly controlled.

✓ Check yourself — redo the last step if the weights were [0.8, 0.2]

The output is \(0.8[10,0]+0.2[0,4]=[8,0.8]\). More attention to token 1 means more of token 1's value appears.

6.End-to-end trace — one decoder-only block

Trace a mini-GPT block with \(T=4\) tokens, model width \(d=8\), two heads, and per-head width \(d_k=4\). Example tokens: The cat sat <next>.

StepWhat happensShape
0 · tokens+posToken embeddings are added to positional encodings.T×d = 4×8
1 · QKVLearned projections produce queries, keys, and values; split into 2 heads.Q,K,V: 4×8 → 2×4×4
2 · causal scoresEach head computes \(QK^\top/\sqrt{d_k}\); future positions are masked.per head: 4×4
3 · softmaxRows become distributions over allowed earlier tokens.per head: 4×4 weights
4 · attn outWeights multiply values; heads concatenate and project back.2×4×4 → 4×8
5 · +residualAttention output is added to the residual stream.4×8
6 · normLayer norm stabilizes vector scale.4×8
7 · MLPEach token independently expands and contracts through a feed-forward network.4×8 → 4×32 → 4×8
8 · +residualThe block hands an updated residual stream to the next block.4×8
Communication vs computation

Attention is communication: token desks exchange notes under a causal rule. MLP is computation: each desk privately processes its updated note. Stacking blocks alternates these two moves until the final token has enough context to predict the next token.

Glossary

TermPlain meaning
TransformerA neural sequence architecture built from attention, MLPs, residual connections, and normalization.
Self-attentionTokens choose weighted information from other tokens in the same sequence.
Query / key / valueLearned vector views used to ask for information, advertise matchability, and provide retrieved content.
Multi-head attentionSeveral attention operations run in parallel in different learned subspaces.
Positional encodingInformation added to token vectors so order is visible.
Decoder-only modelA causal transformer trained to predict the next token; dominant for chat LLMs.
RLHFReinforcement learning from human feedback: training toward outputs humans prefer.
DPODirect Preference Optimization: preference training from chosen/rejected pairs.
KV-cacheStored keys and values reused during generation.
Mixture-of-ExpertsSparse model that routes each token to a few expert networks.
AgentA model-based system that plans, uses tools, observes results, and acts over multiple steps.
