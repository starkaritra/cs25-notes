STANFORD CS25 · TRANSFORMERS UNITED · V6

From Next-Token Prediction to Next-Generation Intelligence

Shrimai's central claim is practical: if human text is becoming the scarce resource, the next leap in LLMs comes from smarter pre-training algorithms—curriculum, early reasoning data, and reinforcement-style thinking—not merely more random next-token prediction.

🎓 Shrimai · Mistral AI / Boston University ⏱ ~50 min ▶ Watch
Why this matters

The talk starts from a looming bottleneck: frontier LLMs have moved from hundreds of billions of training tokens to tens of trillions, and Epoch AI projections suggest most high-quality human-generated text may soon be consumed by training runs. If the corpus is finite, the question changes from “How do we scrape more?” to “How do we learn more from the same tokens?”

Running analogy

Keep the speaker's classroom analogy in mind. Pascal, Volta, Ampere, and Hopper are four students with the same library. Pascal reads randomly. Volta follows a curriculum. Ampere also takes reasoning “AP classes” early. Hopper does all that and learns by actively thinking through problems. The lecture is a study of why Hopper wins.

Common misconception

“Just keep scaling next-token prediction and AGI falls out.” Shrimai's talk does not deny that next-token prediction built today's LLMs. It says the objective is a powerful starting point, not necessarily the ceiling: it mostly rewards matching observed text, while robust reasoning benefits from curriculum, explicit reasoning traces, and test-time or training-time exploration.

Prerequisites — 30-second refresher

Skim these before the training-recipe diagrams; each term appears in the lecture.

  • Next-token prediction — an autoregressive LM learns \(p(x_t \mid x_{<t})\); this is the baseline objective the talk tries to improve, not discard.
  • Pre-training → SFT → RL — base models first learn broad text statistics, then imitate instruction/reasoning formats, then receive reward-based post-training; the talk asks what should move earlier. See Note 37.
  • Test-time compute — spending inference compute on multiple attempts, search, or verification can make a weak single sample a stronger solver; related course context appears in Note 34 and Note 42.
  • Verifier / reward — a signal that scores an attempt. RLVR uses external verifiers; RLP instead builds an intrinsic reward from information gain.
  • Agent / tool use — systems that plan, call tools, remember state, and revise actions; useful next-gen capabilities, but this lecture's core is pre-training algorithms, not agent scaffolding. See Note 36.

1.The lecture's map: same data, different learning strategy

The concrete story is four learners with equal access to data. The lecture then compares three algorithmic additions: a two-phase data curriculum, front-loaded reasoning data, and RLP—reinforcement as a pre-training objective.

1.1 Pascal → Volta → Ampere → Hopper

The names are not decorative; they are the lecture's experimental ladder. Pascal is natural/random next-token pre-training. Volta uses better data mixture and ordering. Ampere sees reasoning traces during pre-training. Hopper additionally learns through generated thoughts with reward.

Key equation — the baseline objective

$$\mathcal{L}_{\text{NTP}}(\theta)=-\sum_{t=1}^{T}\log p_{\theta}(x_t\mid x_{<t})$$

Here \(x_t\) is the observed next token and \(x_{<t}\) is its prefix. The loss is excellent for learning language and world knowledge, but by itself it does not ask the model to search, backtrack, or explain why a token is correct.

✓ Check yourself — what is held constant in the Pascal/Volta/Ampere/Hopper comparison?

The raw access to data is conceptually held constant. The comparison is about learning strategy: order the data, include reasoning early, and reward useful thoughts.

2.Curriculum: use data better before asking for more data

Shrimai first discusses “maximizing your data's potential.” Modern corpora mix web crawl, books, code, math, papers, legal text, and many other sources. If a huge low-quality source dominates because it has more tokens, the model spends too much capacity on noise.

2.1 Quality, epochs, and order

The proposed pre-training recipe estimates three things: source quality, how many useful repeats a domain can tolerate before diminishing returns, and when the model should see each kind of data. The lecture's preferred curriculum is diversity first, quality later: phase 1 exposes the model broadly; phase 2 concentrates on high-quality sources such as math, Wikipedia, and code.

Key equation — mixture as weighted sampling

$$P(d)=\sum_{i=1}^{m} w_i\,P_i(d), \qquad \sum_i w_i=1$$

Dataset \(i\) contributes examples according to weight \(w_i\). Natural distribution sets \(w_i\) mostly by how many tokens exist; the lecture argues for weights informed by quality and useful repeats.

Result to remember

In the talk's reported comparison, Volta's two-phase approach was about 17% better than Pascal's natural/random approach and about 3.4% better than using an optimal mixture without curriculum ordering. Epistemic status: these are experimental results from the presented NVIDIA work, not a universal law.

✓ Check yourself — why not show only the highest-quality data from the start?

The lecture's answer is exploration before exploitation. Early broad diversity helps the model acquire coverage; late high-quality emphasis sharpens the learned distribution. Shrimai noted that swapping the order performed worse in their ablation.

3.Front-load reasoning: do not bolt thinking on at the end

The second major section challenges the standard pipeline: pre-training learns facts; SFT teaches reasoning-like formats; RL teaches reward-seeking problem solving. Shrimai argues this can create an “unreasoning foundation,” because reasoning is treated as a post-hoc patch.

3.1 Durable gains from early reasoning data

The study injects reasoning-style datasets at different phases and compares a no-reason base with a reason base. Reasoning data here means question → long reasoning trace → final solution, mostly STEM/math/code-style in the discussed experiments. It is not claimed to cover every domain of human reasoning.

Key equation — allocating reasoning data

$$D_{\text{reason}}=D_{\text{pretrain}}^{r}\cup D_{\text{SFT}}^{r}, \qquad |D_{\text{reason}}|\;\text{fixed}$$

The important comparison is not “more data beats less data.” The lecture tests where a fixed reasoning budget should go. Splitting some reasoning data into pre-training beat saving it all for SFT.

Reported lessons
  • Adding reasoning data during pre-training gave about 16% average gain immediately after pre-training.
  • After SFT, the gain persisted at about 9.3%; after SFT + RLVR, Ampere remained about 19% ahead of Volta, with especially large gains on hard math such as AIME.
  • High-quality reasoning data sometimes unlocked hidden gains only after post-training; the lecture uses this to argue against judging pre-training data solely by immediate base-model scores.
Common misconception

“If reasoning is post-training anyway, more SFT should catch up.” In the reported experiments, a no-reason base did not catch up even with 2× SFT epochs, and under a fixed reasoning-data budget, using some data in pre-training beat putting all of it into SFT.

✓ Check yourself — what does “durable advantage” mean here?

It means the benefit of early reasoning was not washed away by later SFT/RL. Instead, the later stages refined a stronger foundation.

4.Worked example — next-token sample versus reasoning + verification

This micro-example is not the exact RLP training objective; it is the small arithmetic behind a broader “next-gen” idea from the lecture's neighborhood: spend extra compute on reasoning/search so a weak one-shot predictor becomes a stronger solver.

Step-by-step with real numbers

Suppose a hard math prompt has one-shot success probability \(p=0.4\). A greedy next-token-only run gives one answer, so success is simply:

$$P(\text{success in 1 try})=0.4$$

Now sample \(k=5\) independent reasoning attempts and use a verifier to pick any correct one. Failure means all five attempts fail:

$$P(\text{all fail})=(1-p)^k=(1-0.4)^5=0.6^5=0.07776$$

So pass@5 is:

$$P(\text{at least one succeeds})=1-(1-p)^k=1-0.07776=0.92224\approx0.92$$

The capability did not come from a different corpus; it came from inference-time search plus a way to recognize success. Epistemic status: the independence assumption is simplified; real samples are correlated, and verifier quality matters.

✓ Check yourself — what does a verifier enable that a raw LM sample does not?

It creates a selection signal. Without a verifier, five attempts are just five strings; with a verifier, search can branch, reject, revise, and keep the answer that satisfies the task.

✓ Check yourself — if \(p=0.2\) and \(k=5\), what is pass@5?

\(1-(1-0.2)^5=1-0.8^5=1-0.32768=0.67232\). Even a weak solver can improve sharply if attempts are diverse and verification is reliable.

5.RLP: learning by thinking during pre-training

The final and most distinctive part of the lecture is RLP, reinforcement as a pre-training objective. Shrimai motivates it with Leo and Bolt: Leo builds and tests a bridge; Bolt merely studies bridges and builds one that looks perfect but fails the toy-car test. Standard pre-training mostly learns by watching text. RLP lets the model generate thoughts, then rewards thoughts that improve the next-token prediction.

5.1 Information-gain reward

Given a prefix such as “Photosynthesis uses…”, vanilla pre-training predicts the next token directly. In RLP, the model first produces a thought like “plants use solar energy,” then predicts “sunlight.” The reward asks: did that thought actually increase the log-probability of the true token compared with a no-think baseline?

Key equation — RLP information-gain reward

$$r = \log p_{\theta}(x_t\mid x_{<t}, z)-\log p_{\phi}(x_t\mid x_{<t})$$

\(z\) is the generated thought trace. \(p_{\theta}\) is the thought policy's probability of the true next token after thinking. \(p_{\phi}\) is the no-think baseline, updated as a lagged exponential moving average. Positive reward means the thought helped; zero or negative reward means it did not.

Why this is different from ordinary next-token prediction

Next-token prediction trains the answer token. RLP also trains the process that made the answer token more predictable. Because the reward is dense and verifier-free, the lecture argues it can be applied to ordinary pre-training text—not only curated math problems.

Reported evidence

In the talk, RLP improved Qwen3-1.7B-Base by about 19% over the base model and about 17% over token-matched continued next-token pre-training. A FLOP-matched comparison still favored RLP by about 14% despite the next-token baseline seeing many more tokens. On a 12B Nemotron hybrid model, RLP from an intermediate checkpoint produced large gains despite using far fewer additional tokens. These are reported experimental results; exact transfer depends on model, data, and implementation.

✓ Check yourself — why use a lagged no-think baseline?

It stays current enough to be a meaningful comparison, but its lag makes it harder for the thought policy to exploit a moving reward in degenerate ways. The reward asks whether thinking helps relative to a similar model that did not think.

6.End-to-end trace — one hard problem, two learners

Follow a single multi-step problem through two systems. The point is not that every future model must use exactly this pipeline; it is to separate objective from capability.

StepNext-token-onlyNext-gen approachnote
0 · promptReads the question and begins emitting answer tokens.Reads the question, identifies it as multi-step, and allocates reasoning/search budget.same input
1 · planNo explicit planning requirement; any plan must emerge inside the sampled text.Generates a plan or thought trace before the final answer.front-loaded / test-time reasoning
2 · branchUsually one path; greedy decoding cannot backtrack.Samples several candidate derivations or search branches.compute shifts to inference
3 · verifyConfidence is implicit in token probabilities, which can be miscalibrated.Uses verifier, reward, unit test, calculator, or consistency check to score candidates.selection signal
4 · reviseErrors early in the sequence tend to propagate.Rejects failed branches, revises, or calls tools before committing.search + tool use
5 · learnWeights change only in later training; deployment does not create persistent learning by default.Future-facing systems may add memory/continual learning; this is a broader next-gen direction beyond the lecture's main evidence.speculative / active research
6 · answerOutputs the sampled answer token-by-token.Outputs the answer selected after reasoning, verification, and possible revision.higher reliability if verifier is sound
What the trace reveals

Train-time compute taught the base model a distribution; test-time compute and reward signals can turn that distribution into a problem-solving procedure. The lecture's core message is not “next-token prediction failed.” It is: next-token prediction is an objective, while intelligence is a capability stack built from data curriculum, reasoning data, exploration, verification, memory, tools, and grounding.

Epistemic status of broader “next-gen” claims

The transcript strongly supports curriculum, front-loaded reasoning, and RLP. It only lightly touches broader themes such as agents, persistent memory, hallucination mitigation, and alignment/RLHF bias during Q&A. Treat those as adjacent active research directions, not as the main empirical claims of the lecture. For world-model context, see Note 43.

✓ Check yourself — name two limits of pure next-token prediction that the trace exposes.

It lacks an explicit requirement to plan/search/backtrack, and it lacks an external or intrinsic verifier that selects among attempts. The lecture also emphasizes that it learns by observing text, while RLP adds learning by thinking.

Glossary

TermPlain meaning
Autoregressive LMA model that predicts each token from the previous tokens.
Natural distributionSampling data sources roughly according to how many tokens they contain, regardless of quality.
Optimal blendA data mixture whose weights reflect estimated source quality and useful repeat counts.
Epoch estimationEstimating how many times to repeat a data source before downstream returns diminish.
Front-loading reasoningIncluding reasoning traces during pre-training so the base model learns reasoning earlier.
SFTSupervised fine-tuning: training on curated input-output examples, often to imitate instruction or reasoning formats.
RLVRReinforcement learning with verifiable rewards, such as exact answers, tests, or checkers.
RLPReinforcement as a pre-training objective: reward generated thoughts that improve next-token prediction.
Information gainHere, the improvement in log-probability of the true next token when conditioning on a thought trace.
