Transformers in Language: The Development of GPT Models

How the tiny instruction “predict the next token” became the engine behind BERT, GPT, few-shot prompting, scaling laws, and modern language-model capability.

🎓 CS25 V1 · Lecture 2 ⏱ ~50 min ▶ Watch on YouTube
Why this lecture matters

At first, language modeling sounds narrow: “given some text, continue it.” The lecture’s big idea is that this is a self-supervised compression test for understanding. To predict the next word in a mystery novel, a model may need characters, motives, chronology, style, and the reveal. Scale that objective across the internet, and “autocomplete” becomes a training signal for translation, summarization, sentiment, arithmetic, coding, and instruction following.

Running analogy

Think of the model as an apprentice novelist. Every day it reads a page with the last word covered, guesses the missing word, checks the answer, and updates its instincts. Nobody labels “this is sentiment” or “this is translation”; the apprentice learns those skills because they help it keep the story coherent.

Prerequisites — 30-second refresher

Skim these; dive into any that feel shaky.

  • Softmax — turns logits into a probability distribution over the vocabulary.
  • Language-model objective — predict the next token given the previous ones.
  • Sub-word token — text is chopped into word-pieces (BPE), not whole words.
  • Cross-entropy loss — \(-\log\) of the probability the model assigned to the correct token.

1.WHY language modeling: the self-supervised objective

Concrete example first. If the text is:

The capital of France is ___

the useful answer is probably Paris. If the text is a whole news article followed by TL;DR:, the useful continuation is a summary. If it is French: bonjour\nEnglish:, the continuation is a translation. The same next-token interface can look like many tasks.

WHY before HOW

Language is a unusually good domain for self-supervision because labels are built in. Every sentence supplies its own targets: token 2 predicts token 3, token 3 predicts token 4, and so on. The internet is therefore not just data — it is a giant worksheet with answer keys hidden one position to the right.

✓ Check yourself — why can “just predict the next word” learn sentiment?

Because sentiment helps predict future text. In a review, knowing that the hidden tone has turned negative changes the likely next words (“boring,” “waste,” “returned”) versus positive ones (“excellent,” “loved,” “recommend”). The lecture highlights OpenAI’s 2017 sentiment neuron: a next-character LSTM learned a feature strongly aligned with review polarity without sentiment labels.

1.1 Tokenization: why sub-word / BPE tokens

Before a transformer sees text, the text is chopped into tokens. The extremes are both unattractive:

  • Characters: tiny vocabulary and no unknown words, but sequences become very long; “internationalization” is 20 positions.
  • Words: shorter sequences, but huge vocabulary; rare names, typos, and new words become brittle out-of-vocabulary cases.

Byte Pair Encoding (BPE) and related sub-word tokenizers split the difference: frequent chunks become single tokens, rare words decompose into reusable pieces. For example, unbelievable might become un + believ + able. GPT models usually operate on sub-word tokens, not clean English words.

Pitfall

When a paper says “predict the next word,” read it as “predict the next token.” Tokens may be word pieces, bytes, punctuation, or whitespace-sensitive chunks. This matters for counting context length and for surprising failures on spelling, arithmetic, and rare names.

✓ Check yourself — why not use whole words only?

A whole-word vocabulary explodes in size and still cannot gracefully handle new strings such as product names, URLs, misspellings, or morphology-rich languages. Sub-words keep sequences shorter than characters while still allowing any word to be represented.

2.Two families: BERT understands, GPT generates

Transformers are sequence models, but the masking pattern and training objective determine their personality. The two language families from this era are encoder-only BERT-style models and decoder-only GPT-style models.

2.1 BERT: encoder-only masked language modeling

BERT uses a transformer encoder. During pretraining, some input tokens are replaced by [MASK], and the model predicts the missing token using both left and right context. That bidirectionality makes it excellent for understanding tasks such as classification, entailment, and question answering after fine-tuning.

The original BERT also used a next-sentence prediction (NSP) pretraining task: given two segments, predict whether the second follows the first in the corpus. Later models debated or removed NSP, but for BERT’s historical recipe it was part of the package.

Why BERT is not a generator

BERT is trained to fill holes with full context, not to emit text left-to-right. It is like a proofreader with the whole paragraph visible. Great for “what does this sentence mean?” Less natural for “write the next paragraph.”

2.2 GPT: decoder-only causal language modeling

GPT uses a transformer decoder with a causal mask: position \(t\) can attend only to positions \(\le t\), never future tokens. This matches generation. The model predicts one token, appends it to the prompt, then predicts the next.

Analogy

BERT is a crossword solver: blanks inside a finished grid, clues on both sides. GPT is an improv actor: it hears everything said so far, cannot see the script ahead, and must say the next line in character.

✓ Check yourself — what changes between BERT and GPT attention?

BERT tokens may attend bidirectionally to left and right context because the goal is to reconstruct masked tokens. GPT tokens use a triangular causal mask so token \(t\) cannot inspect \(x_{>t}\), matching the generation setting.

3.The pretrain → adapt paradigm

The lecture frames GPT not as “text continuation for its own sake,” but as a route to unsupervised representation learning. First learn from huge unlabeled text. Then adapt that model to the small labeled task you actually care about — or, for larger GPTs, describe the task in the prompt.

Concrete before general

For sentiment analysis, you could label millions of reviews. Or you can pretrain on raw reviews and the web, then learn a tiny classifier or prompt format. The model already knows that “battery died after one hour” and “absolutely loved it” point in different directions.

✓ Check yourself — what is being transferred?

Not a list of labels, but parameters that encode reusable features: syntax, entities, sentiment, discourse patterns, factual associations, and task formats. Fine-tuning or prompting reuses those features for a downstream objective.

4.The GPT scaling story

The OpenAI GPT line is a clean historical progression: first prove pretraining helps fine-tuning, then ask whether tasks can be expressed as prompts, then scale until prompts themselves become training examples.

4.1 GPT-1: pretrain then fine-tune

GPT-1 pretrained a decoder-only transformer on internet text, then fine-tuned it on supervised tasks such as natural language inference, question answering, semantic similarity, and classification. The architectural change for tasks was deliberately small — often a task head on top — showing that one pretrained language model could adapt broadly.

4.2 GPT-2: zero-shot tasks as text completion

GPT-2 pushed the “task as prompt” idea. Reading comprehension becomes passage + question + Answer:. Summarization becomes article + TL;DR:. Translation becomes French sentence translated into English means:. The model is still only completing text, but the prompt makes the completion behave like a task solution.

4.3 GPT-3: in-context / few-shot learning

GPT-3 scaled to 175B parameters and made in-context learning vivid: put examples in the prompt, and the model infers the pattern with no weight updates. The “learning” happens in the activations over the current context, not by gradient descent.

Translate English to French.

English: The book is on the table.
French: Le livre est sur la table.

English: I like black coffee.
French: J'aime le café noir.

English: Where is the train station?
French:

A GPT-3-style model can infer that it should output a French translation, even though no parameters are changed during this interaction. The transcript emphasizes this as an inner loop: at inference time, the model rapidly picks up the task from the prompt and predicts the next token accordingly.

Important distinction

Fine-tuning changes weights. Few-shot prompting changes only the input context. If the browser tab is the classroom, in-context learning is scratch-work on the whiteboard; fine-tuning is rewiring the student’s brain after class.

✓ Check yourself — why is few-shot prompting surprising?

Because classic ML usually needs gradient updates on labeled examples. GPT-3 can see a handful of examples inside the prompt and immediately continue in the same format, suggesting that scale turns next-token pretraining into a meta-learning-like ability.

5.Scaling laws: why bigger predictably helps

The transcript includes the natural skeptical question: is a tiny perplexity gain worth a huge model? Kaplan et al. gave a useful answer: over wide ranges, language-model loss falls approximately as a power law in model size, data size, and compute. On a log-log plot, a power law looks like a line.

Power-law intuition

A simplified scaling-law shape is \(L(N) \approx A N^{-\alpha} + L_\infty\), where \(N\) is parameter count and \(L_\infty\) is the irreducible entropy floor. The exponent \(\alpha\) is small, so returns are predictable but expensive.

Pitfall

A small average loss improvement can still change qualitative behavior. Near the entropy floor, “one more bit” may be the difference between generic prose and tracking a character, variable, or task format for many paragraphs.

✓ Check yourself — what does “straight line on a log-log plot” mean?

It means the relationship is multiplicative: multiplying scale by a fixed factor tends to reduce loss by a predictable factor. That predictability made it rational to train GPT-3 after fitting trends on smaller models.

6.Emergent abilities & prompting

As models scale, some capabilities appear to turn on only past a threshold. GPT-3 showed sharper gains on tasks like arithmetic and word unscrambling when examples were placed in context. Later work called these emergent abilities: abilities not present in small models but visible in sufficiently large ones.

How to think about emergence

Imagine a student learning many weak clues: digit patterns, equation formatting, answer styles, examples in textbooks. Below a certain skill level, none of this is enough for reliable addition. Above it, the same prompt suddenly lets the student infer “this is an addition worksheet” and follow the pattern.

This also sets up prompting as an interface. We do not only ask for an answer; we shape the computation by giving instructions, examples, rubrics, or intermediate reasoning formats. A later teaser is chain-of-thought prompting: ask the model to produce reasoning steps before the final answer, which can improve multi-step reasoning in large models.

✓ Check yourself — is in-context learning the same as memorizing the examples?

No. Memorization can help with seen strings, but in-context learning is about inferring a task pattern from examples and applying it to a new query in the same prompt. The weights stay fixed; the context supplies temporary task specification.

7.Limitations: what the objective does not solve

The lecture’s story is powerful but not magic. A next-token model learns to produce likely text, not guaranteed truth, safety, or cheap computation.

  • Hallucination. The model can generate plausible but false continuations because likelihood is not truth.
  • Context length. A decoder only conditions on tokens inside its context window; important information outside the window is invisible unless retrieved or summarized.
  • Cost. Scaling improves loss but requires more parameters, data, energy, memory, and inference latency.
  • Alignment. Pretraining imitates internet text; it does not by itself optimize helpfulness, harmlessness, or honesty. This leads to instruction tuning and RLHF in later lectures.
Why RLHF enters the story

The pretraining question is “what token comes next?” The user-facing assistant question is “what response should a helpful, safe assistant give?” Those are related but not identical objectives, so later systems add human preference data and reinforcement learning from human feedback.

9.Worked numeric micro-example: one next-token step

Suppose the vocabulary has only four possible next tokens, and the model emits logits \(z=[2.0,1.0,0.1,0.5]\). Softmax exponentiates each logit, then normalizes by the sum:

Softmax by hand

$$\exp(z)\approx[7.39,2.72,1.11,1.65],\qquad \sum_i \exp(z_i)\approx12.87$$

$$\operatorname{softmax}(z)=\frac{\exp(z_i)}{\sum_j \exp(z_j)}\approx[0.60,0.22,0.09,0.13]$$

If the target is token 1, the cross-entropy loss is the negative log-probability assigned to that correct token:

$$\mathcal{L}=-\log(0.60)\approx0.51$$

Temperature changes how sharp this distribution is. With \(T=2\), divide logits by 2 before softmax:

Higher temperature flattens

$$z/T=[1.0,0.5,0.05,0.25]$$

$$\operatorname{softmax}(z/T)\approx[0.41,0.25,0.16,0.19]$$

The largest token is still most likely, but the lower-scoring tokens get more probability mass, so sampling becomes more diverse.

✓ Check yourself — what does temperature \(T \to 0\) do?

It makes the distribution collapse toward the highest-logit token. In the limit, sampling behaves like greedy argmax: almost all probability goes to the single most likely next token.

10.End-to-end trace: one GPT-style decoding loop

Here is the concrete path from a prompt to the next generated word, using \(d=768\) hidden dimensions and a vocabulary size \(V\approx50{,}000\).

StepWhat happensShape
Prompt"The capital of France is"raw text
TokenizeSplit into 5 sub-word tokens, e.g. The, capital, of, France, is.\(5\)
Token + position embeddingsLook up token embeddings and add position embeddings.\(5\times768\)
Causal transformer stackApply masked self-attention and MLP blocks so each position summarizes only the prefix up to itself.\(5\times768\)
Take last positionUse the hidden state for the final prompt token, because it is the state that has seen the whole prompt prefix.\(768\)
Unembed to logitsProject the last hidden state to one score per vocabulary token.\(V\approx50\text{k}\)
SoftmaxConvert logits into next-token probabilities.\(V\approx50\text{k}\)
Argmax / sampleChoose a token from the distribution.one token id
Decode, append, repeatDecode the token as "Paris", append it to the context, then run the loop again for the following token.\(6\) tokens next
Why only the last position matters during generation

At a decoding step, the question is “what token comes after the entire prompt?” The earlier positions’ logits would answer easier questions such as “what token came after The?”—tokens we already know. The causal mask makes every position a valid prefix predictor, so the model can process the prompt in parallel, then use only the final position’s logits for the next token. When Paris is appended, cached attention keys and values let the next step reuse the old prefix instead of recomputing everything from scratch.

Glossary

TermPlain meaning
Language modelA model that assigns probabilities to text, usually by predicting the next token from prior tokens.
TokenThe discrete unit fed to the model: a word piece, byte chunk, punctuation mark, or similar text fragment.
BPEByte Pair Encoding; a sub-word tokenization method that merges frequent character sequences into reusable pieces.
MLMMasked Language Modeling; hide tokens and predict them from surrounding context.
Causal maskA triangular attention mask preventing a token from attending to future positions.
AutoregressiveGenerates each token conditioned on previously generated tokens.
Fine-tuningUpdating pretrained model weights on a downstream dataset.
Zero-shotSolving a task from instructions or prompt format without examples for that task.
Few-shot / in-context learningSolving a task after seeing a few examples inside the prompt, with no weight updates.
Scaling lawAn empirical power-law relationship between loss and model/data/compute scale.
Emergent abilityA capability that is weak or absent in small models but appears clearly past some scale.
RLHFReinforcement Learning from Human Feedback; training with human preference signals to align behavior.
