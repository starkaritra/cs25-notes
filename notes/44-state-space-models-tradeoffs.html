On the Tradeoffs of State-Space Models and Transformers

Albert Gu’s core claim: attention and modern state-space models are not merely “slow vs fast”; their different autoregressive memories create different modeling strengths.

🎓 Albert Gu ⏱ ~50 min ▶ Watch
Why this matters

Transformers changed sequence modeling because every token can look back at every earlier token. That power is expensive: training attention forms roughly all pairwise token scores, and inference keeps a growing KV-cache. As contexts stretch to books, codebases, genomes, audio streams, and agent histories, the question becomes: do we need to store every past token, or can a model carry a compact state?

State-space models (SSMs), including Mamba-like models, promise linear scaling and constant-size recurrent memory. The price is that the fixed state is a compression of the past. This note is about that exact bargain.

Running analogy — database vs brain

A transformer is like a database: it writes down a record for every previous token, then queries those records precisely. An SSM is like a brain-like working memory: it continuously updates a fixed internal state, keeping the gist and task-relevant signals while inevitably losing some exact details. Hybrids are the practical compromise: a brain with a scratchpad.

Prerequisites — 30-second refresher

You’ll follow this note smoothly if these are fresh. Each is one line; skim, and jump back if one feels shaky.

  • Attention cost — vanilla self-attention compares every token with every other token, giving the O(N²) train-time score matrix discussed in Note 1; this is the baseline SSMs are trying to avoid.
  • KV-cache — at autoregressive inference, a transformer stores past keys and values so the next token can attend to them; cache memory grows with context length.
  • Recurrence / RNN — a model updates a hidden state one token at a time, so the next output depends on the state rather than rereading all past tokens.
  • Convolution — the same linear recurrence can be unrolled into a filter over the whole input sequence, which makes parallel training possible.
  • Hidden state — the compact vector or matrix that carries information forward; in SSMs it is the bottleneck through which the past must pass.

1.The computational bargain

Imagine continuing a 100,000-token document. A transformer can ask: “which exact previous token matters now?” That is wonderful for copying an ID number or retrieving a named fact. But it must keep a growing database of past representations. An SSM instead updates a fixed state at each token. That is wonderful for streaming and long sequences, but it cannot preserve every exact past token unless the fixed state has somehow encoded it.

1.1 Database vs brain

Gu frames the tradeoff through the autoregressive state: what the model stores between generation steps. For attention, that state is the KV-cache. For an SSM, it is the hidden state. The state definition predicts the model’s strengths.

The all-attention transformer: its training attention matrix grows from N² to (2N)² = 4N² scores, and its KV-cache doubles. The SSM still processes twice as many tokens overall, but the per-step state size stays fixed.

2.What an SSM is

Before naming S4 or Mamba, build the smallest picture. A state-space model maps an input sequence to an output sequence by repeatedly updating a hidden state. The hidden state is the model’s compressed memory.

Key equation — continuous and discrete state space

Continuous form:

$$h'(t)=Ah(t)+Bx(t),\qquad y(t)=Ch(t).$$

Discretized sequence form:

$$h_t=\bar A h_{t-1}+\bar B x_t,\qquad y_t=C h_t.$$

Here, \(x_t\) is the input token representation at step \(t\), \(h_t\) is the hidden state after seeing it, \(\bar A\) controls what past state is retained, \(\bar B\) controls how the new input enters memory, \(C\) reads the state into an output, and \(y_t\) is the emitted representation.

2.1 The same SSM has two views

At inference time, the recurrent view is natural: update \(h_t\) online and never store the whole past. At training time, the linearity lets the recurrence be unrolled into a convolutional kernel, so many positions can be processed in parallel. This dual identity is why modern SSMs can be both streaming-friendly and trainable on GPUs.

Common misconception

“SSM” does not mean “old RNN with a new name.” The defining recurrence is linear inside the sequence mixer, the state is often much wider than the scalar/channel input, and the implementation uses structure, scans, or matrix multiplications to train efficiently.

✓ Check yourself — why does linearity matter for training?

Because linear updates compose associatively. That makes it possible to rewrite or parallelize the recurrence with convolution/scan-like algorithms instead of executing every time step serially during training.

3.From S4 to Mamba

The talk’s architecture arc starts with structured state spaces and ends with selective ones. S4 made long-range state-space layers practical by using structured matrices and HiPPO-style initialization to preserve long-range memory. Mamba’s upgrade was to let the recurrence become content-dependent.

3.1 Selectivity: content decides what to remember

In a plain linear time-invariant SSM, \(\bar A,\bar B,C\) are fixed across the sequence. Mamba makes them functions of the current input: the model can decide, token by token, whether to keep the previous state, write new information, or flush stale information. This is the attention-like ingredient: not pairwise lookup, but input-dependent control.

Key equation — selective SSM

$$h_t=\bar A(x_t)h_{t-1}+\bar B(x_t)x_t,\qquad y_t=C(x_t)h_t.$$

The parentheses mean the parameters are produced from the token representation \(x_t\). If a token is unimportant, the model can reduce the write; if it is a boundary or a key fact, it can change how the state is updated.

✓ Check yourself — what can selectivity do that a fixed SSM cannot?

It can change the update rule based on content. For example, it can preserve state through filler tokens, write strongly on a rare keyword, or reset on a boundary. A fixed SSM applies the same dynamics everywhere.

4.The tradeoff: recall vs efficiency

This is the crux of the lecture. The SSM’s fixed state is not merely an implementation trick; it is an inductive bias. It encourages compression. That can help on streams, raw modalities, and abstraction-building, but it hurts tasks that demand exact recall.

4.1 The same coin has two sides

Core intuition — the bottleneck is the memory

Transformers: store all past token representations in the KV-cache. This gives near-database-like recall, but costs O(N) cache memory and O(N²) training attention.

SSMs: compress history into a fixed-size state. This gives O(1) recurrent memory and O(N) sequence processing, but the compression is lossy. Exact copying, associative recall, and “needle in a haystack” retrieval are naturally harder.

Pitfall — “linear” does not mean “strictly better”

If the task genuinely requires keeping a precise record of many past details, attention’s apparent inefficiency may be useful work. The point is not to declare attention wasteful everywhere; it is to ask whether each flop and each cached token is buying the capability the task needs.

✓ Check yourself — which model would you prefer for copying a random 20-digit code from 30k tokens ago?

All else equal, attention or a hybrid with enough attention. The task asks for exact retrieval of an arbitrary string, which is exactly where a growing cache is valuable and a fixed lossy state is strained.

5.Hybrids and raw data

Gu emphasizes that modern large models are increasingly hybrid: Jamba, Zamba, Samba, Hunyuan, Nemotron, Qwen variants, Kimi-Linear, and OLMo-style releases mix linear layers with attention. The recurring empirical pattern: many linear/SSM layers plus a smaller number of attention layers often works better than either extreme.

5.1 Attention as a scratchpad, SSM as the main processor

Early hybrid ablations often found around a 10:1 ratio of linear layers to attention layers from a perplexity standpoint; newer models still commonly use several linear layers per attention layer, such as 3:1 or 4:1. The analogy predicts this: the brain-like state does most streaming processing, while attention is the external scratchpad for precise lookup.

Beyond efficiency — raw data and tokenization

The talk’s later examples sharpen the point. Transformers work best when the input tokens already sit at the right semantic level: BPE tokens in language, patches in vision. On bytes, characters, DNA bases, or data where tokenization is unclear, SSM-like compressive models can be stronger because not every raw symbol deserves its own database row.

H-Net extends this idea: learn dynamic chunking end-to-end, compress raw data into higher-level units, and use SSMs especially in the outer layers that touch fine-grained data.

✓ Check yourself — why might attention struggle on byte-level language?

A single byte or character is often not a meaningful object to query. Attention spends memory and compute caching every tiny symbol, while a compressive model can gradually build word- or phrase-like abstractions.

6.Worked micro-example — one tiny SSM by hand

Now compute the smallest honest SSM recurrence. Let \(\bar A=0.9\), \(\bar B=0.5\), \(C=1\), initial state \(h_0=0\), and inputs \(x=[1,0,0]\). The recurrence is \(h_t=0.9h_{t-1}+0.5x_t\), \(y_t=h_t\).

Step-by-step with real numbers

At \(t=1\):

$$h_1=0.9\cdot 0+0.5\cdot 1=0.5,\qquad y_1=1\cdot h_1=0.5.$$

At \(t=2\):

$$h_2=0.9\cdot 0.5+0.5\cdot 0=0.45,\qquad y_2=0.45.$$

At \(t=3\):

$$h_3=0.9\cdot 0.45+0.5\cdot 0=0.405,\qquad y_3=0.405.$$

The input pulse is still present, but it decays. That is fixed-state compression in miniature: old information survives only through the dynamics of the state.

✓ Check yourself — what would a larger \(\bar A\), closer to 1, do to long-range memory?

It would decay more slowly. For example, with \(\bar A=0.99\), the state after the pulse would be \(0.5, 0.495, 0.49005,\ldots\). Longer memory improves retention, but it can also make it harder to forget irrelevant information.

When to pick which

Choose SSM-heavy designs for long-context processing, streaming, raw modalities, and settings where compression or abstraction is the point. Choose attention-heavy designs when exact recall, copying, tool traces, or sparse lookup over many facts is central. In many frontier systems, use hybrids: SSMs for the main flow, attention for the scratchpad.

Glossary

TermPlain meaning
Autoregressive stateWhatever a model keeps between one generated token and the next: KV-cache for attention, hidden state for an SSM.
KV-cacheThe stored keys and values from previous tokens that let a transformer avoid recomputing the full prefix at inference.
State-space model (SSM)A sequence model that updates a hidden state through equations such as \(h_t=\bar A h_{t-1}+\bar Bx_t\).
S4A structured state-space architecture that made long-range SSM layers practical, including HiPPO-inspired memory structure.
MambaA selective SSM architecture where update parameters depend on the input, implemented with hardware-aware scan algorithms.
SelectivityInput-dependent control over what to remember, write, or forget in the recurrent state.
Hybrid modelA model that interleaves attention layers with SSM/linear recurrent layers to balance recall and efficiency.
Dynamic chunkingLearning how to group raw low-level data into larger meaningful units instead of relying only on a fixed tokenizer.
