STANFORD CS25 · TRANSFORMERS UNITED · V2

Neuroscience-Inspired Artificial Intelligence

Two guest talks ask whether transformer attention is an alien engineering trick—or a rediscovery of older memory computations studied in brains.

🎓 Trenton Bricken & Will Dorrell ⏱ ~50 min ▶ Watch
Why this matters

The brain is the only existing example of broadly flexible intelligence. The lecture's careful question is not "are transformers brains?" but: which computations in transformers resemble biologically plausible memory and prediction mechanisms, and what can each field learn from the other?

Running analogy — a city librarian with a map

Imagine a librarian who stores postcards around a vast city. A partial clue does two things: it points to nearby shelves by address, and it uses the city map to infer where you are. SDM/attention is the shelf lookup; hippocampal cognitive maps are the city map; prediction is guessing the next street or postcard before you arrive.

Prerequisites — 30-second refresher

Skim these before the main argument. They are the only tools the lecture repeatedly uses.

  • Attention — for one query, transformer attention is \\(\operatorname{softmax}(\beta qK^\top)V\\): score stored keys, normalize scores, then average values.
  • Energy minimization — an energy-based memory has preferred low-energy states; retrieval moves a noisy cue downhill toward a stored pattern.
  • Associative memory — content-addressable recall: give part of a pattern and retrieve the completed stored pattern, like a song from its first few notes.
  • Self-supervised prediction — learn by predicting hidden or future observations, such as the next word or the next view after a movement.

1.The two-way street: neuroscience → AI and AI → neuroscience

Start with the lecture's concrete examples. Trenton Bricken asks why transformer attention's useful but heuristic softmax might be implementable by simple high-dimensional geometry. Will Dorrell asks why a hippocampal-entorhinal model built for cognitive maps ends up looking surprisingly transformer-like.

1.1 What the lecture actually asks

The talks are not a generic tour of "brain-inspired AI." They center on two bridges:

  • Sparse Distributed Memory (SDM) → attention: Kanerva's biologically motivated memory stores patterns near sparse high-dimensional addresses. Bricken shows its read operation approximates softmax attention.
  • Hippocampal cognitive maps → transformer-like recall: the Tolman-Eichenbaum Machine (TEM) learns reusable structure, stores observations at positions, and recalls them through Hopfield/attention-like memory.
Epistemic status

Established math: modern Hopfield retrieval and transformer attention share the same softmax-weighted update. Approximation: SDM circle intersections approximate exponentials in high dimensions. Analogy/speculation: cerebellar and hippocampal circuits may implement attention-like operations, but that is not the same as proving brains run transformers.

1.2 Prediction as the shared task

The predictive-coding slogan says cortex constantly predicts inputs and propagates errors. The transcript does not develop predictive coding as a full cortical theory, so treat it as a background lens: both speakers use prediction tasks. Bricken's example is next-token prediction in "the cat sat on the ___"; Dorrell's is predicting what an agent will see after moving in a structured world.

Key equation — self-supervised prediction objective

$$\min_\theta \; \mathbb{E}\big[-\log p_\theta(o_{t+1}\mid o_{\le t},a_{\le t})\big]$$

Here \\(o_t\\) is an observation, \\(a_t\\) is an action or movement when present, and \\(\theta\\) are learned parameters. For language, \\(o_{t+1}\\) is the next token; for navigation, it is the next sensory item at the next position.

✓ Check yourself — what is the lecture's two-way claim?

Neuroscience offers memory mechanisms that make attention less mysterious; AI offers precise mathematical tools and architectures that can generate new neuroscience hypotheses.

2.Sparse Distributed Memory: attention before attention

Before transformers, Pentti Kanerva proposed a memory in which addresses are high-dimensional vectors, only a sparse subset of possible neurons exists, and a query reads from all nearby neurons. The library analogy is literal: you do not search by shelf number; you search by content similarity.

2.1 Write, read, denoise

In the lecture's picture, a pattern is written by activating all neurons within a radius around its address. Each active neuron stores the pattern in superposition with other patterns. Later, a noisy query activates nearby neurons; they vote out the patterns they stored; a majority/threshold step denoises the answer.

Key equation — SDM read as weighted pattern sum

$$\hat{x}=\operatorname{threshold}\!\left(\frac{\sum_{\mu=1}^{M}|N(q)\cap N(k_\mu)|\,v_\mu}{\sum_{\mu=1}^{M}|N(q)\cap N(k_\mu)|}\right)$$

\\(q\\) is the query address, \\(k_\mu\\) is the address of stored pattern \\(\mu\\), \\(v_\mu\\) is the value stored there, and \\(|N(q)\cap N(k_\mu)|\\) counts neurons activated both during the write and the read.

2.2 Why softmax appears

The central geometric fact is that in high dimensions, the intersection of two hyperspheres shrinks approximately exponentially as their centers move apart. That is exactly the shape softmax uses: nearby memories get exponentially more weight; distant memories fade quickly.

Key equation — SDM intersection approximates an exponential

$$|N(q)\cap N(k_\mu)|\approx c_1\exp\!\big(\beta\,\hat q\cdot \hat k_\mu\big)$$

The hats mean L2-normalized vectors, \\(\hat q\cdot\hat k_\mu\\) is cosine similarity, \\(c_1\\) cancels during normalization, and \\(\beta\\) controls sharpness: larger \\(\beta\\) means the librarian strongly favors the nearest shelf.

Substitute that approximation into the SDM read equation and the normalized weighted sum becomes attention:

Bridge equation — SDM read ≈ attention

$$\frac{\sum_\mu \exp(\beta\hat q\cdot\hat k_\mu)v_\mu}{\sum_\mu \exp(\beta\hat q\cdot\hat k_\mu)} =\operatorname{softmax}(\beta\hat qK^\top)V$$

This is Bricken's clean bridge: attention can be viewed as a pattern-based abstraction of an SDM read.

✓ Check yourself — why does \\(c_1\\) disappear?

Every numerator term and the denominator are multiplied by the same constant \\(c_1\\). Dividing by the denominator cancels it, leaving only the relative exponential weights.

3.Modern Hopfield networks ↔ transformer attention

SDM is one associative-memory bridge. The second, emphasized in Will Dorrell's part and in Ramsauer et al., is even more exact: a modern Hopfield network update is mathematically the same kind of softmax retrieval used by attention.

3.1 From classic Hopfield to modern Hopfield

A classic Hopfield network stores patterns as attractors: if the current state is near a memory, recurrent dynamics settle into that memory. The problem is capacity and interference. Modern Hopfield networks sharpen similarity with an exponential/softmax, so the closest memories dominate.

Key equation — modern Hopfield retrieval update

$$\xi^{\text{new}}=X\,\operatorname{softmax}\!\big(\beta X^\top \xi\big)$$

\\(X=[x_1,\dots,x_M]\\) stores memory patterns as columns, \\(\xi\\) is the current partial cue, \\(X^\top\xi\\) scores cue-to-memory similarity, and \\(\beta\\) sets how winner-take-most the retrieval is.

3.2 Attention is one retrieval step

Transformer attention uses the same recipe with different names. Keys are addresses, values are what gets returned, and the query is the partial cue. With \\(X\\) as both keys and values, it is Hopfield auto-association; with different \\(K\\) and \\(V\\), it is hetero-association.

Key equation — attention as associative retrieval

$$\operatorname{Attn}(q,K,V)=\operatorname{softmax}(\beta qK^\top)V$$

\\(qK^\top\\) asks "which stored addresses match this cue?" The softmax turns match scores into retrieval weights. Multiplication by \\(V\\) returns the associated contents.

Common misconception — "attention" means consciousness

Transformer attention is not visual attention, awareness, or a little executive deciding what matters. Here it is a precise algebraic operation: similarity-weighted retrieval. The analogy to brain mechanisms is useful only at that computational level.

✓ Check yourself — when is attention auto-associative versus hetero-associative?

Auto-associative retrieval uses the same stored pattern as address and content, so keys and values coincide. Hetero-associative retrieval stores one thing as the key and another as the value, as in transformer attention where \\(K\\) and \\(V\\) are different projections.

4.Cognitive maps: hippocampus, place cells, grid cells, and TEM

Now shift from "which memory did I store?" to "where am I in the structure of a world?" Dorrell's talk starts with a simple grid world: after moving north, east, then south, you can infer what moving west should do because you know the reusable structure of 2D space.

4.1 Place cells, grid cells, and structure

The neuroscience evidence is that the hippocampal-entorhinal system represents more than raw sensory snapshots.

  • Place cells in hippocampus fire at particular positions.
  • Grid cells in medial entorhinal cortex (MEC) fire on hexagonal lattices, suggesting a reusable coordinate system.
  • Non-spatial maps appear in tasks such as transitive inference and abstract "bird space," where people navigate dimensions like neck length and leg length.
Key equation — path-integrating a structural code

$$g_t=M_{a_t}g_{t-1}$$

\\(g_t\\) is a grid-like structural code, and \\(M_{a_t}\\) is an action-dependent transition matrix. If moving north then south returns you to the same place, the model should learn \\(M_{\text{south}}\approx M_{\text{north}}^{-1}\\).

4.2 TEM as a transformer-like memory

The Tolman-Eichenbaum Machine stores memories that bind a position-like code \\(g_t\\) with an observation code \\(x_t\\). Later, if it knows the structural position, it can recall the observation; if it sees the observation, it can infer the position. In the transformer mapping, positional encodings do more work than usual: they are path-integrated, action-updated codes.

Key equation — binding structure and content

$$m_t=\operatorname{bind}(g_t,x_t),\qquad \hat{x}_t=\operatorname{retrieve}(g_t,\{m_\tau\}_{\tau

\\(g_t\\) says "where/which structural state," \\(x_t\\) says "what I saw," and \\(m_t\\) stores their binding. A Hopfield/attention-like retrieve step compares the current \\(g_t\\) to previous structural codes and returns the associated \\(x\\).

Structural generalization

The librarian does not memorize every street-turn pair in every city. Once the city map is learned, a new city requires only binding new landmarks to known map positions. That is why TEM can learn new environments faster after training on many related ones.

✓ Check yourself — in TEM, what plays the role of a transformer key?

The position-like structural code \\(g_t\\) plays the key/query role: recall depends on similarity between the current structural position and previous structural positions. The value is the observation/content code \\(x_t\\).

5.Where the analogies break—and why that is productive

Good analogies make predictions, but bad analogies smuggle in false identity claims. The lecture repeatedly marks this boundary: attention-like math may be biologically plausible without implying that cortex, cerebellum, or hippocampus implements a literal Transformer block.

5.1 Biological plausibility without ordinary backprop

Bricken's cerebellar story maps SDM operations onto mossy fibers, granule cells, Purkinje cells, and climbing fibers. The important point for AI is not that backprop is hidden somewhere. It is that separate input and teaching/write signals can update synapses in a way that resembles hetero-associative memory.

Key equation — local write signal sketch

$$\Delta w_{i\to j}\propto \underbrace{\text{pre}_i(q)}_{\text{which address fired}}\;\underbrace{\text{teach}_j(v)}_{\text{what value to store}}$$

This is not the global chain-rule gradient used in standard transformers. It is a local associative update sketch: active address-like neurons and a teaching/value signal jointly change a synapse.

Predictive-coding and feedback-alignment theories are broader attempts to approximate useful credit assignment with biologically plausible signals. The transcript's concrete claim is narrower: cerebellar climbing fibers could supply a direct training/write signal for SDM-like storage, avoiding the need to posit full transformer backprop in the brain.

5.2 Brain attention versus transformer attention

Pitfall — same word, different mechanism

Do not equate them. Biological attention includes arousal, task goals, eye movements, neuromodulators, recurrent loops, and embodied constraints. Transformer attention is a differentiable matrix operation. The safe claim is: both can be studied as selective, similarity-weighted routing; the unsafe claim is: they are the same mechanism.

✓ Check yourself — name one reason the analogy breaks.

Examples: biological circuits are recurrent and embodied; they learn through local plasticity and neuromodulatory signals; transformer attention is a static differentiable operation at inference time; brains have many kinds of attention beyond similarity-based lookup.

6.Worked example — modern-Hopfield / attention associative retrieval

Store three memory patterns. A query is closest to pattern 2. We will compute the attention/Hopfield retrieval by hand and watch the output become pattern 2.

Step-by-step with real numbers

Let keys and values be the same three 2D patterns:

$$K=V=\begin{bmatrix}1&0\\0&1\\-1&0\end{bmatrix},\qquad q=\begin{bmatrix}0.1&0.9\end{bmatrix},\qquad \beta=5.$$

First compute similarity scores \\(s_i=q\cdot k_i\\):

$$s=\begin{bmatrix}0.1&0.9&-0.1\end{bmatrix}.$$

Sharpen with \\(\beta=5\\):

$$\beta s=\begin{bmatrix}0.5&4.5&-0.5\end{bmatrix},\qquad \exp(\beta s)\approx\begin{bmatrix}1.65&90.02&0.61\end{bmatrix}.$$

Normalize to get softmax weights:

$$\alpha=\frac{\exp(\beta s)}{1.65+90.02+0.61}\approx\begin{bmatrix}0.018&0.976&0.0066\end{bmatrix}.$$

Finally retrieve the value:

$$\operatorname{softmax}(\beta qK^\top)V=0.018\begin{bmatrix}1&0\end{bmatrix}+0.976\begin{bmatrix}0&1\end{bmatrix}+0.0066\begin{bmatrix}-1&0\end{bmatrix}\approx\begin{bmatrix}0.011&0.976\end{bmatrix}.$$

The retrieved vector is almost pattern 2, \\((0,1)\\). In the librarian analogy, the clue lands almost entirely on shelf 2; the returned postcard is shelf 2's content.

✓ Check yourself — what would a small \\(\beta\\) do?

With \\(\beta=1\\), the weights are roughly \\([0.247,0.550,0.203]\\), so the output is a blend: \\(0.247(1,0)+0.550(0,1)+0.203(-1,0)\approx(0.044,0.550)\\). Small \\(\beta\\) means softer retrieval: less pattern completion, more memory blending.

7.End-to-end trace — one associative-recall step

Follow the same event through two framings. The brain column describes a content-addressable memory story; the transformer column describes the matrix operation.

StepBrain viewTransformer view
0 · partial cueA noisy or incomplete pattern appears: part of a memory, location, or context.query q
1 · similarityMemory neurons / stored patterns respond according to how close their addresses are to the cue.scores s_i = q · k_i
2 · softmax / energy minAssociative dynamics favor low-energy, high-similarity memories; weak matches are suppressed.alpha = softmax(beta s)
3 · completed patternThe system settles toward the stored pattern or returns the associated value.output = alpha V
4 · interpretationPattern completion in cortex/hippocampus/cerebellum-style memory.attention lookup
Why associative memory is the unifying lens

Once you see attention as "retrieve what is stored at similar addresses," SDM, modern Hopfield networks, and transformer attention stop looking like unrelated tricks. They become different implementations of the same librarian move: partial cue → similarity search → sharpened weights → completed or associated content.

✓ Check yourself — where does "pattern completion" appear in the transformer equation?

It appears in the final weighted sum \\(\alpha V\\). If one weight dominates, the output is nearly the stored value for that key; if several weights are comparable, the output blends several memories.

Glossary

TermPlain meaning
Sparse Distributed Memory (SDM)A high-dimensional associative memory where sparse neurons store and retrieve patterns near their addresses.
Associative memoryA memory that retrieves content from a cue by similarity, rather than by an exact address.
SoftmaxA normalization that turns scores into positive weights summing to one, while amplifying large scores.
Temperature / \\(\beta\\)The sharpness of retrieval. Large \\(\beta\\) is winner-take-most; small \\(\beta\\) blends memories.
Modern Hopfield networkAn energy-based associative memory whose retrieval update uses softmax-weighted similarity to stored patterns.
Place cellA hippocampal neuron that fires strongly at a particular location in an environment.
Grid cellAn entorhinal neuron that fires at multiple locations arranged like a hexagonal grid.
Cognitive mapAn internal representation of relational structure, such as physical space or an abstract task space.
Tolman-Eichenbaum Machine (TEM)A model that separates reusable structure from sensory content and binds them in hippocampal-style memory.
Hetero-associationUsing one pattern as an address/key and a different pattern as the stored value to retrieve.
