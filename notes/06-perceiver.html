DeepMind's Perceiver & Perceiver IO: A General-Purpose Architecture

Perceiver asks: can one transformer-like backbone ingest pixels, audio, point clouds, language bytes, and multimodal streams without paying full quadratic attention over the raw input?

🎓 Andrew “Drew” Jaegle, DeepMind ⏱ ~50 min ▶ Watch
Why this matters

A vanilla transformer can compare every input item with every other input item. That is wonderful for generality, but brutal for high-bandwidth data: a 224×224 RGB image has 50,176 pixel locations, audio has long temporal streams, point clouds have unordered sets, and multimodal sensors mix incompatible shapes. Full self-attention over raw inputs costs \(O(N^2)\) in input length \(N\), so a “general” architecture becomes unusable exactly where perception is richest.

Running analogy

Think of a giant evidence warehouse. A standard transformer asks every crate to talk to every other crate. Perceiver sends in a small team of trained investigators — the latents — who interview all crates, meet privately many times, and later answer whatever report format you request. The warehouse can grow; the investigator team stays manageable.

Prerequisites — 30-second refresher

Skim these once; they are the whole mechanical substrate of the note.

  • Self-attention — every item supplies \(Q,K,V\) and attends to the same array; here, that creates an \(N\times N\) attention map.
  • Cross-attention — queries come from one array, keys/values from another; Perceiver uses latent queries to read the large input.
  • Q/K/V — a query asks, a key says “what I match,” and a value says “what information I carry.” Attention mixes values by query-key similarity.
  • Attention complexity — forming \(QK^\top\) for \(N\) queries and \(N\) keys costs \(O(N^2)\); replacing queries with \(M\ll N\) latents gives \(O(NM)\).
  • Positional encoding — attention alone is permutation-invariant; Fourier position features tell the model where pixels, samples, or points live.

1.Why Perceiver exists: generality is not enough

Imagine feeding an image as raw pixels rather than 16×16 patches. You get one row per pixel location. A normal transformer can, in principle, notice a relation between the top-left pixel and the bottom-right pixel in one layer — a non-local inductive bias. But it must also store and compute the full all-pairs table. For ImageNet-scale pixels, the all-pairs table is already billions of entries before depth, heads, batches, or channels are considered.

1.1 The scaling wall

The lecture frames the design pressure as a trade-off. Convnets are efficient and strong on grids, but they bake in locality and grid-specific assumptions. Transformers are more input-agnostic, but raw-input self-attention is quadratic. Perceiver tries to sit between them: retain transformer-like generality while making input cost linear in \(N\).

Key equation — full self-attention cost

$$ \operatorname{Attn}(X)=\operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V, \qquad Q,K,V\in\mathbb{R}^{N\times d_k} $$

The score matrix \(QK^\top\) has shape \(N\times N\). Here \(N\) is the number of input elements — tokens, pixels, audio frames, point-cloud points — and \(d_k\) is the attention key dimension. The matrix is the expensive warehouse-wide “everyone talks to everyone” meeting.

Common misconception

“The problem is only the quadratic attention matrix.” Not quite. Jaegle also emphasizes that standard transformers keep computation proportional to the large input size at every layer. Even linear-in-\(N\) work becomes heavy when \(N\) is tens of thousands and the network is deep.

✓ Check yourself — why not just use image patches?

Patches reduce \(N\), but they inject a 2D grid-specific preprocessing assumption. That may be excellent for images, yet less natural for bytes, point clouds, event cameras, multimodal arrays, or scientific measurements with unknown structure.

2.The Perceiver core: interview the input with a small latent array

Perceiver introduces \(M\) learned latent units, with \(M\ll N\). These latents are not input tokens. They are trainable abstract slots, like the investigator team in the warehouse analogy. The first step is not self-attention over the huge input; it is cross-attention from latents to input.

2.1 Cross-attend in: \(Q=\) latents, \(K,V=\) input

Let \(X\in\mathbb{R}^{N\times D}\) be the input array after content and position features, and \(Z\in\mathbb{R}^{M\times C}\) be the learned latent array. A cross-attention layer computes queries from \(Z\), keys and values from \(X\), then returns \(M\) updated latent rows.

Key equation — asymmetric input attention

$$ Q=ZW_Q\in\mathbb{R}^{M\times d},\quad K=XW_K\in\mathbb{R}^{N\times d},\quad V=XW_V\in\mathbb{R}^{N\times d_v} $$

$$ Z' = \operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt d}\right)V, \qquad QK^\top\in\mathbb{R}^{M\times N} $$

The attention map is rectangular. Each latent query asks all input elements for evidence, but the output length is \(M\), not \(N\).

2.2 Process cheaply: the latent transformer

Once the input has been distilled into \(M\) latent rows, Perceiver runs a normal transformer stack in latent space. This preserves the useful all-to-all comparison, but only among the investigators, not among every crate in the warehouse.

The original Perceiver can iterate this pattern: cross-attend into latents, process with latent self-attention, optionally cross-attend back to the input, and repeat. Jaegle notes that weight sharing across depth can help on limited-data settings, making the repeated blocks feel like an RNN unrolled through computation time.

Key equation — total Perceiver encoder scaling

$$ \text{input cross-attention: }O(NM),\qquad \text{latent self-attention per layer: }O(M^2),\qquad M\ll N. $$

For fixed \(M\), the input-facing part is linear in \(N\). Depth mostly compounds the cheaper \(M\times M\) latent computation.

What the latent bottleneck buys

The architecture chooses where to spend quadratic attention. It does not eliminate attention; it moves the deep all-to-all processing into a controllable latent workspace whose size is a hyperparameter.

✓ Check yourself — what changes shape from square to rectangle?

The attention score matrix. In full self-attention, \(QK^\top\) is \(N\times N\). In Perceiver input cross-attention, there are only \(M\) latent queries, so \(QK^\top\) is \(M\times N\).

3.Position without hard-wiring a modality

Attention treats an array as a set unless positions are supplied. If two identical red pixels are swapped, content alone cannot tell which came from the sky and which came from the jacket. Perceiver avoids image-specific convolutions, but it still needs spatial or temporal coordinates as features.

3.1 Fourier position features

The Perceiver papers often concatenate content features with Fourier encodings of position. For images, positions are 2D; for audio, 1D; for video, 3D. In the lecture, Jaegle notes that dense frequency bands up to the Nyquist frequency helped, and concatenation often beat addition for pixels.

Key equation — Fourier features for a coordinate

For a coordinate \(p\), a simple Fourier feature map is

$$ \gamma(p)=\big[\sin(2\pi f_1p),\cos(2\pi f_1p),\ldots, \sin(2\pi f_Bp),\cos(2\pi f_Bp)\big]. $$

For 2D pixels, apply this to \(x\) and \(y\) coordinates and concatenate with RGB or other content channels. Frequencies let nearby and far-apart positions be distinguishable at multiple scales.

Common misconception

“No modality-specific assumptions” does not mean “no positional information.” The cleanest version can learn anonymous position embeddings, but practical high-performing models often benefit from truthful structure such as 2D image coordinates or modality tags.

✓ Check yourself — why can permuting pixels be harmless for Perceiver?

If the 2D position features stay attached to each pixel row, the array order is irrelevant: attention sees a set of rows containing both content and coordinates. A convnet, by contrast, relies on neighboring memory layout and loses its built-in locality when pixels are shuffled.

4.Worked numeric micro-example — attention matrices by hand

Use the exact scale requested: \(N=1000\) input elements, \(M=32\) latents, and \(D=C=64\). The goal is not to compute learned weights; it is to see the shape and compute reduction that makes the architecture plausible.

Step-by-step with real shapes

Input and latent arrays:

$$ X\in\mathbb{R}^{1000\times 64},\qquad Z\in\mathbb{R}^{32\times 64}. $$

Choose one attention head with projected size \(d=64\). Then:

$$ Q=ZW_Q\in\mathbb{R}^{32\times64},\quad K=XW_K\in\mathbb{R}^{1000\times64},\quad V=XW_V\in\mathbb{R}^{1000\times64}. $$

The cross-attention score table is

$$ QK^\top:(32\times64)(64\times1000)=32\times1000, $$

so it has \(32\cdot1000=32{,}000\) scores. Full input self-attention would use

$$ 1000\times1000=1{,}000{,}000\text{ scores}. $$

The reduction is

$$ \frac{1{,}000{,}000}{32{,}000}=31.25. $$

So the input-facing attention map is about 31× smaller. More importantly, if \(M=32\) stays fixed, the cross-attention score count grows as \(32N\), not \(N^2\).

✓ Check yourself — if N doubles to 2000 and M stays 32, what happens?

Cross-attention becomes \(32\times2000=64{,}000\) scores: it doubles. Full self-attention becomes \(2000\times2000=4{,}000{,}000\) scores: it quadruples. The new reduction is \(4{,}000{,}000/64{,}000=62.5\times\).

✓ Check yourself — if M doubles to 64 while N=1000, what is the cross-attention table?

It is \(64\times1000=64{,}000\) scores. That is still far below \(1{,}000{,}000\), but the bottleneck is less tight and the latent transformer also becomes costlier because \(M^2\) changes from \(32^2=1024\) to \(64^2=4096\) per latent layer.

5.Perceiver IO: generalize the output too

The original Perceiver is strongest as an encoder for classification or regression: big arbitrary input in, compact latent state out, task head on top. Perceiver IO extends the same asymmetry to the decoder. Instead of forcing one fixed output, it lets a learned or constructed output query array ask the final latents for exactly the outputs needed.

5.1 Query the output you want

For image classification, the output query can be one row asking for class evidence. For dense optical flow, there can be one query per pixel asking, “what is the motion at this location?” For masked language modeling, each query can encode a masked byte or character position. The decoder cross-attends from output queries to the final latents.

Key equation — IO decoder cross-attention

Let \(Y\in\mathbb{R}^{O\times C_y}\) be \(O\) output queries and \(H\in\mathbb{R}^{M\times C}\) be final latents.

$$ Q_{out}=YW_Q,\quad K_{lat}=HW_K,\quad V_{lat}=HW_V, $$

$$ \operatorname{Decoder}(Y,H)=\operatorname{softmax}\!\left(\frac{Q_{out}K_{lat}^{\top}}{\sqrt d}\right)V_{lat}, \qquad Q_{out}K_{lat}^{\top}\in\mathbb{R}^{O\times M}. $$

Now output cost scales linearly with the number of requested outputs \(O\), while the expensive memory being queried remains the fixed-size latent array.

The IO move in the warehouse analogy

The investigator team finishes its private meeting. A report request arrives: one executive summary, 50,000 pixel-level labels, or several task-specific answers. Each report line asks the investigators a targeted question; the warehouse does not need to be reread from scratch.

✓ Check yourself — what is an output query?

It is a feature vector describing one requested output element. For classification, it can be a learned “class summary” query. For dense prediction, it often includes the position and output semantics for one pixel, point, or time step.

6.End-to-end trace — one ImageNet-style example

Follow a single raw-pixel image through Perceiver IO. The exact channel widths vary by implementation; this trace uses the requested representative shapes and marks where the \(O(N^2)\) input attention is avoided.

What the trace reveals

The model never forms a pixel-by-pixel \(50{,}176^2\) attention table. It spends one linear pass reading the image into 512 latents, then performs deep reasoning in a fixed-size workspace.

✓ Check yourself — where exactly is the quadratic-in-pixels cost avoided?

At step 2. The queries are 512 latents rather than 50,176 pixels, so the input score table is \(512×50{,}176\), not \(50{,}176×50{,}176\). Later self-attention is quadratic only in 512 latent rows.

7.What worked, what it costs, and where it is fragile

The lecture's empirical message is not “Perceiver beats every specialist everywhere.” It is stronger in a different way: one attention-based backbone can be competitive across very different modalities with far fewer hand-built assumptions.

7.1 Results across modalities

  • ImageNet from pixels. Perceiver reached competitive accuracy without 2D convolutions, and the architecture is insensitive to row order when positions remain attached.
  • Audio and video. Multimodal arrays can be concatenated after padding/tagging modality features, so audio and video become one large input table.
  • Point clouds and 3D data. The set-like interface is a natural fit when no regular image grid exists.
  • Language bytes. Perceiver IO can work without tokenization by modeling bytes/characters and querying masked positions.
  • Optical flow. Perceiver IO produced strong dense predictions by decoding one output per pixel, competing with heavily engineered vision systems.

7.2 Limitations to keep in your mental model

Bottleneck is a feature and a risk

The latent array makes compute tractable by compressing the input. If \(M\) is too small, fine details can be lost. Perceiver IO's dense decoder helps recover structured outputs, but it can only query information that survived in the latents.

  • Position features matter. Fourier frequency range, density, concatenation vs addition, and learned alternatives can change performance substantially.
  • Generality trades off with domain speed. Convnet or hybrid vision models may be faster or better when the right inductive bias is known.
  • Repeated cross-attention is heavy. The first Perceiver found iterative input re-attention useful, but the lecture notes that repeated cross-attends are expensive and not always recommended as best practice.
  • Small-data settings are hard. If the model must learn an inductive bias from limited data, specialized structure can still be valuable.
✓ Check yourself — why might a specialist model still win?

If the domain structure is known and valuable — for example, local translation structure in images — a convnet or hybrid transformer can exploit that bias efficiently. Perceiver is attractive when inputs are huge, heterogeneous, or poorly matched to known architectural biases.

Glossary

TermPlain meaning
Latent arrayA fixed-size learned set of hidden vectors that reads and stores information from the large input.
Input element \(N\)One row of the input array: a token, pixel, audio frame, point-cloud point, byte, or modality feature.
Latent count \(M\)The number of latent vectors. It is chosen by the model designer and is much smaller than \(N\).
Cross-attentionAttention where queries come from one array and keys/values from another.
Latent transformerA standard self-attention transformer stack applied to the latent array, so its quadratic cost is \(M^2\).
Fourier position featuresSine and cosine features of coordinates that give attention access to spatial or temporal location.
Output queryA vector describing one requested output item in Perceiver IO, such as a class summary or one pixel's flow.
Dense predictionA task that returns many structured outputs, often one per input location, such as segmentation or optical flow.
Latent bottleneckThe compression from a huge input array to a smaller latent array; it saves compute but can discard detail.
