Self-Attention Between Datapoints: Non-Parametric Transformers (NPTs)

NPTs ask a transformer to predict a masked table entry by looking not only across features, but across the other datapoints in the dataset.

🎓 Jannik Kossen & Neil Band, introduced by Aidan Gomez ⏱ ~50 min ▶ Watch
Why this matters

Most deep models are like students who study a training set, compress it into weights \(\theta\), close the book, and answer each test question alone. That is convenient, but it throws away a powerful clue: at prediction time, nearby training examples, other test examples, and missing-feature patterns may still be sitting right there. Non-Parametric Transformers challenge that habit: feed the dataset itself to the model, and let datapoints look at each other before predicting.

Running analogy — a study hall, not solo exams

A parametric model takes a solo exam using only what it memorized in its notebook. An NPT sits the whole class in a study hall: each student is a datapoint, each subject column is an attribute, and attention lets students compare notes. The model still has learned rules, but the answer for one student can depend on what similar students wrote down.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh. Each is one line; skim, and dive into any that feel shaky.

  • Self-attention and Q/K/V — each item makes a query, compares it to keys, softmaxes scores, and averages values; here the "items" can be datapoints or attributes.
  • Parametric vs non-parametric prediction — an MLP predicts from learned weights alone; kNN or Gaussian processes explicitly use stored data at prediction time.
  • BERT-style masking — hide selected entries, ask the model to reconstruct them, and compute loss only where the answer was hidden.
  • Tabular data — rows are examples and columns are semantically named attributes such as age, income, pixel, category, or target.

1.Why parametric prediction is not the only game

Imagine predicting a house price. A standard neural network can use the house's square footage, neighborhood, and age, plus whatever it encoded in \(\theta\) during training. But if three nearly identical houses in the current dataset sold yesterday, a human would look at them. NPTs give the model that option.

1.1 The frozen-cookbook problem

In the lecture's framing, most supervised deep learning does this:

  1. Collect training pairs \((x_i,y_i)\).
  2. Optimize parameters \(\theta\) on those pairs.
  3. At test time, predict one new input \(x_*\) from \(\theta\) and \(x_*\), independent of the rest of the data.
Key equation — parametric vs dataset-conditioned prediction

$$ \text{parametric:}\quad p(y_*\mid x_*,\theta) $$

$$ \text{NPT-style:}\quad p(y_*\mid x_*,\mathcal{D}_{\text{context}},\theta) $$

\(\mathcal{D}_{\text{context}}\) is the available batch or whole dataset, including observed features and sometimes observed training targets. \(\theta\) still matters, but it no longer has to be the only memory.

Common misconception

"Non-parametric" means "no learned parameters." Not here. NPTs have transformer weights. The non-parametric part is that predictions can explicitly depend on other datapoints supplied at prediction time, like a learned version of nearest-neighbor reasoning.

✓ Check yourself — if the test batch changes, can an NPT's prediction for the same row change?

Yes. Because the row attends to other rows, the context set is part of the input. A parametric MLP's prediction for one row would not change merely because a different neighboring test example was present; an NPT's can.

2.Data as a matrix, not isolated rows

The primitive object in NPT is not a single feature vector. It is a table: \(n\) datapoints by \(d\) attributes. Some entries are observed; some are masked because they are targets to predict or features we deliberately hide during training.

2.1 Entries, masks, and embeddings

Let \(X\in\mathbb{R}^{n\times d}\) be the dataset matrix. NPT also receives a binary mask \(M\in\{0,1\}^{n\times d}\), where \(M_{ij}=1\) says "the entry \(X_{ij}\) is hidden and should be predicted." Each cell is embedded into an \(e\)-dimensional vector, so the working representation has shape \(n\times d\times e\).

Key equation — cellwise embedding

$$ H^{(0)}_{i,j} = \operatorname{Embed}_j\!\left(X_{i,j}, M_{i,j}, \operatorname{type}(j)\right)\in\mathbb{R}^{e} $$

\(i\) indexes rows/datapoints, \(j\) indexes attributes/columns, and \(\operatorname{Embed}_j\) may be different per attribute because "age" and "diagnosis" should not be exchangeable.

Why the mask is part of the input

In the study-hall analogy, a blank answer is not the same as a zero. The mask tells the class which boxes are blank because we are asking for help, so attention can route evidence toward those blanks.

✓ Check yourself — why learn different embeddings for different columns?

Because tabular columns have different semantics. A value of 1 in "is smoker" does not mean the same thing as a value of 1 in "number of rooms." Column-specific embeddings let the model treat attributes as named fields, not anonymous positions.

3.Attention between datapoints and attention between attributes

The architectural trick is to alternate two views of the same tensor. First, let rows compare complete datapoint representations. Then reshape and let columns inside each row compare feature representations. Rows talk; columns talk; repeat.

3.1 ABD: rows talk to rows

Attention Between Datapoints (ABD) treats each row as one token by flattening its \(d\) attribute embeddings into a \(d e\)-dimensional row vector. Self-attention then runs across the \(n\) rows.

Key equation — ABD shapes

$$ H\in\mathbb{R}^{n\times d\times e} \quad\xrightarrow{\operatorname{reshape}}\quad R\in\mathbb{R}^{n\times (de)} $$

$$ \operatorname{ABD}(R)=\operatorname{softmax}\!\left(\frac{Q_RK_R^\top}{\sqrt{h}}\right)V_R, \qquad Q_RK_R^\top\in\mathbb{R}^{n\times n}. $$

The \(n\times n\) attention matrix is the learned "who should I listen to?" graph over datapoints. \(h\) is the head dimension for one attention head.

3.2 ABA: columns talk inside each row

Attention Between Attributes (ABA) reshapes back to \(n\times d\times e\). For each datapoint \(i\), the \(d\) attribute tokens attend to one another. This lets "age" interact with "blood pressure," or "card value" interact with "suit" in the poker-hand example from the discussion.

Key equation — ABA shapes

$$ H_i\in\mathbb{R}^{d\times e},\qquad \operatorname{ABA}(H_i)=\operatorname{softmax}\!\left(\frac{Q_iK_i^\top}{\sqrt{h}}\right)V_i, \qquad Q_iK_i^\top\in\mathbb{R}^{d\times d}. $$

ABD exchanges evidence between students; ABA lets each student organize their own subjects before the next round of comparison.

✓ Check yourself — which axis gets the quadratic cost in ABD?

The row axis. ABD builds an \(n\times n\) score matrix, so its main attention memory/time grows like \(O(n^2)\) with the number of datapoints in the input batch.

4.The masked training objective makes rows genuinely useful

If a model always saw all labels, it could cheat. If it never saw any labels from other rows, it would collapse toward ordinary parametric prediction. NPT uses stochastic masking to land between those extremes: hide some entries, reveal others, and train only on what was hidden.

Key equation — masked negative log-likelihood

$$ \mathcal{L}(\theta)= \lambda_y\sum_{(i,j)\in\Omega_y} -\log p_\theta\!\left(X_{i,j}\mid X_{\neg\Omega},M\right) + \lambda_x\sum_{(i,j)\in\Omega_x} -\log p_\theta\!\left(X_{i,j}\mid X_{\neg\Omega},M\right) $$

\(\Omega_y\) are masked target entries, \(\Omega_x\) are masked feature entries, and \(X_{\neg\Omega}\) means all unmasked entries. The model is never scored on a target value that was visible as input.

The lecture emphasizes two roles:

  • Feature masking adds BERT-like self-supervision and regularization: the model learns useful structure by reconstructing hidden feature values.
  • Target masking lets some training labels be visible while other labels are hidden. This lets NPT learn functions like "find rows with matching features, then use their labels" without memorizing every input-output pair in \(\theta\).
Common misconception

"Visible training targets mean label leakage." The leakage problem would be scoring the model on a label it could directly see. NPT avoids that: visible labels can be context, but the loss is computed on masked labels. At test time, test targets are masked.

✓ Check yourself — what forces the model to use other datapoints?

Masked target prediction can reveal some rows' targets and hide others. To reduce loss on hidden targets, the model benefits from learning how observed rows relate to masked rows. The corruption experiments in the lecture then test whether performance drops when those row relationships are destroyed.

5.Worked numeric micro-example — one ABD row, by hand

Use the smallest honest table: \(n=3\) datapoints and \(d=2\) attributes. Think of attribute 1 as a feature and attribute 2 as a target. Datapoint 3 has its target masked, and ABD lets it listen to datapoints 1 and 2.

DatapointFeature \(x\)Target \(y\)Status
1010observed
2120observed
30.8?masked target

Suppose one ABD head produces these raw compatibility scores over datapoints. Row 3 is the row we care about:

Step 1 — raw ABD scores

$$ S=QK^\top= \begin{bmatrix} 2.0 & 0.5 & 0.3\\ 0.4 & 2.1 & 1.0\\ 0.0 & 1.0 & -2.0 \end{bmatrix} $$

The third row says datapoint 3 scores datapoint 1 by \(0\), datapoint 2 by \(1\), and itself by \(-2\). A real NPT learns these scores from embedded rows; here we choose round numbers.

Step 2 — softmax the third row

$$ \operatorname{softmax}([0,1,-2]) =\frac{[e^0,e^1,e^{-2}]}{e^0+e^1+e^{-2}} =\frac{[1,2.718,0.135]}{3.853} \approx [0.260,0.705,0.035]. $$

Datapoint 3 mostly listens to datapoint 2, somewhat to datapoint 1, and barely to itself.

Step 3 — weighted blend of label evidence

If this head's value channel carried target evidence \(v=[10,20,0]\), then datapoint 3 receives

$$ 0.260\cdot 10 + 0.705\cdot 20 + 0.035\cdot 0 =2.60+14.10+0 =16.70. $$

That is the learned-nearest-neighbor flavor: the masked row's prediction can be a learned weighted blend of other rows, then refined by later ABD/ABA layers and output heads.

✓ Check yourself — what changes if datapoint 2's target is also masked?

Its label value can no longer be used as visible evidence. In this toy value channel, replace \(v_2=20\) with a mask embedding rather than the true label. The same attention weight \(0.705\) may still point to datapoint 2 because its feature is similar, but it cannot copy the hidden label directly; the model must infer from other observed entries or learned parametric structure.

6.End-to-end trace — follow one concrete masked target

Trace datapoint 3's missing target through the whole NPT pipeline. The important thing to track is both the shape and the axis on which attention runs.

What the trace reveals

NPT is not "a transformer on a long list of cells." It preserves two axes and alternates who communicates: whole datapoints compare globally, then each datapoint reorganizes its own attributes locally.

✓ Check yourself — where would a feature interaction like "age matters only with high blood pressure" be modeled?

Primarily in ABA, because it attends across attributes within one row. ABD can then share that enriched row representation with similar datapoints.

7.Why it matters, when it helps, and where it hurts

The lecture's empirical emphasis is tabular data: a domain where tree ensembles like XGBoost, CatBoost, and LightGBM are hard to beat. NPTs performed strongly across the tabular benchmarks in the paper, and the authors used corruption experiments to show that, on many datasets, performance drops when cross-row information is destroyed.

7.1 What NPTs can learn beyond vanilla kNN

A fixed kNN model chooses neighbors with a hand-designed distance. NPT learns the representation, the attention weights, and the value transformation end-to-end. In the lecture's semi-synthetic duplicate-row experiment, NPT learned to find a matching duplicate row and copy its target; when duplicate targets were shifted by +1, NPT could also learn to subtract 1 before predicting.

The practical promise

NPTs are a learned relational predictor for tables: they can behave like a neural nearest-neighbor model when that helps, and like an ordinary parametric model when cross-row evidence is not useful.

7.2 The cost: quadratic row attention

ABD creates an \(n\times n\) attention matrix. If \(n=8{,}000\), that is 64 million pairwise scores per head per layer before values, gradients, and activations. For much larger datasets, the paper and lecture rely on mini-batching or point toward sparse/retrieval-style approximations.

Key equation — where scaling bites

$$ \text{ABD cost per layer} \sim O(n^2 h),\qquad \text{ABA cost per layer} \sim O(n d^2 h). $$

ABD is the bottleneck when many datapoints are present. ABA becomes important when a table has many attributes.

  • Helps most when: labels or features of nearby rows are informative, dataset context at prediction time is available, and relational/lookup behavior is useful.
  • May not help when: a single row already contains almost all predictive information, the batch is too small or unrepresentative, or cross-row attention cost dominates.
  • Engineering limits: memory, batching/subsampling, heterogeneous feature preprocessing, and careful masking to avoid accidental leakage.
✓ Check yourself — why can NPT fall back to parametric behavior?

If other rows are not useful for a dataset, the learned attention can place little useful weight on them, and the model can rely on transformations of the row's own features. The lecture notes this as a feature: NPT is not forced to use non-parametric evidence when it does not improve loss.

Glossary

TermPlain meaning
Parametric predictionPrediction that depends on learned parameters and the current example, but not explicitly on stored training examples at test time.
Non-parametric predictionPrediction that explicitly uses available data points as part of the predictor, as in kNN, Gaussian processes, or NPT context rows.
NPTNon-Parametric Transformer: a transformer architecture that takes a dataset or batch as input and predicts masked entries using attention.
ABDAttention Between Datapoints: self-attention over rows, producing an \(n\times n\) interaction map.
ABAAttention Between Attributes: self-attention over columns within each row, producing \(d\times d\) maps per datapoint.
Mask matrix \(M\)A binary table that marks which entries are hidden and should be reconstructed or predicted.
Stochastic target maskingRandomly hiding some labels during training while leaving others visible as context; loss is computed only on hidden labels.
Permutation equivarianceIf rows are reordered, outputs reorder the same way; the model should not care about arbitrary dataset row order.
Corruption experimentA test from the lecture: shuffle other rows' attributes while preserving one row, to see whether predictions relied on cross-row relationships.
