Transformers in Diffusion Models for Image Generation & Beyond

Diffusion models turn noise into images by many small denoising moves; DiT replaces the old U-Net denoiser with a transformer, making image, video, and other continuous generation scale like the rest of deep learning.

🎓 Sayak Paul ⏱ ~50 min ▶ Watch
Why this matters

For a decade, the best image generators were full of image-specific machinery: convolutions, U-Nets, up/down paths, skip connections, custom blocks. Sayak Paul’s lecture argues that the center of gravity has shifted: if we can express a noisy image as tokens, the same transformer scaling playbook that worked for language and vision classification can now power generation too — from Stable Diffusion 3 to Sora-like video systems.

Running analogy

Think of diffusion as a restoration studio. The forward process slowly covers a clean painting with layers of static. The reverse model is the restorer: at each visit it removes a tiny, context-aware layer of static. A DiT is a restorer who does not look through a convolutional stencil; it cuts the canvas into patches, lets every patch talk to every other patch, and uses the prompt plus timestep as the work order.

Prerequisites — 30-second refresher

You will follow this note smoothly if these are fresh. Each is one line; skim, and jump if anything feels shaky.

  • Gaussian noise — random values from a bell curve; diffusion deliberately adds this noise so generation can be learned as denoising.
  • ViT patchify — split an image into fixed-size patches and treat them as tokens; DiT reuses this idea on noisy image latents, building on Note 4 on Vision Transformers.
  • Transformer block — LayerNorm, self-attention, MLP, and residual connections; DiT keeps this familiar backbone.
  • VAE / latent space — an encoder compresses pixels into a smaller continuous grid and a decoder expands it back; Stable Diffusion does denoising in this cheaper latent canvas.
  • LayerNorm — normalizes each token’s features; adaptive LayerNorm lets timestep/text/class conditions scale and shift those features.

1.Diffusion in 60 seconds: generation as repeated repair

Start with one real image of a cat. If we add a little noise, it is still recognizably a cat. Add more noise, and only a fuzzy silhouette remains. Add enough, and the image becomes pure static. Diffusion training uses that artificial destruction process to create unlimited supervised examples: “Here is a corrupted image at timestep \(t\); predict the noise that corrupted it.”

1.1 Forward noising: deliberately destroy the sample

The forward process is fixed, not learned. It gradually mixes the clean data \(x_0\) with Gaussian noise \(\epsilon \sim \mathcal{N}(0,I)\). A schedule chooses how much clean signal remains at timestep \(t\).

Key equation — closed-form noising

$$x_t=\sqrt{\bar\alpha_t}\,x_0+\sqrt{1-\bar\alpha_t}\,\epsilon$$

Here \(x_0\) is the clean image or latent, \(x_t\) is the noisy version at timestep \(t\), \(\epsilon\) is sampled Gaussian noise, and \(\bar\alpha_t\) is the remaining signal power. When \(\bar\alpha_t\) is high, the painting is still visible; when it is low, static dominates.

1.2 Reverse denoising: learn the repair move

The neural net learns the reverse move. During training we sample a clean example, a timestep, and a noise vector, form \(x_t\), and ask the network \(\epsilon_\theta\) to predict the noise that was added. At inference, we start from pure noise and repeat that prediction/repair step until a sample appears.

Key equation — epsilon prediction objective

$$\mathcal{L}=\mathbb{E}_{x_0,\epsilon,t}\big[\lVert \epsilon - \epsilon_\theta(x_t, t)\rVert^2\big]$$

The target is not “draw a cat in one shot.” The target is smaller: given the current static-covered canvas \(x_t\) and timestep \(t\), predict the noise \(\epsilon\) so the sampler can step toward the clean data manifold.

Why this works

One enormous generation problem becomes many local correction problems. The restorer does not need to imagine the whole painting from scratch; each pass asks, “what noise should I remove from this partially restored canvas?”

✓ Check yourself — why does the model need the timestep?

The same-looking amount of pixel variation can mean different things at different noise levels. The timestep tells the model where it is on the destruction/repair trajectory: early steps remove coarse static from near-noise, late steps make smaller high-frequency corrections.

2.Latent diffusion: restore a compressed canvas first

Running the full denoising loop directly on pixels is expensive. Stable Diffusion’s key engineering move is to compress the image into a smaller continuous latent grid with a VAE, denoise there, and decode only at the end.

2.1 Why the latent canvas matters

If an image is \(512\times512\times3\), the pixel canvas has hundreds of thousands of values. A latent might be \(64\times64\times4\): still spatial, still continuous, but far cheaper. Sayak emphasized that modern state-of-the-art systems are usually latent-space systems because pixel-space diffusion is computationally prohibitive.

Common misconception

“Latent diffusion ignores pixels.” Not quite. The VAE still learns a pixel-to-latent-to-pixel representation; the diffusion model simply performs the expensive iterative part in a smaller coordinate system. The restorer works on a thumbnail blueprint, then the decoder paints the final full-resolution canvas.

✓ Check yourself — what has to have the same shape in a standard epsilon-prediction denoiser?

The denoiser’s input noisy latent and output predicted noise usually have the same latent shape, because the model predicts one noise value for each latent value it wants the sampler to correct.

3.Diffusion Transformer: replace the U-Net restorer

Early diffusion systems used large U-Nets: convolutional downsampling blocks, middle blocks, upsampling blocks, skip connections, and inserted attention modules. They worked, but they were visually and conceptually heavy. DiT asks a cleaner question: if a noisy latent is a grid, why not patchify it into tokens and run a standard transformer?

3.1 Patchify noisy latents, like ViT

DiT takes a noisy latent \(x_t\in\mathbb{R}^{H_l\times W_l\times C}\), cuts it into \(P\times P\) patches, linearly projects each patch to hidden width \(d\), adds positional embeddings, and processes the resulting token sequence. The number of tokens is

Key equation — latent patches become tokens

$$N=\frac{H_l}{P}\cdot\frac{W_l}{P},\qquad X\in\mathbb{R}^{N\times d}$$

\(H_l,W_l\) are latent height and width, \(P\) is patch size, \(N\) is token count, and \(d\) is transformer width. This is the same patchify move as ViT, but the tokens represent a noisy generative state rather than a clean classification image.

3.2 adaLN: inject timestep and conditioning without a giant detour

The DiT must know both where it is in the denoising trajectory and what it is supposed to generate. For class-conditional DiT, the timestep embedding and class embedding are summed into a conditioning vector \(c\). Instead of cross-attending to that small vector, DiT uses adaptive LayerNorm: \(c\) produces scale, shift, and sometimes gate parameters inside each transformer block.

Key equation — adaptive LayerNorm modulation

$$\operatorname{adaLN}(h;c)=\gamma(c)\odot \operatorname{LN}(h)+\beta(c)$$

\(h\) is a token feature vector, \(\operatorname{LN}\) is ordinary LayerNorm, and \(\gamma(c),\beta(c)\) are condition-dependent scale and shift vectors. In the restoration analogy, the work order does not become another patch; it changes how every restoration step is normalized and applied.

3.3 Scaling behavior: more DiT compute buys better samples

The original DiT result was important not only because it was elegant, but because it scaled: as model size and Gflops increased, FID decreased and images improved. That is the strategic reason transformers matter here. The field gets to reuse a well-understood scaling machine instead of hand-designing ever-larger U-Nets.

Scaling knobObserved effectLearning meaning
More blocks / widthMore Gflops per denoising callMore capacity for global patch interactions.
Better denoiserLower FID in the DiT scaling curvesGenerated images move closer to the real-image feature distribution.
Transformer backbonePredictable scaling trendThe image generator can benefit from the same architecture improvements as other transformer systems.
Compute warning — attention sees all patches

$$\text{self-attention cost}\;\approx\;O(N^2d)$$

This quadratic token cost is the price of global patch interaction. It is manageable for compressed latents and moderate resolutions, but becomes painful for 4K images or video spacetime tokens.

✓ Check yourself — why is DiT more than “ViT copied into diffusion”?

It reuses ViT patch tokens and transformer blocks, but it must also predict noise, condition on timestep, preserve the input/output latent shape, and inject class/text guidance through mechanisms such as adaLN or cross-/joint-attention.

4.Conditioning & guidance: telling the restorer what to reveal

An unconditional denoiser learns “make this look like data.” A useful generator needs “make this look like this prompt.” Sayak’s lecture moved from simple class conditioning to text encoders, PixArt-alpha, Stable Diffusion 3’s MMDiT, and later simplifications.

4.1 Text conditioning: cross-attention, MMDiT, and multiple text encoders

For natural language, a class embedding is not enough. PixArt-alpha uses a text encoder and cross-attention so noisy latent tokens can attend to text tokens. Stable Diffusion 3 uses a multimodal DiT variant: image and text tokens get separate projections, then co-evolve in a joint attention space. The lecture’s practical point: diverse text encoders such as CLIP and T5 can give the denoiser complementary prompt representations, which helps prompt following and text rendering.

Key equation — classifier-free guidance

$$\hat\epsilon=\epsilon_\theta(x_t,t,\varnothing)+w\big(\epsilon_\theta(x_t,t,c)-\epsilon_\theta(x_t,t,\varnothing)\big)$$

\(\epsilon_\theta(x_t,t,c)\) is the conditional noise prediction, \(\epsilon_\theta(x_t,t,\varnothing)\) is the unconditional prediction, and \(w\) is guidance strength. Larger \(w\) pushes harder toward the prompt, but too much can reduce realism or diversity.

Common misconception

“More guidance always means a better prompt match.” Guidance is a steering knob, not truth serum. Past a point, the restorer over-obeys the work order and can create oversaturated, brittle, or less diverse images.

✓ Check yourself — why did Sayak say multiple text encoders may help?

They can supply diverse representations of the same prompt: CLIP contributes contrastive image-text alignment, while T5 or an LLM contributes richer language structure and longer prompt handling. The lecture notes that this is a strong empirical belief rather than a fully settled systematic law.

5.Beyond images: spacetime patches, controls, and limits

Once generation is “tokens plus denoising,” the recipe generalizes. Audio can be represented as continuous latent tokens. Video can be represented as spacetime patches. Structural controls such as pose, depth, or segmentation can be added by adapters, ControlNet-style auxiliary networks, or extra input channels in DiT-like backbones.

5.1 Video and other continuous modalities

OpenAI describes Sora as a diffusion transformer trained on spacetime patches: instead of patchifying only height and width, the model tokenizes across time as well. The idea is powerful but costly: video adds a temporal axis, so full attention becomes a 3D dependency problem. Sayak emphasized that factorized shortcuts are not automatically enough when temporal consistency and cinematic realism matter.

Where the frontier is moving

Recent systems explore flow matching, linear attention, MMDiT variants, parameter sharing, stronger text encoders, and hybrid LLM/image architectures. The common thread is simplification under scale: keep the transformer core, then reduce the cost of making enough denoising steps over enough tokens.

Limitations to keep honest

Diffusion is sequential, so sampling can be slow; distillation and flow-matching try to reduce the number of steps. Training and inference are compute-hungry. Metrics such as FID, preference scores, compositionality tests, and human evaluation each see only part of image quality. And interpretability of these generative transformers remains difficult.

✓ Check yourself — why can GANs still matter if diffusion quality is high?

Diffusion usually needs many denoising steps, while GANs generate in one shot. For ultra-real-time or narrow well-curated domains, a GAN or GAN-assisted distillation objective can still be useful despite GAN failure modes such as mode collapse.

6.Worked example — one denoising formula by hand

Numbers make the forward process concrete. Use a single scalar “pixel” or latent coordinate. Let the clean value be \(x_0=1.0\), choose a timestep with \(\bar\alpha_t=0.36\), and sample noise \(\epsilon=0.5\).

Step-by-step with real numbers

First compute the square roots:

$$\sqrt{\bar\alpha_t}=\sqrt{0.36}=0.6,\qquad \sqrt{1-\bar\alpha_t}=\sqrt{0.64}=0.8$$

Now apply the forward-noise formula:

$$x_t=0.6\cdot 1.0+0.8\cdot 0.5=0.6+0.4=1.0$$

At this timestep, the noisy value happens to equal the original scalar, but it is now a mixture: 60% clean-signal coefficient and 80% noise coefficient times the sampled noise. If the model predicts the noise perfectly, \(\epsilon_\theta=0.5\), we can estimate the clean value:

$$\hat{x}_0=\frac{x_t-\sqrt{1-\bar\alpha_t}\,\epsilon_\theta}{\sqrt{\bar\alpha_t}}=\frac{1.0-0.8\cdot0.5}{0.6}=\frac{0.6}{0.6}=1.0$$

If it predicts \(0.45\) instead, the estimate becomes \((1.0-0.8\cdot0.45)/0.6\approx1.067\), so noise-prediction error turns into reconstruction error.

✓ Check yourself — what does \(\bar\alpha_t\to 0\) mean?

The clean-signal coefficient \(\sqrt{\bar\alpha_t}\) goes to 0, while the noise coefficient \(\sqrt{1-\bar\alpha_t}\) goes to 1. The state becomes almost pure Gaussian noise: the painting is fully covered by static.

✓ Check yourself — redo the recovery if \(\epsilon_\theta=0.25\)

Use \(\hat{x}_0=(1.0-0.8\cdot0.25)/0.6=(1.0-0.2)/0.6=0.8/0.6\approx1.333\). Under-predicting the noise leaves too much noise in the “clean” estimate.

7.End-to-end trace — one DiT image generation run

Now follow one text-to-image sample through the whole pipeline. The concrete prompt could be “tiny astronaut hatching from an egg on the moon,” one of the playful examples Sayak used to motivate text-to-image generation.

StepWhat happensShape
0 · pure noise latentSample an initial latent from a Gaussian distribution.h×w×c
1 · patchifySplit the noisy latent grid into patches and project each patch to transformer width.N×d
2 · add conditionsAdd positional embeddings; compute timestep embedding and text conditioning from the prompt.N×d + timestep + text cond
3 · DiT blocksSelf-attention mixes latent patches while adaLN, cross-attention, or joint attention injects conditioning.N×d
4 · predict noiseThe final layer unpatchifies a prediction of the noise in the current latent.h×w×c
5 · denoise one stepThe scheduler uses the predicted noise to move from \(x_t\) toward \(x_{t-1}\).h×w×c
6 · repeat T stepsRun the same denoiser sequentially across the sampling schedule.h×w×c
7 · VAE decodeDecode the final latent into pixels.H×W×3 image
What the trace reveals

Iterative denoising trades one hard generation leap for many smaller repair steps. The model repeatedly asks “what static should be removed now?” rather than “what entire image should appear?” That is why the process is slower than one-shot generation but often easier to train to high fidelity.

✓ Check yourself — where does Note 4’s ViT patch idea reappear in this trace?

At Step 1: the noisy latent grid is split into patches and projected to tokens. The difference is the goal: ViT classifies or represents an image, while DiT predicts denoising updates for a generative sampler.

Glossary

TermPlain meaning
Diffusion modelA generative model trained to reverse a gradual noising process, usually by predicting noise at many timesteps.
Forward processThe fixed corruption process that mixes clean data with Gaussian noise.
Reverse processThe learned denoising process that turns noise back into data step by step.
\(\bar\alpha_t\)The schedule term measuring how much clean signal remains at timestep \(t\).
Latent diffusionDenoising in a compressed VAE latent space instead of directly in pixel space.
DiTDiffusion Transformer: a transformer denoiser that patchifies noisy latents and predicts noise.
adaLNAdaptive LayerNorm: condition-dependent scale/shift/gate parameters that steer transformer blocks.
Classifier-free guidanceA sampling trick that combines conditional and unconditional predictions to strengthen prompt adherence.
MMDiTMultimodal DiT: a joint text-image diffusion transformer design used in Stable Diffusion 3.
FIDFréchet Inception Distance, an image-generation metric comparing generated and real image feature distributions.
Flow matchingA related generative training approach that learns a velocity field along a path from noise to data, often enabling straighter sampling trajectories.
