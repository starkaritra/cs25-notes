Transformers that Transform Well Enough to Support Near-Shallow Architectures

Jake Williams argues that the path to shallower language models is not merely deleting transformer blocks: first make token vectors meaningful, then analytically warm-start softmax attention/feedforward layers so a few transformations already do useful work.

🎓 Professor Jake Williams / Drexel University ⏱ ~50 min ▶ Watch
Why this matters

Deep transformers are powerful, but every extra layer is sequential work: attention comparisons, feedforward passes, memory traffic, and gradient bookkeeping. Williams' lecture asks a sharper question than "can we prune depth?" It asks: if each layer starts from a statistically meaningful transformation instead of random parameters, how much depth do we still need?

Running analogy

Think of a deep transformer as a committee that revises a draft through many rounds. A near-shallow architecture is a tiny expert team that begins with a good index card system: the words are already filed sensibly, the first attention pass already knows useful pairwise scores, and the remaining layers polish rather than discover the whole filing system from scratch.

Prerequisites — 30-second refresher

You'll follow the talk smoothly if these are fresh.

  • Transformer depth — a transformer block repeats attention plus feedforward transformations; fewer blocks reduce sequential latency but usually reduce modeling power.
  • Residual stream / iterative refinement — deep transformers repeatedly update token representations; Williams' near-shallow pattern tries to make early representations and layers already informative.
  • Self-attention — tokens compare vectors, softmax turns scores into weights, and weighted values become context features.
  • Softmax logits — if logits are log-probabilities plus a constant, softmax recovers the intended probability distribution.
  • Expressivity vs depth — a shallow network can be useful if its few transformations are well chosen, but it may fail where many rounds of abstraction are genuinely needed.

1.The real question: not fewer layers alone

Imagine replacing a 12-layer transformer with a 2-layer model initialized randomly. That is usually not a principled architecture; it is a handicap. Williams' thesis is different: near-shallow models become plausible only when embeddings, attention weights, and output layers are initialized from language statistics rather than lottery-ticket randomness.

1.1 Standard attention spends depth on learned bases

In a standard transformer, key and query matrices learn a comparison space. The model starts with low-dimensional token vectors, projects them into query/key bases, computes similarities, and lets training discover which basis makes useful attention weights.

Key equation — standard self-attention

$$A=\operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right),\qquad Q=XW_Q,\quad K=XW_K$$

Here \(X\) is the block of token vectors, \(W_Q\) and \(W_K\) are learned query/key projections, \(d_k\) is the comparison dimension, and \(A\) is the attention-weight matrix. Williams emphasizes that this projection is another embedding-like factorization: useful, but hard to initialize deterministically.

1.2 The near-shallow move

The talk studies a stripped alternative: keep the vectors in a known basis, compute their similarities, then use a learned matrix as a feedforward transform from similarity patterns to attention weights. That isolates one question: can an analytically initialized self-attention mechanism work well enough without deep stacks of learned basis changes?

Common misconception

"This is just a tiny transformer." Not quite. The lecture's architecture changes the role of attention: instead of relying on random query/key projections to discover a comparison basis, it asks whether fixed, meaningful vectors plus warm-started softmax layers can carry much of the work.

✓ Check yourself — what is being traded for depth?

Depth is traded for stronger per-layer prior structure: deterministic low-dimensional vectors, co-occurrence-based warm starts, cached similarity comparisons, and multiple explicit context models.

2.Deterministic vectors before learned layers

Williams repeatedly returns to one constraint: if attention is going to compare vectors directly, the vectors had better be good. A random or poorly organized basis makes direct similarity meaningless.

2.1 Discernibility and bit-cipher

The low bar is not semantic elegance; it is discernibility. The most frequent tokens are the ones most likely to confuse the model repeatedly, so the bit-cipher assigns them the most distinguishable low-dimensional binary patterns first. In a 5-bit example, the first five tokens get one-hot-like vectors; later tokens receive two-hot and higher-order combinations, normalized to controlled norms.

Intuition

The filing-cabinet analogy: give the most-used folders the clearest labels. The bit-cipher does not know that "Lamp" and "lamp" are semantically similar; it only makes common items easy to tell apart in a small vector space.

2.2 Co-occurrence embeddings add similarity

The naive bit-cipher deliberately ignores usage similarity. To recover that, Williams backs off to classic distributional statistics: use co-occurrences of bit-cipher vectors to prescribe token vectors that place similarly used tokens nearer to each other. This is the bridge from "distinct folders" to "folders used in similar situations sit on nearby shelves."

Key equation — co-occurrence as outer products

$$F=\sum_{n=1}^{N} y_n x_n^\top$$

\(x_n\) is the input/context vector for example \(n\), \(y_n\) is the target or neighboring vector, and \(F\) accumulates how often input features and output features appear together. The same outer-product idea later warms up softmax layers.

✓ Check yourself — why does bit-cipher alone not know that uppercase and lowercase variants are related?

Because the naive assignment is based on frequency and distinguishability, not on shared contexts. Similarity appears only after co-occurrence statistics are used to reshape or prescribe vectors.

3.Warm-start softmax layers

The central technical lever is an analytic initialization for softmax-activated layers. Instead of sampling weights randomly and waiting for backpropagation, estimate the probabilities the layer should initially express, then set logits that softmax will decode into those probabilities.

3.1 From counts to logits

For a feedforward softmax layer, co-occurrence counts act like a conditional-probability table. The exact derivation in Williams' work includes normalization factors such as the effective context size \(k\), but the heart of the move is simple: normalized co-occurrences become probabilities; logarithms turn probabilities into logits.

Key equation — schematic warm start

$$\widehat{p}_{ij}=\frac{F_{ij}+\epsilon}{\sum_{j'}F_{ij'}+m\epsilon},\qquad W^{(0)}_{ij}\approx \log \widehat{p}_{ij}+c_i$$

\(F_{ij}\) is a co-occurrence count from input feature \(i\) to target feature \(j\), \(m\) is the number of targets, \(\epsilon\) prevents zero logs, and \(c_i\) is any row-wise constant because softmax ignores adding the same constant to all logits in a row.

3.2 Hidden targets for attention

For the output layer, the target is visible: the next token or label. For attention, the target is hidden: which position should receive high weight? Williams' analysis derives a criterion for these hidden attention targets: attention should massage the current vectors into something unsurprising for the layer above. Once those hidden targets are estimated, the same softmax warm-start principle can initialize the attention matrix \(W\), the feedforward/value matrix \(U\), and the output matrix \(O\).

Common misconception

"Warm start means pretraining on another giant model." In this lecture, warm start means computing an initialization from the target data's own co-occurrence statistics. It is not RLHF, distillation, or downloading a pretrained transformer.

✓ Check yourself — why is non-negativity important in the current derivation?

The initialization uses logarithms of probability-like normalized counts. Negative inputs do not behave like counts or probabilities, and a logarithm of a negative value is not a real-valued logit. Williams notes that other activations or signed features would require new derivations.

4.Context models and cached attention

A standard transformer block context is not the only language context. Williams argues that different contexts carry different information, and a near-shallow model can keep them separate instead of forcing everything through one giant block.

4.1 Block, radial, document context

The lecture distinguishes three useful views: a block context like ordinary transformer chunks, a radial context over the last \(r\) tokens, and a document context like a TF-IDF/Naive Bayes bag of accumulated content. Concatenating them lets the output layer learn which evidence stream matters for a prediction.

Context modelWhat it seesWhy keep it
BlockFixed chunk positions, e.g. tokens 1-10, 11-20Matches transformer-style local structured context.
RadialThe last \(r\) tokens before the predictionCaptures immediate recency, often crucial for next-token prediction.
DocumentThe accumulated document or dialogue so farProvides a changing global state even when the current block repeats.

4.2 Why static vectors change cost

If embeddings are static during a training phase or at inference time, the similarity between token types is fixed. Williams' key systems observation is then straightforward: precompute those vector comparisons once, retrieve them later, and avoid recomputing the same dot products inside attention.

Key equation — expensive comparisons become cached lookups

$$\text{standard score}_{ij}=x_i^\top x_j\quad\leadsto\quad \text{cached score}_{ij}=C[\mathrm{tok}_i,\mathrm{tok}_j]$$

\(x_i\) and \(x_j\) are token vectors. If they are fixed, their dot product can be stored in a table \(C\). The block may still contain many pairs, but the expensive vector arithmetic is replaced by memory reads; for pivot-style prediction, comparisons over a context become linear in the number of tokens scanned.

✓ Check yourself — when does this cache stop being valid?

When the embedding vectors are updated. If backpropagation changes \(x_i\) or \(x_j\), the stored comparison no longer equals the current dot product, so the model must recompute comparisons until vectors are frozen again.

5.What the experiments suggest

The talk is careful about scope. Williams does not claim that these models already beat frontier LLMs. The evidence says something narrower and interesting: non-random near-shallow models can train quickly, fit small data surprisingly well, and open edge-training regimes that deep pretrained systems do not target.

5.1 Training recipe and speed

The recipe described in the lecture has three phases. First, warm-start the bit-cipher embeddings and matrices on a subset of data. Second, freeze vectors and train middle/upper parameters quickly while cached comparisons avoid the normal attention cost. Third, optionally unfreeze embeddings; this slows training and becomes less stable, echoing the usual difficulty of learning embedding layers far from the loss.

On a 100M-token BabyLM-style setup, the warm start saturates quickly on a subset of data, the frozen-vector phase can make several passes quickly, and unfreezing embeddings slows the process back down because comparisons must be recomputed and embedding learning is delicate.

5.2 Edge use case: a trainable lamp

The concrete application is deliberately small: a lamp with a microphone and a switch. Speech is transcribed, the local language model predicts whether the user wants the lamp lit or dark, and switch corrections become new training data. Williams reports that a small model, trained locally from scratch with roughly 20 minutes of interaction data, could begin predicting the lamp action; larger hyperparameters improve performance, but the research goal is feasibility and local adaptation, not a cloud chatbot.

Practical implication

The exciting regime is not "replace GPT-4 tomorrow." It is private, on-device, target-data training: a model that starts with no personal corpus, learns only from your interaction stream, and can keep improving without shipping raw data to a server.

✓ Check yourself — why is the lamp example a good stress test for near-shallow models?

It demands local training, tiny compute, low latency, continual correction, and useful behavior from little data. Those are exactly the conditions where analytic initialization and shallow computation might matter most.

6.Worked example — one warm-started softmax by hand

Numbers make the core idea click. We compute the smallest honest version of Williams' warm-start claim: co-occurrence counts can initialize a softmax layer closer to the right prediction than random logits.

Step-by-step with real numbers

Suppose a tiny lamp dataset has one context feature, "please", followed by two possible labels: lit or dark. Counts are:

$$F_{\text{please, lit}}=8,\qquad F_{\text{please, dark}}=2$$

Normalize the row into empirical probabilities:

$$\widehat p(\text{lit}\mid\text{please})=\frac{8}{8+2}=0.8,\qquad \widehat p(\text{dark}\mid\text{please})=\frac{2}{10}=0.2$$

Turn probabilities into logits with a logarithm:

$$\ell_{\text{lit}}=\log(0.8)=-0.223,\\ \ell_{\text{dark}}=\log(0.2)=-1.609$$

Now softmax decodes those logits:

$$\operatorname{softmax}(\ell)_{\text{lit}}=\frac{e^{-0.223}}{e^{-0.223}+e^{-1.609}}=\frac{0.8}{0.8+0.2}=0.8$$

A cold zero-logit start would predict \((0.5,0.5)\). For a true lit example, the one-token perplexity drops from \(1/0.5=2.0\) to \(1/0.8=1.25\) before any gradient step.

✓ Check yourself — redo the last step if counts changed to lit=9, dark=1

The empirical probabilities become \((0.9,0.1)\), logits can be \((\log 0.9,\log 0.1)=(-0.105,-2.303)\), softmax returns \((0.9,0.1)\), and true-lit perplexity becomes \(1/0.9\approx1.11\).

What the trace reveals

The near-shallow design saves depth by moving work earlier: deterministic vectors make comparisons meaningful, analytic warm starts make softmax layers initially competent, and freezing vectors lets attention reuse precomputed comparisons.

Glossary

TermPlain meaning
Near-shallow architectureA neural architecture with only a few effective layers, made viable by strong initialization and carefully chosen transformations.
Bit-cipherWilliams' deterministic scheme for assigning low-dimensional binary-like token vectors so frequent tokens are highly distinguishable.
Discernibility hypothesisThe idea that low-dimensional token vectors mainly need to make important, frequent features easy to tell apart.
Warm startAn initialization computed from data statistics, here co-occurrence counts and softmax structure, rather than random weights.
Cold startA random or uninformed initialization; for a vocabulary-sized softmax, initial perplexity is often near the vocabulary size.
Hidden attention targetsImplicit target attention distributions estimated so the layer above receives unsurprising, useful features.
Block contextA fixed-position transformer-style context window.
Radial contextA context formed from the last \(r\) tokens before a prediction.
Document contextAn accumulated bag-like representation of the document or dialogue so far.
Vector comparison cacheA table of precomputed similarities between static token vectors, used to avoid repeated dot-product arithmetic.
