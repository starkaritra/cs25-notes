Generalist Agents in Open-Ended Worlds

Jim Fan's lecture asks how agents can stop being one-trick optimizers and become lifelong learners that use internet-scale knowledge, learned rewards, language plans, code skills, and video pretraining inside open worlds like Minecraft.

🎓 Jim Fan / NVIDIA ⏱ ~50 min ▶ Watch
Why this matters

AlphaGo, AlphaStar, and OpenAI Five were historic, but each lived inside a narrow arena with one scoreboard. Real intelligence looks more like a child in a world: invent goals, try tools, reuse skills, ask what is worth doing next, and recover from mistakes. This lecture is about open-ended agency, where the agent keeps expanding its own competence instead of maximizing one fixed reward forever.

Running analogy

Think of the agent as an adventurer in an enormous sandbox game. MineDojo is the world plus the library of guidebooks and videos; MineCLIP is the quest judge that says whether a clip looks like the quest text; Voyager is the adventurer who writes reusable spells into a notebook; VPT is apprenticeship by watching thousands of human playthroughs.

Prerequisites — 30-second refresher

Skim these before entering the open world.

  • RL agent / reward — an agent chooses actions to maximize accumulated reward; here the hard part is that open-ended goals rarely have clean hand-written rewards.
  • Contrastive vision-language model (CLIP) — train matching images/videos and text to have high similarity and mismatches low similarity; MineCLIP turns that similarity into a reward.
  • LLM code generation — a model such as GPT-4 can emit executable code; Voyager uses this as its action interface through Mineflayer JavaScript APIs.
  • Foundation model — a large pretrained model that transfers across prompts/tasks; the lecture uses text, video, multimodal, and reward-model versions of this idea.
  • Curriculum / self-play — learning improves when tasks are sequenced from reachable to harder; Voyager asks an LLM to propose its own next quest.

1.The open-world problem

Jim Fan opens with Held and Hein's 1963 kitten experiment: one kitten actively moved through the world; the other saw nearly the same visual stream but was passively carried. Only the active kitten developed a healthy visuomotor loop. The point is not that text models are useless. It is that passive observation alone leaves knowledge ungrounded.

1.1 From one scoreboard to endless quests

Classic game agents solved magnificent but closed objectives: win Go, win Dota, win StarCraft. A generalist agent needs a different contract: receive semantically rich language goals, draw on broad prior knowledge, and multitask across an effectively unbounded task space.

Key equation — fixed-task RL is too small a contract

$$J(\pi; g)=\mathbb{E}_{\pi}\left[\sum_{t=0}^{T}\gamma^t r_g(s_t,a_t)\right]$$

Here \(\pi\) is the policy, \(g\) is the goal, \(r_g\) is the reward for that goal, and \(\gamma\) discounts future reward. The open-ended setting asks: who writes \(g\), who supplies \(r_g\), and how does \(\pi\) keep improving across many goals?

1.2 Why Minecraft is the lecture's low-fi Earth

Minecraft has no mandatory plot, no single opponent, and no one winning condition. Its procedural world lets players mine, craft, farm, fight, build redstone circuits, or recreate Hogwarts block by block. That makes it a useful laboratory for open-ended AI: the environment is rich enough to reward creativity, but cheap enough to run on research clusters.

The three ingredients Fan emphasizes

An open-ended agent needs an environment with many possible goals, internet-scale data as guidebooks and demonstrations, and foundation models that connect language, perception, reward, planning, and action.

✓ Check yourself — why would a perfect Go agent still fail this lecture's test?

Because it is optimized for one board game objective. It does not invent new goals, transfer to unrelated environments, use broad internet knowledge, or accumulate reusable skills across open-ended tasks.

2.MineDojo and MineCLIP: a world plus a learned quest judge

In the adventurer analogy, MineDojo builds the sandbox and stocks the library. It provides a Minecraft API, thousands of benchmark tasks, and a huge multimodal knowledge base harvested from how humans actually play and explain the game.

2.1 MineDojo as environment and internet-scale knowledge base

MineDojo combines a flexible simulator API, thousands of programmatic and creative tasks, and human knowledge: YouTube play videos with transcripts, Minecraft Wiki recipes/diagrams, and Reddit posts where players debug one another's builds.

2.2 MineCLIP: reward without hand-coding every quest

The core reward-design problem is easy to feel. "Shear sheep to obtain wool" can be detected with inventory code. But "build a cozy house" or "make a beautiful underground temple" is not a tidy Boolean. MineCLIP trains a video-text contrastive model on Minecraft YouTube clips and narration, then scores whether the agent's recent video matches the language goal.

Key equation — MineCLIP learned reward

$$s(v,g)=\frac{f_V(v)\cdot f_T(g)}{\lVert f_V(v)\rVert_2\,\lVert f_T(g)\rVert_2},\qquad r_t=\sigma\!\left(\alpha(s(v_t,g)-\tau)\right)$$

\(v_t\) is the agent's recent video window, \(g\) is the text goal, \(f_V\) and \(f_T\) are video/text encoders, \(s\) is cosine similarity, \(\tau\) is a success threshold, and \(r_t\) becomes the reward for an RL algorithm.

Common misconception

MineCLIP does not guarantee human-like understanding. It learns a useful alignment space between short video behavior and text descriptions, but can still be fooled by visual shortcuts or fail on long-horizon plans.

✓ Check yourself — why does MineCLIP help with creative tasks more than a hand-coded reward?

A hand-coded reward needs explicit success rules. Creative tasks have fuzzy semantics: a house can look many ways. A video-text reward can use patterns learned from human examples and language, so the same model can score many open-vocabulary goals without new Python reward engineering each time.

3.Voyager: an LLM-powered lifelong Minecraft agent

MineCLIP helps judge short clips, but it does not plan for hours. Voyager tackles the long-horizon side by turning Minecraft into text and code. Mineflayer exposes nearby blocks, entities, inventory, and actions through JavaScript; GPT-4 proposes goals, writes code, reads errors, and stores successful skills.

3.1 Code skills are the adventurer's notebook

Voyager's crucial move is not only using GPT-4 to write one action script. It saves working JavaScript functions in a growing skill library. When a later task resembles an old one, Voyager retrieves the relevant code instead of relearning from scratch. The library becomes a compositional memory: craft an iron sword can support hunt pig; mining wood can support tool crafting; navigation can support exploration.

Key equation — retrieving a reusable skill

$$i^*=\arg\max_i\operatorname{sim}\big(e(q),e(d_i)\big)$$

\(q\) is a natural-language hint for the current task, \(d_i\) is the summary of skill \(i\), \(e(\cdot)\) embeds text into a vector, and the retrieved code for \(i^*\) becomes in-context help for the next program.

3.2 The Voyager loop

Fan describes three GPT-4-driven components: a curriculum module that proposes the next goal, a coding module that writes JavaScript using Mineflayer, and a critic/self-reflection path that uses environment feedback, interpreter errors, and task-progress critique to refine the code.

✓ Check yourself — what does Voyager gain from storing code rather than only text memories?

Code is executable. A stored skill is not merely advice such as "get wood first"; it is a callable routine that can be composed into later routines and debugged through environment feedback.

4.Learning from video: VPT and the next data frontier

The lecture then zooms out: Minecraft videos are abundant, but passive video lacks action labels. A model can see that a player chopped a tree, but not which keyboard and mouse actions caused each frame transition. That missing action channel is the central obstacle to turning YouTube into embodied pretraining.

4.1 Video PreTraining: GPT for actions

OpenAI's VPT solves the label problem with a two-stage trick. First, collect a smaller high-quality dataset where humans play Minecraft and their actions are recorded. Train an inverse dynamics model to infer actions from consecutive observations. Then use that model to pseudo-label huge amounts of unlabeled online gameplay video and train an imitation policy at scale.

Key equation — pseudo-label actions from video

$$\hat a_t=h_\phi(o_t,o_{t+1}),\qquad \max_\theta\sum_t\log \pi_\theta(\hat a_t\mid o_{\le t})$$

\(h_\phi\) is the inverse dynamics model, \(o_t\) and \(o_{t+1}\) are neighboring video observations, \(\hat a_t\) is the guessed action, and \(\pi_\theta\) is the policy pretrained to imitate those guessed actions.

4.2 Beyond video: R3M, VIP, Eureka, VIMA, RT-2, RoboCat

Fan also surveys adjacent systems. R3M learns reusable visual features from videos. VIP learns similarity-based visual rewards. Eureka applies the Voyager-style code-refinement idea to robotics reward design: GPT-4 writes Python reward functions, RL trains controllers in a GPU simulator, and performance feedback guides the next reward candidates. VIMA uses multimodal prompts of text, images, and video demonstrations for robot manipulation, while RT-2 and RoboCat point toward internet-pretrained, multimodal, cross-embodiment robot policies.

The unifying theme

The field is trying to turn the internet into embodied priors: text tells the agent what humans care about, video shows what humans do, code gives reusable procedures, and multimodal models bind those signals to perception and action.

✓ Check yourself — why is watching video harder than reading text for pretraining an agent?

Text already contains its own token sequence. Video lacks the hidden motor commands that caused the pixels to change. Without action labels or grounding, the learner may imitate the wrong correlation, like Fan's example of a cat copying boxers' head-shaking without understanding the fighting policy.

5.Limits and research frontier

The lecture is optimistic, but not triumphalist. These systems are early active kittens, not human-level inventors. The open-world setting exposes failure modes that benchmark scores can hide.

  • Grounding and hallucination: LLM code may call nonexistent APIs, misunderstand spatial geometry, or propose actions that sound plausible but fail in the simulator.
  • Sim-only brittleness: Minecraft and Isaac Sim are powerful laboratories, but real robots face sensor noise, contact physics, safety constraints, and human environments.
  • Evaluating open-endedness: counting unique items or traveled distance is useful, but it is not the same as measuring creativity, usefulness, or scientific discovery.
  • Compute and data cost: VPT-style video pretraining, Eureka-style RL loops, and large multimodal models all demand substantial data, simulation, and GPU budgets.
Common misconception

"Open-ended" does not mean "unmeasurable". It means no single reward captures the whole space. We still need probes, milestone tasks, diversity metrics, human evaluation, and failure analysis; we just should not pretend one scoreboard is intelligence.

✓ Check yourself — name one metric and one thing it misses for Voyager.

Unique items discovered is a good metric because it tracks exploration and tech-tree progress. It misses aesthetics, robustness, safety, whether skills are understandable, and whether the agent can transfer outside Minecraft.

6.Worked example — MineCLIP learned reward by hand

Suppose the goal text is "chop a tree". The agent produces three short candidate video clips. MineCLIP returns CLIP-style similarity scores:

ClipWhat it looks likeSimilarity to goal
AAgent punches a tree trunk0.8
BAgent swims in a river0.2
CAgent walks near a forest0.5
Step-by-step with real numbers

One simple dense reward is min-max normalization:

$$r_i=\frac{s_i-\min(s)}{\max(s)-\min(s)}$$

Here \(\min(s)=0.2\) and \(\max(s)=0.8\), so:

$$r_A=\frac{0.8-0.2}{0.8-0.2}=1.0$$

$$r_B=\frac{0.2-0.2}{0.8-0.2}=0.0$$

$$r_C=\frac{0.5-0.2}{0.8-0.2}=0.5$$

If we instead want a thresholded success signal at \(\tau=0.6\), then \(\mathbb{1}[s_i\ge \tau]\) gives:

$$[0.8,0.2,0.5]\rightarrow[1,0,0]$$

The RL algorithm can now prefer actions that produce clip A-like behavior without anyone hand-writing "detect tree block, detect axe swing, detect log drop" as a reward.

✓ Check yourself — if clip C's score rose to 0.7, what are its normalized and thresholded rewards?

With the original min and max \(0.2,0.8\), the normalized reward is \((0.7-0.2)/(0.8-0.2)=0.5/0.6\approx0.83\). With threshold \(\tau=0.6\), clip C becomes a success: \(1\).

✓ Check yourself — why does this beat a sparse hand-designed reward for open-ended goals?

A sparse hand-designed reward only fires when a programmer anticipated the exact success condition. MineCLIP gives graded feedback for many language goals, including fuzzy ones, because it learned from human video-text pairs rather than from a separate reward program for each task.

7.End-to-end trace — one Voyager iteration

Now follow one concrete Voyager cycle. The agent is hungry, has a small skill library, and sees a pig nearby. Track the state, not just the components.

StepWhat happensOutput
0 · current state + skill libraryMineflayer textualizes the world: low hunger, nearby pig, inventory includes iron; library contains craftIronSword().state text + skill summaries
1 · LLM proposes next goalCurriculum GPT-4 chooses a reachable, useful goal: get food by hunting the pig rather than farming wheat.goal = "hunt pig for food"
2 · LLM writes skill codeCode-generation GPT-4 retrieves sword-crafting skill and writes a Mineflayer routine to craft/equip sword, approach pig, attack, and collect drops.JavaScript function draft
3 · execute in MinecraftThe bot runs the code in the environment; actions change the inventory and world state.trajectory + logs
4 · environment feedback / errorIf the code has an undefined variable, pathing failure, missing item, or incomplete task, the interpreter and world state report it.error message or progress critique
5 · LLM self-refines codeGPT-4 patches the function using the error and current state, then executes again.revised JavaScript
6 · on success, add skillAfter food is collected, GPT-3.5 summarizes the code, the summary is embedded as a key, and the working program is saved as the value.new skill: huntPig()
Why code skills enable lifelong learning

A code skill library is compositional memory. The adventurer does not merely remember "pigs are food"; it gains an executable routine that future quests can call, adapt, and combine with other routines. That is why Voyager's exploration accelerates instead of resetting at every task.

Glossary

TermPlain meaning
Open-ended worldAn environment with many possible goals and no single final objective, such as Minecraft.
MineDojoA Minecraft-based benchmark, API, and internet-scale knowledge base for training generalist agents.
MineCLIPA video-text contrastive model that scores whether a Minecraft video clip matches a language goal.
Learned rewardA reward produced by a model rather than manually written as program logic for one task.
VoyagerAn LLM-powered Minecraft agent that proposes goals, writes executable code, self-refines, and stores successful skills.
Skill libraryVoyager's persistent store of working code skills, indexed by natural-language summaries for retrieval and reuse.
Inverse dynamics modelA model that predicts which action caused a transition from one observation to the next.
Video PreTraining (VPT)OpenAI's method for pretraining Minecraft policies from large unlabeled gameplay videos by pseudo-labeling actions.
EurekaAn LLM-driven reward-design system where GPT-4 writes and iteratively improves reward code for robot RL.
Multimodal promptingSpecifying a task with mixed text, image, and video tokens, as in VIMA-style robot manipulation.
