Advancing Science and Medicine with Collaborative AI Agents

Vivek Natarajan explains why hard scientific and medical work needs not a single chat response, but a collaborating team of agents that can search, debate, rank, verify, and hand useful uncertainty back to experts.

🎓 Vivek Natarajan · Google DeepMind ⏱ ~50 min ▶ Watch
Why this matters

A doctor diagnosing a patient, or a scientist chasing a disease mechanism, is not doing one next-token prediction. They read papers, form hypotheses, check constraints, ask a colleague, revise, design an experiment, and decide what is worth scarce lab or clinical attention. This lecture's central claim is that multi-agent LLM systems can mimic more of that long expert workflow than a single fast answer can.

Running analogy

Think of a research lab meeting. One person proposes an idea, another pulls papers, a skeptic attacks weak links, a senior scientist ranks what is worth doing, and the PI keeps the goal grounded. Co-scientist-style systems turn that lab meeting into computation: agents play roles, share memory, use tools, and iterate until the proposal is stronger.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh. Each is one line; skim, and dive into any that feel shaky.

  • LLM agent — an LLM wrapped in a loop that can plan, call tools, observe results, and update state; see Note 36 for the agent framing.
  • ReAct / tool use — interleave reasoning traces with actions such as search or code execution; see Note 24.
  • Retrieval — fetch relevant external evidence instead of relying only on parametric memory; see Note 25.
  • Self-critique / verifier — use a separate pass to find unsupported claims, bad assumptions, or missing tests before finalizing an answer.
  • Multi-agent orchestration — coordinate several specialized agents through a shared scratchpad, queue, memory, or controller.

1.The problem: expert workflows are long

The transcript begins with Med-PaLM and Med-PaLM 2, systems tuned for medical QA and summarization, then asks a harder question: could an LLM help generate a new biomedical hypothesis? Early PaLM-era scaffolds could retrieve literature and suggest genes for rare disease work, but hallucination made naive prompting unreliable. The project therefore shifted from “ask one model” to “build a structured thinking engine.”

1.1 System 1 vs System 2

Vivek frames ordinary chat-model answers as mostly System 1: fast, fluent, pattern-matching. Scientific discovery needs more System 2: slow, deliberate, skeptical, and willing to spend time. That is why the lecture repeatedly returns to test-time compute: more agents, more rounds, more retrieval, and more critique can buy better hypotheses when the search space is rich enough.

Key equation — useful progress from extra inference

$$\text{quality}(h_T) \approx f\big(\text{model},\ \text{evidence},\ \text{roles},\ T\big)$$

Here \(h_T\) is the best hypothesis after \(T\) agent rounds. The lecture's empirical bet is not that more compute always helps; it helps when evidence and critique let the system search a large hypothesis space intelligently.

Common misconception

“If the base model is smart enough, roles are unnecessary.” The talk argues the opposite: even strong Gemini models are wrapped in roles because science is partly a coordination problem — generating, checking, ranking, and deciding are different cognitive jobs.

✓ Check yourself — why is AlphaFold both inspiring and insufficient for this lecture's goal?

It is inspiring because it solved a grand protein-structure problem at enormous scale. It is insufficient as the model for “scientific superintelligence” because it is specialized: sequence in, structure out. Co-scientist aims for generality: natural-language goals across many scientific tasks.

2.Collaborative agent architecture

The co-scientist system takes a scientist's natural-language research goal, optional constraints, ranking rubrics, seed directions, PDFs, and experimental data. It then runs a multi-agent computation and returns a research report: hypotheses, rankings, uncertainty, and suggested next validation steps.

2.1 Co-scientist as a while loop

Vivek describes the architecture as a computer program with a while loop. The loop has four recurring functions: generate ideas, review/critique them, rank/prioritize them through debates, and evolve them using feedback stored in memory.

Key equation — debate ranking with Elo intuition

$$P(i \succ j)=\frac{1}{1+10^{(R_j-R_i)/400}}$$

\(R_i\) and \(R_j\) are ratings for hypotheses \(i\) and \(j\). After debate matches, hypotheses that repeatedly win under the scientist's rubric rise in priority. The exact implementation can differ, but the idea is tournament-style ranking of ideas.

✓ Check yourself — what does the ranking agent add beyond generating many good ideas?

Scientists are not short on ideas; they are short on time, attention, and validation budget. Ranking adds prioritization, uncertainty, and epistemic humility so the report can say what is worth reading or testing first.

3.Science: propose → critique → refine

The scientific part of the talk centers on AI co-scientist, a general-purpose multi-agent system for hypothesis generation. Its agents use strategy libraries: read related papers, simulate expert debates, review novelty, rank by constraints, and feed debate summaries back into memory.

3.1 Validation examples from the talk

The lecture's evidence is not a single leaderboard. It is a set of biomedical collaborations where the system proposed or recapitulated hypotheses that scientists could inspect and sometimes test:

  • Antimicrobial resistance — a recapitulation task with Imperial researchers, where the system surfaced a horizontal gene-transfer mechanism that was not yet public.
  • AML drug repurposing — Houston Methodist physician-scientists tested AI-nominated drugs such as KIRA6 and observed promising IC50 curves in cell lines.
  • Liver fibrosis — Gary Peltz's group used the system for epigenomic targets, organoid validation protocols, and candidate drugs such as vorinostat.
  • Protein design — co-scientist used tools such as AlphaFold in the loop to design improved Oct4-like proteins with desired activity and safety properties.
  • Alzheimer's mechanism — the system recapitulated a multi-step ACE-inhibitor cascade and proposed a missing bradykinin → B2R receptor link later tested in a chase assay.
  • Plant immunity — researchers used it to build a structural novelty index over AlphaFold predictions and identify an unusual potato immunoprotein architecture.
Key equation — hypothesis search as constrained optimization

$$h^*=\arg\max_h\ \Big[U(h\mid g,e)-\lambda_1\,\text{risk}(h)-\lambda_2\,\text{cost}(h)\Big]$$

\(g\) is the scientist's goal, \(e\) is retrieved evidence, \(U\) is usefulness or expected scientific value, and the penalties encode safety, feasibility, novelty uncertainty, and validation cost.

✓ Check yourself — why can “go broad” complement a human scientist's “go deep”?

A specialist may know one domain deeply but miss analogies from cancer, plant immunity, protein design, or pharmacology. A retrieval-and-agent system can scan broadly, surface surprising links, and let the human expert judge feasibility.

4.Medicine: diagnostic and safety loops

The medical arc begins with Med-PaLM and Med-PaLM 2, which showed medically tuned LLMs could reach passing and then expert-level performance on USMLE-style benchmarks. It also names AMIE, a Google/DeepMind research program for conversational diagnostic AI. The same collaboration pattern appears in medicine: a single confident answer is dangerous; a safer system should gather evidence, simulate specialist consultation, verify claims, and defer to clinicians.

4.1 Why verification matters

Medical and biomedical agents sit in high-stakes domains. A verifier agent can ask: Is the claim supported by retrieved evidence? Is the proposed experiment safe? Does a diagnostic suggestion need urgent human escalation? The talk's safety answer is layered: check the user goal at the start, monitor generated ideas during the run, halt unsafe explorations, and inherit base-model safety work from Gemini.

Key equation — reliability with a verifier

$$P(\text{unsafe final})=P(\text{unsafe draft})\,P(\text{verifier misses it})$$

This is only a simplification. It assumes the verifier's misses are not perfectly correlated with the solver's mistakes — the worked example below shows why correlation is the enemy.

Common misconception

“Multi-agent means automatically safer.” Not necessarily. If all agents share the same blind spot, cite the same weak evidence, or optimize the wrong reward, they can amplify an error. Collaboration helps when roles and evidence sources create genuinely different failure modes.

✓ Check yourself — name two failure modes that become worse in a multi-agent system.

Coordination overhead can waste compute or hide responsibility. Error propagation can occur when one agent writes a false claim into shared memory and later agents treat it as established evidence.

5.Worked example — one tiny case, by hand

Numbers make the reliability story concrete. Suppose a solo agent answers a medical QA or hypothesis-screening subtask correctly with probability \(p=0.7\). Its error rate is therefore \(1-p=0.3\). Now add an independent critic/verifier that catches a wrong answer with probability \(c=0.6\).

Step-by-step with real numbers

The only errors that reach the final answer are the solo agent's errors that the critic misses:

$$\begin{aligned} P(\text{final error}) &= P(\text{solo wrong})\times P(\text{critic misses wrong})\\ &=0.3\times(1-0.6)\\ &=0.3\times0.4\\ &=0.12. \end{aligned}$$

So the effective accuracy becomes

$$P(\text{final correct})=1-0.12=0.88.$$

The critic improved reliability from \(0.70\) to \(0.88\), but only because it caught mistakes the solver would otherwise ship.

✓ Check yourself — if the critic catches wrong answers with probability 0.8 instead of 0.6, what is the new accuracy?

The final error is \(0.3\times(1-0.8)=0.3\times0.2=0.06\). Accuracy is \(1-0.06=0.94\).

✓ Check yourself — when does adding more agents stop helping?

When errors are highly correlated, when agents share the same bad evidence, when communication overhead dominates, or when the task is already easy enough that extra rounds only add cost. In the extreme, ten agents repeating the same hallucination are not ten independent checks.

6.End-to-end trace — follow one concrete instance

Now trace one biomedical task through a collaborative pipeline: “Find plausible repurposed drugs for acute myeloid leukemia, with evidence and a validation plan.”

StepAgentActionOutput
0 · taskHuman scientistSpecifies disease, constraints, novelty threshold, and assay budget.goal + rubric
1 · decomposePlannerSplits work into literature search, target-pathway mapping, candidate ranking, and validation design.subtasks
2 · retrieveResearcherUses tools to retrieve papers, drug-target databases, and prior AML pathway evidence.evidence bundle
3 · draftSolverProposes candidate drugs and explains mechanistic rationale.candidate hypotheses
4 · verifyCriticFlags weak citations, missing controls, toxicity concerns, and claims already tested clinically.issues + uncertainty
5 · reviseSolverRemoves weak candidates, tightens novelty claims, and adds experiment protocols.ranked report
6 · decideHuman scientistChooses what to validate in cell lines or organoids.experiment plan
What the trace reveals

Role specialization plus critique beats a monolithic answer because each stage leaves an artifact that the next stage can inspect. The researcher grounds the solver, the critic protects the human from overconfident drafts, and the final report preserves both promising ideas and reasons for doubt.

Debate loopA process where agents argue for or against hypotheses so strengths and weaknesses become explicit.
Elo rankingA rating method from games used here as an intuition for ranking hypotheses after pairwise debate matches.
Epistemic humilityThe system's ability to say what it does not know, where evidence is weak, and what must be validated.
