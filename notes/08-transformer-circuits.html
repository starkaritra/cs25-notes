Transformer Circuits, Induction Heads & In-Context Learning

A mechanistic-interpretability view of transformers: treat weights as a compiled program, decompose attention into circuits, then see how a two-head algorithm can copy patterns inside the prompt.

🎓 Chris Olah — Anthropic interpretability ⏱ ~50 min ▶ Watch
Why this matters

Transformers can translate, imitate new formats, and solve few-shot prompts, but their learned algorithms are buried inside billions of numbers. Mechanistic interpretability asks a sharper question than “which neurons activate?”: can we reverse-engineer the actual computation into human-readable circuits?

The lecture starts from a safety and science motivation. If a model has an unanticipated failure mode, behavior-only testing may miss it. Understanding the internal program gives us a second handle: inspect what computation the model is really running.

Running analogy

Treat the transformer as a busy newsroom. The residual stream is the shared newsroom whiteboard. Each attention head reads notes from the board, sends a courier to retrieve a past clue, and writes a new note back. Mechanistic interpretability is the forensic editor reconstructing the newsroom’s workflow from the scribbles and courier routes.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh. Each is one line; skim, and dive into any that feel shaky.

  • Multi-head attention & Q/K/V — each head forms queries, keys, and values; here we split a head into a “where to look” part and a “what to move” part.
  • Causal masking — token position \(t\) can only attend to positions \(\le t\); induction heads copy from earlier context without peeking at the answer.
  • Residual connections — layers add updates to a shared vector stream; this makes “read from / write to the whiteboard” the right mental model.
  • Softmax — exponentiates scores and normalizes them; a small score gap can become a very sharp attention pattern.
  • Logits and unembedding — final residual vectors are projected into vocabulary scores; if a head writes “B-like” information, the B logit rises.

1.Mechanistic interpretability: reverse-engineering the program

Start with the lecture’s concrete image example: a “car neuron” in a vision model can be explained by sub-features such as wheels on the bottom, windows on the top, and chrome in the middle. That is not merely a saliency story; it is a tiny algorithm.

Transformer circuits applies the same ambition to language models. The parameters are like a compiled computer program; activations are like temporary variables; the goal is to decompile the program into steps a human can inspect.

1.1 Circuits, not vibes

A circuit is a small set of components whose combined weights and activations implement a recognizable computation. The claim is not “every head has a one-word label.” The stronger claim is: some model behavior is carried by concrete computational motifs that can be tested by reading matrices, inspecting attention, and ablating components.

Key equation — behavior from a decomposed computation

For a simplified attention-only transformer, the final vocabulary scores are

$$\text{logits} = x_{\text{resid}} W_U,$$

where \(x_{\text{resid}}\) is the final residual-stream vector at a position and \(W_U\) is the unembedding matrix that maps that vector into one score per token. Interpretability asks: which earlier paths wrote the features in \(x_{\text{resid}}\)?

Common misconception

Misconception: “Interpretability means looking at attention heatmaps.” Heatmaps are useful clues, but not explanations by themselves. A head can attend to a position and move irrelevant information, or another head may have already moved information there. Circuits track both the route and the payload.

✓ Check yourself — what would make an explanation mechanistic?

It should identify concrete components, state what computation they implement, and survive interventions: if you ablate or patch the proposed components, the predicted behavior should change in the predicted way.

2.The residual stream is the central object

Imagine token position 20 in a prompt. It is not passed through a private pipeline isolated from every other layer. Instead, it carries a vector — the residual stream — that every layer can read and update. Attention heads and MLPs add new information to this shared vector.

2.1 The newsroom whiteboard: every layer reads and writes

In the simplified attention-only setup from the talk, each layer is a sum of attention-head updates added to the residual stream. With MLPs and LayerNorm omitted for clarity, layer \(\ell\) looks like:

Key equation — residual stream update

$$x^{(\ell+1)} = x^{(\ell)} + \sum_{h \in \ell} \operatorname{AttnHead}_{\ell,h}\!\left(x^{(\ell)}\right).$$

Here \(x^{(\ell)}\) is the whiteboard state after layer \(\ell\). Each head reads \(x^{(\ell)}\), computes an update, and writes by addition. This linear additivity is why circuit paths can often be separated and inspected.

Why the residual stream matters

If a layer-0 head writes “the previous token was A” into the residual stream at position 2, a layer-1 head can use that fact while computing keys or values at position 2. That is the doorway to multi-step algorithms.

✓ Check yourself — why does additive writing help circuit analysis?

Because separate paths contribute additively to the residual stream. In simplified settings, you can expand the model into terms such as direct bigram paths, single-head paths, and composed multi-head paths, then ask which terms matter.

3.An attention head decomposes into QK and OV circuits

Take a head that attends from the current word “are” back to a previous word “perfect” and raises the probability of “perfect” again. Two questions are mixed together in the usual attention formula: where did it look, and what did it move?

3.1 QK = where, OV = what

Transformer circuits splits a head into two low-rank matrix products:

  • QK circuit: computes attention scores — which source positions should the destination attend to?
  • OV circuit: transforms the attended source information into an update written at the destination — what information is copied or transformed?
Key equation — low-rank head matrices

$$W_{QK}=W_Q W_K^{\top}, \qquad W_{OV}=W_V W_O.$$

\(W_{QK}\) scores query–key compatibility, so it controls the attention pattern. \(W_{OV}\) maps source-token information through values and output weights, so it controls the payload written back to the residual stream. In the vocabulary lens, \(W_E W_{OV} W_U\) tells us how attending to an input token changes output logits.

Common misconception

Misconception: “The value vector is the information, and the output matrix is just formatting.” In this framework, \(W_V\) and \(W_O\) are interpreted together as \(W_{OV}\), one low-rank map from source residual features to destination residual features.

✓ Check yourself — if a head has a copying OV circuit but random QK, what happens?

It knows how to copy useful information, but it does not reliably know where to copy from. The output may be noisy because the “what” circuit is good while the “where” circuit is not targeted.

4.Composition and virtual weights: heads become algorithms

A one-layer attention-only transformer can do crude in-context learning: it can notice a token occurred before and increase related tokens later. The lecture describes these as giant tables of skip-trigram-like effects. But the sharp phase change in in-context learning appears in two-layer attention-only transformers, where heads can compose.

4.1 Q-, K-, and V-composition

Layer-1 heads can modify the residual stream in ways that layer-2 heads use to form queries, keys, or values. This gives three important composition routes:

  • Q-composition: an earlier head writes features later used to make the destination query.
  • K-composition: an earlier head writes features later used to make a source key.
  • V-composition: an earlier head writes features later copied as the source value.
Key equation — composed paths

For two simplified attention layers, expanding the residual stream gives direct paths, single-head paths, and composed paths:

$$\left(I + H_1\right)\left(I + H_0\right) = I + H_0 + H_1 + H_1H_0.$$

The \(H_1H_0\) term is a composed circuit: layer 0 writes something that layer 1 uses. Some expansions behave like virtual attention heads — not literal heads in the architecture, but effective read/write paths created by composition.

Virtual weights in newsroom language

If reporter 0 pins “the previous token was A” to the board, reporter 1 can route couriers by that note. The effective workflow looks like a new hidden courier rule even though the architecture only contains the two original reporters.

✓ Check yourself — why is a two-layer attention-only model more expressive than a one-layer one?

The second layer can compute its attention pattern from representations already modified by the first layer. That allows “write a clue, then use the clue to route attention,” which one layer cannot do in the same direct way.

5.Induction heads: the star circuit

Now the motivating mystery snaps into focus. Given a prompt fragment like ... A B ... A, a useful model should predict B. It should notice: “last time I saw A, B came next; maybe that repeats.” This is in-context copying or pattern completion.

5.1 The two-head “match then copy” algorithm

An induction circuit often uses two heads:

  1. A previous-token head copies each token into the residual stream of the next position. At position \(j+1\), the model now has a feature saying “the previous token was \(x_j\).”
  2. A later induction head at the current \(A\) looks for source positions whose stored previous-token feature is also \(A\). That source is the token after the earlier \(A\), namely \(B\). The OV circuit then copies \(B\)-like information forward, raising the \(B\) logit.
Key equation — induction as nearest-neighbor copying

For current position \(t\), an induction head approximately computes

$$j^* = \arg\max_{j<t}\; \langle q(x_t),\, k(\text{prev}(x_j))\rangle, \qquad \text{then copy } x_j.$$

Because the previous-token head made \(\text{prev}(x_j)=x_{j-1}\) available at position \(j\), matching the current token \(x_t=A\) to \(\text{prev}(x_j)=A\) selects \(j\), the token after the earlier \(A\). That is why the payload is \(B\), not \(A\).

Connection to in-context learning

Olsson et al. found that induction heads emerge during a sharp training phase change that coincides with a drop in in-context-learning loss: late tokens suddenly become much easier to predict than early tokens. Ablation evidence in small models shows these heads account for much of that meta-learning score. In larger models, their emergence aligns tightly with the same phase change, suggesting induction heads are a major mechanism behind few-shot prompting.

Common misconception

Misconception: “Induction heads can only repeat exact text.” Exact copying is the clean toy case. In larger models, “soft” induction heads can copy abstractly: translated equivalents, similar words, or structural labels in made-up functions. The boundary gets blurrier, but the motif remains nearest-neighbor pattern completion inside the context.

✓ Check yourself — in ... A B ... A, why does the induction head attend to B rather than the earlier A?

Because the previous-token head shifted information forward. The key at the earlier B position contains “previous token was A,” so the current A query matches that position. The OV circuit then copies B-like information.

6.Worked example — induction on [A, B, C, A, ?]

Numbers make the mechanism concrete. Suppose the prompt so far is A B C A and we are predicting position 5. The desired continuation is B because the earlier occurrence of A was followed by B.

Step-by-step with real numbers

The previous-token head has already written “previous token” features into the keys of positions 2, 3, and 4:

$$\text{key feature at pos 2}=\text{prev token A},\quad \text{pos 3}=\text{prev token B},\quad \text{pos 4}=\text{prev token C}.$$

At the last observed token, position 4 is A. Let the induction head score candidate source positions 1, 2, and 3 as follows:

$$s=[1.0,\;4.5,\;0.2].$$

Softmax converts these scores into attention weights:

$$\operatorname{softmax}(s)=\frac{[e^{1.0},e^{4.5},e^{0.2}]}{e^{1.0}+e^{4.5}+e^{0.2}} = \frac{[2.72,90.02,1.22]}{93.96} \approx [0.029,0.958,0.013].$$

So the induction head puts about \(95.8\%\) of its attention on position 2, the token after the earlier A. The OV circuit copies B-like information from position 2 into the residual stream at position 4, and the unembedding raises the logit for B.

✓ Check yourself — what does the previous-token head attend to?

At each position \(i\), a previous-token head attends mostly to position \(i-1\). For example, at position 2 (B) it attends to position 1 (A) and writes “previous token was A” into position 2’s residual stream.

✓ Check yourself — redo the softmax if the middle score drops from 4.5 to 2.0

With scores \([1.0,2.0,0.2]\), the exponentials are approximately \([2.72,7.39,1.22]\), total \(11.33\). The weights are \([0.240,0.652,0.108]\). The head still prefers position 2, but much less sharply, so copying B is weaker.

7.End-to-end trace — the induction algorithm as a circuit

Now follow one concrete repeated sequence all the way through. The state column tracks what is on the newsroom whiteboard: the residual-stream features each head reads or writes.

StepWhat happensResidual-stream state / value
0 · inputPrompt contains a repeated prefix: A B C A. The model must predict the next token.tokens = [A, B, C, A]
1 · embeddingsEach token is embedded into the residual stream at its position.x₁≈A, x₂≈B, x₃≈C, x₄≈A
2 · layer-0 previous-token headAt each position, the head attends one step back and writes previous-token information forward.pos2 gets prev=A; pos3 gets prev=B; pos4 gets prev=C
3 · residual stream carries the clueThe source position holding B now also carries “my previous token was A.”pos2 = B + prev(A)
4 · layer-1 induction QKAt current A, the query searches for earlier positions whose key says “previous token was A.”match current A → key at pos2
5 · layer-1 induction OVThe head attends to position 2 and copies its token identity / continuation feature into the current residual stream.write B-like feature at pos4
6 · unembeddingThe final residual stream is projected into vocabulary logits.logit(B) increases
7 · outputThe model predicts the repeated continuation.next token ≈ B
What the trace reveals

The circuit does not memorize a global rule like “A always means B.” It performs an in-context nearest-neighbor lookup: find the earlier local context that matches the present one, then copy what came next in this prompt.

✓ Check yourself — which component performs “match,” and which performs “copy”?

The QK circuit performs the match by producing the attention pattern. The OV circuit performs the copy by moving B-like source information into the current residual stream and ultimately toward the B logit.

Glossary

TermPlain meaning
Mechanistic interpretabilityReverse-engineering a neural network’s internal computation into components and algorithms humans can understand and test.
CircuitA set of heads, neurons, paths, or weights that jointly implement a recognizable computation.
Residual streamThe shared vector state at each token position that layers read from and add updates to.
QK circuitThe part of an attention head that decides which source positions to attend to — the “where” circuit.
OV circuitThe part of an attention head that decides what information is moved from source to destination — the “what” circuit.
CompositionA later head using information written by an earlier head, enabling multi-step computations.
Virtual attention headAn effective path created by composing heads; it behaves like a head in an expansion even though it is not a separate architectural module.
Previous-token headA head that attends one position back and writes the previous token’s identity into the current position.
Induction headA head that finds an earlier matching context and copies the token that followed it, enabling pattern completion.
In-context learningAdapting behavior using examples in the prompt, with no weight updates during inference.
Meta-learning scoreIn the lecture’s framing, a measure such as how much better the model predicts late-context tokens than early-context tokens.
