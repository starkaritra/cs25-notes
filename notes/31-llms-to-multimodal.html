From Large Language Models to Large Multimodal Models

Ming Ding's CS25 lecture explains how the LLM era reached multimodality: keep the language model's reasoning machinery, then give it a visual front end whose outputs look like tokens the LLM can read.

🎓 Ming Ding · Zhipu AI ⏱ ~50 min ▶ Watch
Why this matters

A text-only LLM can write code, summarize papers, and reason over documents, but it cannot directly inspect a chart, read a screenshot, identify a typo in a receipt, or ground an answer in image regions. Large multimodal models (LMMs) attack that missing sense: they extend a pretrained LLM so images, video frames, documents, and eventually audio can become part of the same conversation.

Running analogy

Think of the LLM as an expert translator who only understands one language: text tokens. A vision encoder is the photographer, CLIP is the bilingual dictionary between pictures and captions, and the connector is the interpreter who rewrites visual evidence into the token-dialect the translator already speaks. The whole note reuses this interpreter analogy.

Prerequisites — 30-second refresher

You'll follow this note smoothly if these are fresh.

  • LLM / next-token prediction — a decoder-only model such as GPT predicts the next token from previous tokens; this note cross-links the language-model recipe from Note 3.
  • ViT and patches — a Vision Transformer splits an image into patches, embeds each patch, and processes the patch sequence; refresh with Note 4.
  • CLIP / contrastive learning — train image and text encoders so matching image-caption pairs have high similarity and mismatches have low similarity; this gives vision features a language-shaped geometry.
  • Embedding and cosine similarity — vectors represent tokens or images; cosine similarity \(\frac{x\cdot y}{\|x\|\|y\|}\) measures alignment independent of length.
  • Tokenization — text becomes discrete token IDs; LMMs make image patches behave like additional continuous "tokens" in the LLM's embedding space.

1.The missing sense: sight

Start with a concrete failure. Ask a text-only chatbot: "What is wrong with this circuit diagram?" It can explain circuits in general, but unless the diagram was converted into text by another system, it never saw the evidence. The multimodal move is not to throw away GPT-like LLMs; it is to add a perceptual input channel.

1.1 From GPT to a visual assistant

Ding begins with the LLM arc: BERT/GPT/T5 era, GPT-3 scaling laws, and the ChatGPT lesson that task adaptation is cheap compared with pretraining knowledge. That matters for LMMs because the dominant recipe treats the LLM as the expensive reasoning core and adds vision around it.

Intuition

If the translator already knows how to answer questions, follow instructions, and produce fluent explanations, don't retrain that whole mind from scratch. Hire an interpreter who can turn the photograph into a sequence of facts and feature vectors the translator can use.

1.2 The recipe in one sentence

Core recipe: a frozen or lightly tuned vision encoder, usually a CLIP-style ViT, turns an image into visual features; a connector/projector maps those features into the LLM embedding space; the LLM processes visual tokens and text tokens jointly and emits text.

Key equation — projecting vision into the LLM's language

$$Z_{\text{vis}} = P\big(E_{\text{vision}}(I)\big), \qquad H_0 = [Z_{\text{vis}};\, E_{\text{text}}(x)]$$

Here \(I\) is the image, \(E_{\text{vision}}\) is a ViT/CLIP encoder, \(P\) is the connector, \(x\) is the question text, and \(H_0\) is the initial sequence fed to the LLM. The semicolon means concatenation along the token dimension.

✓ Check yourself — why not feed raw pixels directly into the LLM?

Raw pixels are not in the LLM's learned embedding space and would be a huge sequence. A vision encoder compresses local image structure into patch features; the connector then translates those features into the hidden dimension the LLM expects.

2.CLIP: the visual backbone that makes the bridge possible

Before an LLM can use images, the visual representation needs to be language-adjacent. CLIP supplies that bridge by training on image-text pairs: real captions should be near their images, and other captions in the batch should be far away.

2.1 Contrastive alignment

In the interpreter analogy, CLIP is the dictionary training phase. It does not yet run a chat assistant, but it learns that the picture of a red bus should land near captions about red buses and far from captions about microscopes.

Key equation — contrastive image-text loss

$$\mathcal{L}_{i\to t}=-\log \frac{\exp(s(v_i,t_i)/\tau)}{\sum_j \exp(s(v_i,t_j)/\tau)},\qquad s(v,t)=\frac{v\cdot t}{\|v\|\,\|t\|}$$

\(v_i\) is image embedding \(i\), \(t_i\) is its paired text embedding, \(s\) is cosine similarity, and \(\tau\) is temperature. Smaller \(\tau\) makes the softmax more decisive.

2.2 What a ViT encoder hands over

A CLIP ViT can output a single pooled image embedding, but LMMs often keep patch-level features. For a \(336\times336\) image with patch size \(14\), the image becomes \(24\times24=576\) visual positions. Those 576 features preserve where details live, which helps OCR, grounding, charts, and screenshots.

✓ Check yourself — how many visual tokens does a 336×336 image at patch size 14 yield, and why map them to the LLM dim?

\(336/14=24\) patches per side, so \(24\times24=576\) patch tokens. The projector maps each token from the vision encoder's feature dimension into \(d_{\text{LLM}}\), so the LLM can concatenate visual and text embeddings in one sequence.

3.Connector design choices: three ways to hire the interpreter

The connector is where many famous LMM architectures differ. All solve the same problem — translate visual features into something the LLM can use — but they choose different amounts of compute, trainable parameters, and interaction between modalities.

3.1 Linear / MLP projector: LLaVA

LLaVA made the simple recipe popular: take CLIP patch features and pass them through a learned linear layer or MLP projector. Ding highlights this as the simpler alternative to BLIP-2's Q-Former, and it quickly became a default architecture because it is easy to train and scale.

Key equation — projector connector

$$z_k = W_2\,\sigma(W_1 f_k + b_1)+b_2,\qquad z_k\in\mathbb{R}^{d_{\text{LLM}}}$$

\(f_k\) is vision feature \(k\), \(z_k\) is the projected visual token, \(\sigma\) is a nonlinearity, and \(d_{\text{LLM}}\) is the LLM hidden size. A one-layer projector drops the \(W_2\sigma(\cdot)\) part.

3.2 Q-Former: BLIP-2

BLIP-2 inserts a trainable query transformer between the image encoder and the LLM. A small set of learned query tokens cross-attend to image features and extract a compact visual summary. This reduces the number of visual tokens, but the connector itself is more structured.

3.3 Gated cross-attention: Flamingo

Flamingo keeps a strong language model and inserts gated cross-attention layers so text tokens can attend to visual features at selected points. The gate protects the pretrained language behavior: if visual information is unhelpful, the model can lean back toward its language-only path.

Common misconception

"Multimodal means the LLM learned vision from scratch." Usually not. Many systems import a pretrained vision encoder, align its outputs with the LLM, and fine-tune. The LLM gains a visual interface; the low-level visual features often come from CLIP/ViT-style pretraining.

✓ Check yourself — which connector is simplest to implement?

The LLaVA-style linear/MLP projector is simplest: it directly maps every CLIP visual feature into the LLM hidden dimension. Q-Former and cross-attention introduce additional transformer modules or attention layers.

4.Training and systems: alignment first, then instruction following

Ding's broader theme is that modern model building is often engineering plus data. In LMMs, architecture is only part of the story: the model must first learn that projected visual tokens correspond to language, then learn to follow visual instructions.

4.1 Alignment then visual instruction tuning

  1. Feature alignment pretraining — train the connector, and sometimes parts of the LLM, on image-caption or image-text pairs so visual tokens land in the right semantic neighborhood.
  2. Visual instruction tuning — train on conversational examples: image + question → helpful answer, including VQA, OCR, document QA, captioning, and grounding prompts.
  3. Component freezing decisions — freeze the vision encoder to preserve robust visual features; freeze or LoRA-tune the LLM to preserve language ability; train the connector aggressively because it is the new interpreter.
Intuition

Alignment teaches the interpreter the vocabulary: "this patch feature means dog ear, this one means receipt total." Instruction tuning teaches the full team how to answer questions politely and usefully.

4.2 Models named in the talk

The lecture surveys both understanding and generation. For visual understanding, Ding names BLIP-2, LLaVA, CogVLM, CogAgent, Vary, GLM-4V, and compares progress with systems such as GPT-4V, Gemini, and Claude 3. He emphasizes high resolution for screenshots, OCR, web agents, and document tasks. For generation, he connects autoregressive image tokenization (CogView, DALL·E-like, Parti-like) to diffusion (Relay Diffusion, CogView3), transformer diffusion (DiT, MM-DiT), and video generation (CogVideo, Sora-like scaling).

✓ Check yourself — why might training all LLM weights during visual alignment hurt?

If the visual data distribution is narrow, updating the full LLM can overwrite or distort language-only behavior. Freezing, partial tuning, LoRA, or vision-specialist parameters help add perception while preserving the pretrained language skill.

5.Worked example — CLIP-style image-text matching

Numbers make the bridge concrete. Suppose one image embedding \(v\) is compared against three caption embeddings \(t_1,t_2,t_3\). The model scores them by cosine similarity, then softmax turns scores into probabilities.

Step-by-step with real numbers

Cosine similarity is

$$s(v,t)=\frac{v\cdot t}{\|v\|\,\|t\|}.$$

For one numeric normalization, take \(v=(1,2)\) and \(t_2=(2,3)\):

$$v\cdot t_2=1\cdot2+2\cdot3=8,$$

$$\|v\|=\sqrt{1^2+2^2}=\sqrt5\approx2.236,\qquad \|t_2\|=\sqrt{2^2+3^2}=\sqrt{13}\approx3.606,$$

$$s(v,t_2)=\frac{8}{2.236\cdot3.606}=\frac{8}{8.062}\approx0.99.$$

Now suppose the trained encoders produce three final image-caption similarities:

$$[s_1,s_2,s_3]=[0.31,0.82,0.44].$$

With temperature \(\tau=0.1\), divide scores by \(\tau\):

$$[3.1,8.2,4.4].$$

Softmax gives

$$p_j=\frac{e^{s_j/\tau}}{\sum_k e^{s_k/\tau}},$$

so the unnormalized weights are approximately \([22.20,3637.97,81.45]\), the sum is \(3741.62\), and

$$p\approx[0.006,0.972,0.022].$$

The image matches caption 2. Temperature made the already-best caption decisively best.

✓ Check yourself — if \(\tau\) were 1 instead of 0.1, would caption 2 still win?

Yes. Softmax preserves the ranking because 0.82 is still the largest score, but the distribution is less sharp: \(\operatorname{softmax}([0.31,0.82,0.44])\approx[0.27,0.45,0.30]\).

6.End-to-end trace — one LLaVA-style question

Follow one concrete input: an image plus the question, "What is written on the sign?" The important state is the shape of the representation at each stage.

What the trace reveals

The unifying trick is not mystical: the image becomes tokens the LLM understands. Once the connector has translated visual evidence into \(d_{\text{LLM}}\)-dimensional vectors, the same decoder machinery can condition on both modalities.

✓ Check yourself — where does OCR ability enter this pipeline?

OCR depends on high-resolution visual features that preserve small text, training examples that reward reading text correctly, and a connector/LLM that can bind those visual features to answer tokens. If resolution collapses the letters, the LLM cannot recover them by reasoning alone.

7.Capabilities, evals, and limitations

Once the image stream reaches the LLM, many tasks become promptable. The same model can answer a question about a chart, caption a photo, identify a UI button, or reason over a document screenshot — but that flexibility makes evaluation harder.

7.1 What we ask LMMs to do

  • VQA — answer questions about images, diagrams, scenes, or documents.
  • Captioning — summarize visible content in natural language.
  • OCR and document understanding — read text in screenshots, PDFs, tables, receipts, forms, and handwritten notes.
  • Grounding — connect phrases to boxes, regions, coordinates, or UI elements.
  • Chart and scientific figure understanding — extract trends, compare values, and avoid confusing visual marks.
  • Web and agent tasks — Ding's CogAgent example uses screenshots to decide where to click or type.

7.2 Where they still break

Common misconception

"If an LMM gives a fluent answer, it must have looked carefully." Fluency is inherited from the LLM. Visual grounding can still fail: the model may hallucinate objects, misread small text, count incorrectly, or answer from language priors rather than pixels.

  • Visual hallucination — naming objects or facts not present in the image.
  • Resolution and detail — small text, dense charts, and web screenshots require many visual tokens or high-resolution modules.
  • Modality fusion — the model may ignore the image when the text prior is strong, or ignore the question when visual features dominate.
  • Evaluation gaps — multiple valid phrasings, ambiguous images, and missing grounding annotations make automatic scoring brittle.
  • Video and physical understanding — Ding predicts video understanding will matter, but notes that learning physical rules may need better self-supervised signals than annotated text-video pairs alone.
Research taste from the lecture

Ding's advice is pragmatic: multimodal progress is increasingly about data quality, high-resolution systems, benchmarks, and infra. Architecture still matters, but a clever connector without the right data and evaluation loop will not become a reliable visual assistant.

✓ Check yourself — name two reasons high resolution matters for LMMs.

It preserves small visual evidence such as text, icons, table cells, and chart ticks; and it supports grounding actions in screenshots or documents where a tiny region can change the answer.

Glossary

TermPlain meaning
Large multimodal model (LMM)An assistant model that can condition on more than text, usually images plus text, and produce language outputs.
Vision encoderA model, often CLIP ViT, that converts pixels into feature vectors.
Visual tokenA projected image-patch feature placed in the LLM's embedding sequence.
Connector / projectorThe trainable bridge that maps vision features into the LLM hidden dimension.
CLIPContrastive Language-Image Pretraining; learns aligned image and text embeddings from paired data.
Q-FormerBLIP-2's query transformer that extracts a compact set of visual features for the LLM.
Gated cross-attentionA module that lets language tokens attend to visual features while controlling how strongly vision changes the LLM path.
Visual instruction tuningFine-tuning on image-question-answer conversations so the model follows multimodal instructions.
GroundingLinking language to image regions, boxes, coordinates, or screen elements.
Visual hallucinationA fluent but visually unsupported answer about an object, attribute, count, or relation.
